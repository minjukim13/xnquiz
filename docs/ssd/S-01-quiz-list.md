## 5. 기능 분류

### 5-1. [A] Canvas 기존 기능 - 기존 Canvas Classic Quizzes 제공 기능 재구현

| **ID** | **기능** | **동작** |
|---|---|---|
| A-01-1 | 퀴즈 목록 조회 | 과목에 속한 모든 퀴즈를 카드 형태로 조회 |
| A-01-2 | 퀴즈 생성 진입 | `새 퀴즈` 버튼 → `/quiz/new` 이동 |
| A-01-3 | 퀴즈 편집 진입 | 카드 메뉴 → `편집` → `/quiz/:id/edit` |
| A-01-4 | 퀴즈 삭제 | 카드 메뉴 → `삭제` → 확인 다이얼로그 → 삭제 처리 |
| A-01-5 | 퀴즈 복사 | 카드 메뉴 → `복사` → 대상 과목 선택 → 신규 퀴즈 생성 + 문항 복제 |
| A-01-6 | 학생/교수자 역할별 화면 분기 | 로그인 역할에 따라 다른 목록 렌더 |
| A-01-7 | 학생 - 응시 가능 퀴즈 조회 | `draft` 제외 + `visible !== false` 필터 |
| A-01-8 | 학생 - 응시 이력 표시 | 카드에 본인 attempt 의 점수/제출일 표시 |
| A-01-9 | 학생 공개/비공개 토글 | 카드 메뉴에서 1클릭 토글 (Canvas published/unpublished 와 동일 개념) |
| A-01-10 | 임시저장 - 학생 비공개 자동 강제 | `draft` 상태는 visible 토글과 무관하게 항상 비공개 표시 (Canvas unpublished 와 동일) |
| A-01-11 | 지각 제출 안내 라벨 | 지각 제출 허용 시 "지각 제출: yyyy-mm-dd 까지" 또는 "무제한 허용" 표기 |
| A-01-12 | 이용 종료(lockDate) 학생 카드 처리 | 학생 화면에서 lockDate 경과 시 카드 자체를 lock 표시로 대체 |

### 5-2. [B] 학교 요구사항 반영 - Canvas 에 없거나 부족하여 학교가 요구한 신규 기능

| **ID** | **라벨** | **기능** | **동작** |
|---|---|---|---|
| F-01-1 | [B-#12] | 시험 전용 목록 + 주차/차시 동시 노출 | 카드에 `n주차 m차시` 배지 표시. Canvas 시험 메뉴는 주차 정보 없음 |
| F-01-2 | [B-#12] | 주차/차시 필터링 | 주차 선택 시 차시 옵션이 동적으로 좁혀짐 |
| F-01-3 | [B-#03] | 다른 과목에서 퀴즈 가져오기 | `가져오기` 버튼 → 과목/퀴즈 다중 선택 → 일괄 복사. LTI 모드는 Canvas REST 로 교수자 담당 과목 전체 조회 |
| F-01-4 | [B-#03] | 복사/가져오기 후 자동 초기화 안내 | 복사 시 주차/응시기간/공개정책/지각제출/접근제어를 자동 초기화하고, 안내 박스로 사전 고지 |

### 5-3. [C] 자체 도출 개선 - Canvas 기존 기능도 아니고 학교 요구도 아닌, 프로젝트 자체 발굴 개선

|---|---|---|---|

| F-01-6 | [C] | 3단계 상태 배지 (초안/진행중/마감) | 퀴즈 상태값 + 마감일 자동 판정으로 표시 상태 결정 |
| F-01-7 | [C] | D-day 배지 | 진행중 퀴즈의 마감일까지 남은 일 수를 `D-N` 형식으로 표시. 당일은 빨강, 그 외는 주황 |
| F-01-8 | [C] | 인라인 통계 (응시율/응시인원/미제출/평균점수) | 카드 우측에 4개 핵심 지표 노출 (전체 수강생 기준), 미제출 0 초과 시 빨강 강조 |
| F-01-9 | [C] | 카드 전체 클릭으로 상세 진입 | 카드 어디든 클릭하면 상세로 이동. 카드 안 메뉴(점 3개)는 별도 동작으로 분리 |
| F-01-10 | [C] | 미리보기 진입 | 카드 메뉴 → `미리보기` → `/quiz/:id/attempt?preview=true` (교수자 전용 학생 뷰) |
| F-01-11 | [C] | 채점/통계 직접 진입 | 카드 메뉴 → `채점` / `통계` (응시 가능 상태일 때만 노출) |
| F-01-12 | [C] | 퀴즈 전역 설정 진입점 | 헤더 톱니바퀴 → 전역 설정 다이얼로그 (전 퀴즈 공통 설정) |

## 9. 예외 / 엣지 케이스

- 임시저장(`draft`) 퀴즈는 학생에게 노출 금지 (StudentQuizList 에서 필터)
- 임시저장 퀴즈는 카드 메뉴에서 공개/숨기기 비활성 + "임시저장 상태에선 자동 비공개입니다" 안내
- 예약 발행(`startDate` 미래)은 `scheduled` 상태로 별도 표시, D-day/통계 비노출
- 마감일 경과 시 DB status 는 그대로 `open` 이지만 화면 상태만 `closed` 로 표시 (resolveDisplayStatus)
- 지각 제출 허용 + lateSubmitDeadline 미설정 시 "무제한 허용" 표기
- 학생 화면에서 lockDate 경과 시 카드 내용 자체를 차단 (isLockDatePassed)
- LTI 모드에선 서버가 과목 필터링 수행 → 클라이언트 `course === CURRENT_COURSE` 체크 skip
- 가져오기 모달에서 LTI 모드는 Canvas REST 로 교수자 담당 과목 전체 조회 (mock 모드는 MOCK_COURSES 사용)
