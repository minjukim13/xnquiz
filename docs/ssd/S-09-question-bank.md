# S-09. 문제은행 상세

> **라벨 규칙**: `[A]` Canvas 기존 기능 / `[B-#NN]` 학교 요구사항 신규 / `[C]` 자체 도출 개선

## 1. 화면 정보

| **항목** | **내용** |
|---|---|
| 화면 ID | S-09 |
| 라우트 | `/question-banks/:bankId` |
| 진입 권한 | 교수자 (학생 접근 시 자동 리다이렉트) |
| 진입 경로 | 문제은행 목록(S-08) 카드 클릭 |

## 2. 화면 목적

- 단일 문제은행 안의 문항을 CRUD (생성/수정/삭제/순서 변경)
- 일괄 업로드(엑셀/CSV) 로 대량 문항 등록 효율화
- 유형/난이도/검색 필터로 빠른 탐색

## 3. 영역 구성 (Region Map)

| **영역** | **설명** | **주요 컴포넌트** |
|---|---|---|
| 헤더 | PageHeader (은행명 인라인 수정 + 설명 + 일괄업로드/문항추가 버튼) | PageHeader |
| 필터 / 검색 | 유형 필터 + 난이도 필터 + 검색 입력 + 결과 수 표시 | DropdownSelect, Search |
| 본문 | 문항 목록 (드래그 핸들 + 유형 배지 + 점수 + 난이도 배지 + 텍스트 미리보기 + 삭제) | Card, TypeBadge, Badge |
| 빈 상태 | 문항 0개 또는 검색 결과 0건 | 안내 문구 |
| 모달 | 신규/편집 문항 / 일괄 업로드 / 삭제 확인 | AddQuestionModal, ExcelUploadModal, ConfirmDialog |

## 4. 기능 분류

### 4-1. [A] Canvas 기존 기능 - 기존 Canvas 제공 기능 재구현

| **ID** | **기능** | **동작** | **구현 위치** |
|---|---|---|---|
| A-09-1 | 문항 목록 조회 | 단일 은행의 모든 문항 표시 | QuestionBank |
| A-09-2 | 문항 생성 | "문항 추가" → AddQuestionModal (신규 모드) | AddQuestionModal |
| A-09-3 | 문항 수정 | 문항 클릭 → AddQuestionModal (편집 모드) | AddQuestionModal |
| A-09-4 | 문항 삭제 | 휴지통 → ConfirmDialog → deleteQuestion | ConfirmDialog |
| A-09-5 | 유형/난이도 필터 | 드롭다운 → 실시간 filtered 갱신 | DropdownSelect |
| A-09-6 | 검색 | 본문 키워드 → 실시간 필터 | Search |

### 4-2. [B] 학교 요구사항 반영 - Canvas 에 없거나 부족하여 학교가 요구한 신규 기능

| **ID** | **라벨** | **기능** | **동작** | **구현 위치** |
|---|---|---|---|---|
| F-09-1 | [B-#01] | 문항 단위 난이도 메타 | 문항별 난이도 등록 + 카드에 배지 표시 | Badge |
| F-09-2 | [B-#01] | 은행 난이도 우선 동기화 | 은행에 난이도가 지정된 경우 하위 문항 난이도 자동 동기화 | difficulty 동기화 로직 |
| F-09-3 | [B-#04] | 일괄 업로드 (엑셀/CSV) | 양식 다운로드 → 파싱 → 검증 후 일괄 등록 | ExcelUploadModal |
| F-09-4 | [B-#06] | 문항 제목 별도 설정 | 본문과 별개로 짧은 제목 입력 (학생 화면 노출용) | AddQuestionModal title |
| F-09-5 | [B-#05] | 정답 판정 옵션 (대소문자/띄어쓰기) | 단답형 정답 비교 시 대소문자/공백 무시 옵션 | AddQuestionModal 옵션 |

### 4-3. [C] 자체 도출 개선 - Canvas 기존 기능도 아니고 학교 요구도 아닌, 프로젝트 자체 발굴 개선

| **ID** | **라벨** | **기능** | **동작** | **구현 위치** |

| F-09-7 | [C] | 드래그로 문항 순서 변경 | 핸들 드래그 → from/to 추적 → 순서 갱신. 필터 적용 중에는 비활성 | dragIndexRef |

## 5. 화면 상태 (State)

| **상태** | **트리거** | **표현** |
|---|---|---|
| 로딩 | 초기 fetch 중 | 스켈레톤 |

## 7. 예외/엣지 케이스

- 은행 미존재 (URL 직접 접근): `/question-banks` 리다이렉트
- 학생 접근: `/` 리다이렉트
- 필터/검색 적용 중 드래그 시도: 비활성 (순서 변경 금지)
- 일괄 업로드 시 정답 형식 오류: 오류 행 표시 + 양식 다운로드 링크 제공
- 은행 난이도 지정 + 문항 난이도 충돌: 은행 난이도 우선 적용
- 일괄 업로드 후 다수 문항 생성: 토스트에 생성 개수 명시

## 8. 미구현 / Open Issue

- 문항 멀티 선택 + 일괄 삭제/이동 미구현 (현재는 단건만)
- 문항 단위 권한 공유 (다른 교수자에게 일부 문항만 공유) 미구현
- 일괄 업로드 시 매뉴얼 채점 옵션 (서술형 등) 양식 미정 - 현재는 자동 채점 가능 유형만
