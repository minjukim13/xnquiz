# S-09. 문제은행 상세

> **라벨 규칙**: `[A]` Canvas 기존 기능 / `[B-#NN]` 학교 요구사항 신규 / `[C]` 자체 도출 개선

## 1. 화면 정보

| **항목** | **내용** |
|---|---|
| 화면 ID | S-09 |
| 라우트 | `/question-banks/:bankId` |
| 진입 권한 | 교수자 (학생 접근 시 자동 리다이렉트) |
| 진입 경로 | 문제은행 목록(S-08) 카드 클릭 |

## 2. 화면 목적

- 단일 문제은행 안의 문항을 CRUD (생성/수정/삭제/순서 변경)
- 일괄 업로드(엑셀/CSV) 로 대량 문항 등록 효율화
- 유형/난이도/검색 필터로 빠른 탐색

## 3. 영역 구성 (Region Map)

| **영역** | **설명** | **주요 컴포넌트** |
|---|---|---|
| 헤더 | PageHeader (은행명 인라인 수정 + 설명 + 일괄업로드/문항추가 버튼) | PageHeader |
| 필터 / 검색 | 유형 필터 + 난이도 필터 + 검색 입력 + 결과 수 표시 | DropdownSelect, Search |
| 본문 | 문항 목록 (드래그 핸들 + 유형 배지 + 점수 + 난이도 배지 + 텍스트 미리보기 + 삭제) | Card, TypeBadge, Badge |
| 빈 상태 | 문항 0개 또는 검색 결과 0건 | 안내 문구 |
| 모달 | 신규/편집 문항 / 일괄 업로드 / 삭제 확인 | AddQuestionModal, ExcelUploadModal, ConfirmDialog |

## 4. 기능 분류

### 4-1. [A] Canvas 기존 기능 - 기존 Canvas 제공 기능 재구현

| **ID** | **기능** | **동작** |
|---|---|---|
| A-09-1 | 문항 목록 조회 | 단일 은행의 모든 문항 표시 |
| A-09-2 | 문항 생성 | "문항 추가" → AddQuestionModal (신규 모드) |
| A-09-3 | 문항 수정 | 문항 클릭 → AddQuestionModal (편집 모드) |
| A-09-4 | 문항 삭제 | 휴지통 → ConfirmDialog → deleteQuestion |
| A-09-5 | 유형/난이도 필터 | 드롭다운 → 실시간 filtered 갱신 |
| A-09-6 | 검색 | 본문 키워드 → 실시간 필터 |
| A-09-7 | 문항 드래그 정렬 | 핸들 드래그 → from/to 추적 → 순서 갱신. 필터 적용 중에는 비활성 |

### 4-2. [B] 학교 요구사항 반영 - Canvas 에 없거나 부족하여 학교가 요구한 신규 기능

| **ID** | **라벨** | **기능** | **동작** |
|---|---|---|---|
| F-09-1 | [B-#01] | 문항 단위 난이도 메타 | 문항별 난이도 등록 + 카드에 배지 표시 |
| F-09-2 | [B-#01] | 은행 난이도 우선 동기화 | 은행에 난이도가 지정된 경우 하위 문항 난이도 자동 동기화 |
| F-09-3 | [B-#04] | 일괄 업로드 (엑셀/CSV) | 양식 다운로드 → 파싱 → 검증 후 일괄 등록 |
| F-09-4 | [B-#06] | 문항 제목 별도 설정 | 본문과 별개로 짧은 제목 입력 (학생 화면 노출용) |
| F-09-5 | [B-#05] | 정답 판정 옵션 (대소문자/띄어쓰기) | 단답형 정답 비교 시 대소문자/공백 무시 옵션 |

### 4-3. [C] 자체 도출 개선 - Canvas 기존 기능도 아니고 학교 요구도 아닌, 프로젝트 자체 발굴 개선

| **ID** | **라벨** | **기능** | **동작** |
|---|---|---|---|
| F-09-6 | [C] | 문항 작성 모달 - 우측 예시 문제 동시 노출 | 좌측 유형 선택 시 우측에 해당 유형의 예시 문제 즉시 표시 |
| F-09-7 | [C] | 일괄 업로드 검증 + 오류 행 별도 표시 | 배점/유형/필수값 등 검증, 오류 행만 표시 + 양식 다운로드 링크 |
| F-09-8 | [C] | 은행명 인라인 수정 | PageHeader 안에서 직접 수정 (Enter/Escape/Blur) |
| F-09-9 | [C] | 결과 수 표시 | 필터/검색 적용 시 "n개 문항 / 전체 m개" 표기 |
| F-09-10 | [C] | 미리보기 (학생 시점) | 문항 클릭 시 우측 상세에 학생이 보게 될 형태 미리보기 |

## 5. 화면 상태 (State)

| **상태** | **트리거** | **표현** |
|---|---|---|
| 로딩 | 초기 fetch 중 | 스켈레톤 |

| 모달 (신규/편집/업로드/삭제) | 버튼/문항/휴지통 트리거 | 각 모달 활성 |

## 7. 예외/엣지 케이스

- 은행 미존재 (URL 직접 접근): `/question-banks` 리다이렉트
- 학생 접근: `/` 리다이렉트
- 필터/검색 적용 중 드래그 시도: 비활성 (순서 변경 금지)
- 일괄 업로드 시 정답 형식 오류: 오류 행 표시 + 양식 다운로드 링크 제공
- 고정 난이도가 지정된 은행에서 문항 추가/편집: 난이도 선택 UI 가 비활성화되어 "상/중/하 고정" 으로 표시됨 (사용자가 다른 값으로 변경 불가)
- 고정 난이도 은행에 엑셀로 다른 난이도 문항을 업로드한 경우: 해당 행은 등록 차단되고 결과 모달에 행 번호 + 사유 표시. 사용자가 엑셀을 수정 후 재업로드 (자동 통일 금지)
- 엑셀 업로드 결과 모달의 오류 목록이 길어지는 경우: 모달 최대 높이를 화면의 85% 로 제한하고 오류 영역만 내부 스크롤
- 일괄 업로드 후 다수 문항 생성: 토스트에 생성 개수 명시
