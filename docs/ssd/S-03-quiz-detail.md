# S-03. 퀴즈 상세

> **라벨 규칙**: `[A]` Canvas 기존 기능 / `[B-#NN]` 학교 요구사항 신규 / `[C]` 자체 도출 개선

## 1. 화면 정보

| **항목** | **내용** |
|---|---|
| 화면 ID | S-03 |
| 라우트 | `/quiz/:id` |
| 진입 권한 | 교수자 (관리 뷰) / 학생 (응시 진입 + 결과 확인 뷰) |

## 2. 화면 목적

- **교수자**: 퀴즈의 모든 메타데이터를 한 화면에서 검토하고, 편집/채점/통계 등 후속 액션으로 이동
- **학생**: 퀴즈 응시 전 안내 확인, 응시하기 / 재응시 시작, 응시 후 본인 결과 + 교수자 코멘트 조회

## 3. 영역 구성 (Region Map)

| **영역** | **설명** | **주요 컴포넌트** |

| 설명 카드 | 퀴즈 설명 (RichText) | Card |

## 4. 기능 분류

### 4-1. [A] Canvas 기존 기능 - 기존 Canvas 제공 기능 재구현

| **ID** | **기능** | **동작** |
|---|---|---|
| A-03-1 | 퀴즈 메타 조회 | 제목/설명/일정/제한시간/재응시/공개정책/지각/접근제어 표시 |
| A-03-2 | 학생 - 응시하기 진입 | "응시하기" 버튼 → `/quiz/:id/attempt` |
| A-03-3 | 학생 - 재응시 진입 | 응시 횟수 남았을 때 "재응시" 버튼 활성 |
| A-03-4 | 교수자 - 편집 진입 | DropdownMenu → 편집 → `/quiz/:id/edit` |
| A-03-5 | 교수자 - 삭제 | DropdownMenu → 삭제 → ConfirmDialog → 목록으로 리다이렉트 |
| A-03-6 | 교수자 - 미리보기 진입 | 학생 시점 응시 화면을 실제 기록 없이 확인 (`?preview=true`) |
| A-03-7 | 학생 결과 조회 | 점수 + 문항별 정답/오답/부분점수 표시 |
| A-03-8 | 학생 - 비공개 정책 처리 | scoreReveal 정책에 따라 점수/응답 노출 제어 |
| A-03-9 | 비공개 (visible=false) 명시 | 공개되지 않은 퀴즈는 교수자 화면에서 "숨김" 라벨 (Canvas published/unpublished 와 동일 개념) |
| A-03-10 | 학생 - 1회 조회(oneTimeResults) 정책 명시 | 1회 조회 후 응답·정답 비공개. 사전에 안내 + 조회 즉시 마크 (Canvas one_time_results 옵션과 동일) |
| A-03-11 | 학생 - 지각 제출 결과 라벨 | 본인이 지각 제출했을 경우 결과 화면에 "지각 제출" 명시 (Canvas 의 late submission 상태 표시) |
| A-03-12 | 교수자 - lockDate 종료 표시 | 이용 종료일 경과 시 "(종료됨)" 표기 (Canvas 의 lock_at 상태 표시) |
| A-03-13 | 마감 자동 전환 표시 | DB status 가 open 이지만 dueDate 경과 시 화면만 closed 로 표시 (Canvas due_at 경과 후 자동 잠금 표현) |
| A-03-14 | 학생 - 서술형 채점 대기 안내 | 자동 채점 외 항목이 있을 때 "채점 대기" 라벨 (Canvas needs_grading 상태 표시) |

## 5. 화면 상태 (State)

| **상태** | **트리거** | **표현** |
|---|---|---|
| 로딩 | API 모드 + 데이터 fetch 중 | 스켈레톤 |
| draft (교수자) | status=draft | 편집/미리보기/통계 비활성, "임시저장" 배지 |
| scheduled (양측) | status=open + startDate 미래 | "예약 발행" 배지, 응시 버튼 비활성 |
| open (학생) | status=open + 기간 내 + 횟수 남음 | "응시하기" / "재응시" 활성 |
| open (교수자) | status=open | 채점/통계/편집 모두 활성 |
| closed (양측) | dueDate 경과 또는 status=closed | 응시 비활성 (학생), 채점/통계 활성 (교수자) |
| 결과 보기 (학생) | 응시 + 점수 공개 정책 충족 | 점수 + 문항별 결과 + 코멘트 노출 |
| 비공개 (학생) | scoreReveal 정책 미충족 | "공개 예정" / "공개 안 함" 배지 |
| oneTimeResults 조회 후 | isResultViewed = true | 응답·정답 영역 마스킹, 점수만 노출 |

## 7. 예외/엣지 케이스

- 예약 발행(scheduled): 학생은 응시 진입 차단, 교수자는 미리보기 가능
- lockDate 경과: 학생은 카드/상세 모두 잠금 표시, 교수자는 편집/통계 여전히 가능
- visible=false: 학생 목록에 노출 안 됨, 직접 URL 접근 시 차단
- oneTimeResults: 1회 조회 후 응답/정답은 마스킹되지만 점수만은 영구 노출
- 재응시 횟수 초과: "응시 횟수를 모두 사용했습니다" 안내 + 버튼 비활성
- 비공개 점수 정책: 학생 화면에서 점수 영역 자체를 숨기고 안내 라벨로 대체
- 추가 기간 부여 학생: 본인의 마감일을 별도로 표시 (assignments 매칭)
