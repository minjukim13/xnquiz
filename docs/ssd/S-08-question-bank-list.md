## 2. 화면 목적

- 교수자가 본인 과목의 모든 문제은행을 한 화면에서 조회/생성/복사/내보내기/가져오기

## 3. 레이아웃 (Region Map)

```
[헤더: "문제모음" 타이틀 | 가져오기 | 내보내기 | 새 문제모음]
[본문: 카드 그리드 (1/2/3열 반응형)]
  └── 카드
       ├── 은행명 (인라인 수정 가능)
       ├── 난이도 배지
       ├── 문항 수
       ├── 최종 수정일
       └── 카드 메뉴 (복사 / 삭제)
[빈 상태: 은행 0건 → EmptyState + "새 문제모음 만들기"]
[모달: 새 은행 생성 / 가져오기 / 내보내기 / 삭제 확인]
```

| 헤더 | 타이틀 + 액션 3종 |

## 4. 사용 컴포넌트 (DS Baseline 참조)

| **컴포넌트** | **variant / state** | **용도** |
|---|---|---|

| Card | bank-card / default·hover | 문제은행 카드 |
| Input | inline-edit | 은행명 인라인 수정 |
| Badge | difficulty (easy / normal / hard) | 난이도 배지 |

| Dialog | add-bank / import / export / confirm | 4종 모달 |
| Skeleton | bank-card | 로딩 placeholder |
| EmptyState | no-bank | 은행 0건 |
| Toast | success-with-action (복사 후 바로가기) | 결과 안내 |

## 5. 기능 분류

### 5-1. [A] Canvas 기존 기능 - 기존 Canvas Classic Quizzes 제공 기능 재구현

| **ID** | **기능** | **동작** |
|---|---|---|
| A-08-1 | 문제은행 목록 조회 | 본인 과목의 은행 카드 그리드 표시 |
| A-08-2 | 새 문제은행 생성 | "새 문제모음" → 모달 → 은행명/난이도 입력 → 생성 |
| A-08-3 | 문제은행 삭제 | 카드 휴지통 → ConfirmDialog → 삭제 |
| A-08-4 | 문제은행 진입 | 카드 클릭 → 상세 (S-09) |

### 5-2. [B] 학교 요구사항 반영 - Canvas 에 없거나 부족하여 학교가 요구한 신규 기능

| **ID** | **라벨** | **기능** | **동작** |
|---|---|---|---|
| F-08-1 | [B-#01] | 그룹 단위 난이도 메타 표시 | 카드에 난이도 배지 (쉬움/보통/어려움) |
| F-08-2 | [B-#01] | 최종 수정일 표시 | 카드에 "n일 전 수정" 라벨 |
| F-08-3 | [B-#02] | 문제은행 그룹 복사 | 카드 메뉴 → "복사" → 새 은행 + 모든 문항 복사 |
| F-08-4 | [B-#04] | 다른 과목에서 가져오기 | "가져오기" → 과목 선택 → 은행 다중 선택 → 일괄 import |
| F-08-5 | [B-#04] | 다른 과목으로 내보내기 | "내보내기" → 대상 과목 선택 → 은행 + 문항 export |

### 5-3. [C] 자체 도출 개선 - Canvas 기존 기능도 아니고 학교 요구도 아닌, 프로젝트 자체 발굴 개선

| **ID** | **라벨** | **기능** | **동작** |
|---|---|---|---|
| F-08-6 | [C] | 은행명 인라인 수정 | Pencil 아이콘 → 카드 안에서 직접 수정 (Enter/Escape/Blur) |
| F-08-7 | [C] | 복사 후 즉시 진입 토스트 | 복사 완료 시 "바로가기" 액션이 포함된 토스트 |
| F-08-8 | [C] | 문항 수 카운트 카드 표시 | 은행별 문항 수를 카드에 즉시 노출 |
| F-08-9 | [C] | 1/2/3열 반응형 카드 그리드 | 모바일 1열 → 태블릿 2열 → 데스크톱 3열 |

## 8. 비정상 상태 UX

| **상태** | **트리거** | **표현** |
|---|---|---|
| 로딩 | 초기 fetch 중 | Skeleton 카드 × 3 |
| 빈 상태 | 은행 0건 | EmptyState + "새 문제모음 만들기" CTA |
| 정상 | 1건 이상 | 카드 그리드 |
| 인라인 편집 중 | 은행명 클릭 | 입력 필드로 변경, Enter 저장 / Escape 취소 |
| 복사 진행 | 복사 트리거 | 토스트 (바로가기 액션 포함) + 즉시 반영 |
| 에러 | 목록 fetch / 생성 / 복사 / 삭제 실패 | ErrorState + 재시도 / 토스트 |
| 권한 없음 | 학생 접근 | 즉시 `/` 리다이렉트 |
| 오프라인 | 네트워크 단절 | 전역 OfflineBanner, 생성·복사 버튼 비활성 |

## 9. 예외 / 엣지 케이스

- 학생 접근: 즉시 `/` 리다이렉트
- 빈 상태: EmptyState + 새 문제모음 만들기 CTA
- 복사 중 오류: 토스트 안내
- 가져오기 시 동일 은행명: 자동 suffix ("(복사본)") 추가
- 은행명 빈 값으로 저장 시도: 검증 차단
