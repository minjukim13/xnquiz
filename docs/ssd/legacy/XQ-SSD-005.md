## 1. 역할별 네비게이션 구조

```
교수자:
- 전역 기본값: 퀴즈 목록 헤더 톱니바퀴 → QuizSettingsDialog → "정답 판정" 절
- 문항 단위: 문항 작성/편집 모달 (AddQuestionModal) 안의 입력형 유형 영역 → 4종 옵션 (현재 미구현, 간극)

학생:
- 응시 결과 확인 화면에서 오답 시 적용된 판정 기준 안내 (현재 미구현, 간극)

운영자:
- 응대 시점에 문항에 저장된 판정 기준 확인 (현재 권한 분기 미구현)
```

## 3. 화면별 상세 설계

### SCR-I-GLOBAL-ANSWER-POLICY. 전역 정답 판정 기본값

**구현 파일**: `src/components/QuizSettingsDialog.jsx` 의 "정답 판정" 절

**레이아웃**

```
[QuizSettingsDialog → "정답 판정" SettingsSection]
  └── 입력형 문항(단답형 / 빈칸 채우기 / 복수 빈칸) 정답 판정 기본값
       ├── 대소문자 (Toggle: 허용 / 구분)
       ├── 앞뒤 공백 (Toggle: 허용 / 구분)
       ├── 연속 공백 (Toggle: 허용 / 구분)
       └── 모든 공백 제거 (Toggle: 허용 / 구분)
```

**사용 컴포넌트**: `Dialog`, `SettingsSection` (내부), `Switch`, 안내 카피.

**상태**

| **상태** | **표현** |
|---|---|
| 기본값 (저장 없음) | 대소문자 무시 + 앞뒤 공백 무시 + 연속 공백 무시 + 모든 공백 제거 OFF (Canvas 기본값) |
| 사용자 변경 후 | 변경된 토글이 신규 문항 작성 시 기본값으로 사용됨 (UX-P07-002) |
| 변경 적용 범위 | 본 옵션은 신규 문항에만 적용. 기존 문항의 채점 결과 변경 없음 (UX-P07-004 호환 기준) |

---

### SCR-I-QFORM-INPUT-ANSWER. 입력형 문항 정답 판정 옵션 (문항 단위)

**구현 파일**: `src/components/AddQuestionModal.jsx` 안의 short_answer / fill_in_multiple_blanks 유형 입력 영역

**현재 프로토타입 동작**

문항 단위 4종 옵션 개별 설정은 **현재 미구현** (URD-005 PD 결정 1차 진행 후 후속 작업). 전역 기본값(SCR-I-GLOBAL-ANSWER-POLICY) 이 모든 입력형 문항에 일괄 적용 → "간극" 절 G-1.

**레이아웃 (목표 동작, 미구현)**

```
[AddQuestionModal - short_answer / fill_in_multiple_blanks 유형]
  ├── 본문 / 배점 / 난이도 / 정답 후보 (SSD-024 / SSD-028)
  └── 정답 판정 옵션 영역 (목표)
       ├── 4종 Toggle (대소문자 / 앞뒤 공백 / 연속 공백 / 모든 공백 제거)
       ├── 기본값 표시 ("전역 기본값 적용 중")
       └── 응시자 있음 + 변경 시 → 영향 안내 (UX-P07-010 / 011)
```

---

### SCR-L-RESULT-ANSWER-POLICY. 학생 응시 결과 - 적용 판정 기준 안내

**현재 프로토타입 동작**

응시 결과 화면에서 오답 시 적용된 판정 기준 안내는 **현재 미구현** → "간극" 절 G-2.

**목표 동작 (미구현)**

```
[학생 응시 결과 화면 안의 오답 문항]
  ├── 본인 답안 표시
  ├── 정답 후보 표시 (성적 공개 정책 허용 시)
  └── 적용된 판정 기준 카피
       └── "대소문자 / 공백 차이로 오답 처리됨" 같은 학생 친화적 표현 (UX-P08-002)
```
