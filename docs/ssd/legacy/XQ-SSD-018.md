## 1. 역할별 네비게이션 구조

```
교수자 (instructor):
홈 → 퀴즈 목록 (/) → [필터/정렬/검색] → 카드 클릭 → 퀴즈 상세 (/quiz/:id)
                                                          ├ 5그룹 분리 (주요/응시 조건/응시 정책/성적 공개/접근 제한)
                                                          ├ 문항 구성 영역
                                                          └ 작업 진입 (편집 / 미리보기 / 채점 / 통계 / 응시 모니터 / 삭제)
     ├ "새 퀴즈" → /quiz/new (SSD-008)
     ├ "가져오기" → 가져오기 모달
     └ 톱니바퀴 → QuizSettingsDialog (전역 기본값 설정)

학생 (student):
홈 → 퀴즈 목록 (/) → 카드 클릭 → 학생 퀴즈 상세 (/quiz/:id, 학생 뷰)
                                  ├ 응시 정보 (간단)
                                  ├ 본인 응시 결과 (점수 / 다회 응시 / 회차별 공개)
                                  ├ 접근 제한 / 결과 미공개 / 채점 대기 안내
                                  └ 응시 시작 또는 결과 확인

핵심 태스크 클릭 뎁스:
- 교수자: 채점 진입 = 퀴즈 목록 → 카드 메뉴 → 채점 (3단계)
- 학생: 응시 시작 = 퀴즈 목록 → 카드 → 응시 시작 (3단계)
```

**도달 원칙**

- 단일 라우트(`/quiz/:id`) 가 역할별 분기. instructor 는 QuizDetail.jsx 의 교수자 뷰, student 는 학생 뷰 렌더.
- 진행 상태와 공개 상태는 별도 컴포넌트(StatusBadge + VisibilityBadge) 로 분리 표시 (UX-COM-002).
- 빈 상태 분기: 시험 없음 / 조건 결과 없음 / 학생 응시 가능 시험 없음 3종.

### SCR-I-QUIZLIST. 시험 목록 (교수자)

**구현 파일**: `src/pages/QuizList.jsx`

**레이아웃**

```
[헤더]
  ├── h1 "퀴즈 관리"
  └── 우측 액션: 톱니바퀴 (QuizSettingsDialog) + "가져오기" + "새 퀴즈"

[필터/정렬 행]
  ├── WeekSessionFilter (주차 / 차시 드롭다운, 미지정 옵션 포함)
  └── 정렬 드롭다운 (최근 생성 / 주차 오름차순 / 주차 내림차순 / 마감 임박순)

[카드 그리드]
  └── 퀴즈 카드 × N (CS301 코스 시험)
       ├── 카드 헤더 (제목 + StatusBadge + D-day 배지)
       ├── 카드 메타 (주차/차시 배지 / 응시 기간 / 지각 정책 / 공개 여부 VisibilityBadge)
       ├── 인라인 운영 성과 (응시율 / 응시 인원 / 미제출 / 평균 점수)
       └── 카드 메뉴 (점 3개)
            ├── 편집 (status === 'draft' || (instructor && !응시자))
            ├── 미리보기
            ├── 복사
            ├── 학생에게 공개 / 숨기기 (visible 토글)
            ├── 채점 (status !== 'draft')
            ├── 통계 (마감 또는 진행 중)
            ├── 응시 모니터 (진행 중)
            └── 삭제

[빈 상태]
  ├── 시험 없음 → "아직 등록된 퀴즈가 없습니다" + 새 퀴즈 만들기 CTA
  └── 필터 결과 없음 → "해당 조건의 퀴즈가 없습니다" + 필터 초기화 링크
```
