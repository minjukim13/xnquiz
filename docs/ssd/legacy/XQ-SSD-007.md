## 3. 화면별 상세 설계

### SCR-I-RANDOM-COMPOSE. 랜덤 출제 조합 모달

**구현**: `src/components/RandomQuestionBankModal.jsx`

**레이아웃 (3 step)**

```
[Step 1: 그룹 선택]
  ├── 접근 가능 문제모음 후보 목록 (분류 정보 함께)
  └── 다중 선택 (체크박스) → 다음 단계 활성

[Step 2: 그룹별 조건 일괄 구성]
  └── 선택된 그룹별로 행 표시
       ├── 그룹명 + 실제 문항 수
       ├── 추출 문항 수 입력 (1 이상 정수)
       └── 배점 입력
       └── 합계 정보 표시 (총 출제 문항 수 / 총 배점)

[Step 3: 출제 전 통합 검토]
  ├── 포함 그룹 요약
  ├── 그룹별 상태 (완료 / 일부 설정 / 미설정, 충분 / 부족)
  ├── 4종 안내 (문항 수 부족 / 배점 미입력 / 조건 불완전 / 그룹 점검)
  ├── 출제 전 문항 미리보기 (확인 보조, 편집 불가)
  └── 확정 / 이전 단계
```

**사용 컴포넌트**: `Dialog`, `Button`, 그룹 카드 (체크박스), Input (수치 입력), 단계 인디케이터, 안내 박스.

**상태**

| 상태 | 표현 |
|---|---|
| Step 1 그룹 0개 | "다음" 비활성 |
| Step 2 조건 미설정 | 해당 그룹에 "미설정" 표시 + 검토 단계에서 안내 |
| Step 2 추출 문항 수 < 1 | 입력 거부 + 사유 안내 (UX-P07-013) |
| Step 2 추출 문항 수 > 실제 문항 수 | "부족" 경고 (UX-P07-022, FRD-007-04 PD 결정) |
| Step 3 4종 안내 | 어느 그룹에 어떤 문제 + 어떻게 수정 (UX-COM-005) |
| 발행 후 출제 조합 수정 | 영향 안내 (UX-P07-030) — 현재 미구현 → 간극 G-1 |
