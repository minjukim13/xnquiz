## 3. 화면별 상세 설계

### SCR-I-LX-ACTIVITY-LINK. 학습활동현황 진입점

**구현**: 채점 대시보드 학생 상세 패널 안에서 "학습활동현황으로 이동" 버튼 또는 링크.

**현재 프로토타입 동작**: 채점 대시보드의 학생 상세에 학습활동현황 직접 진입점 구현 여부 확인 필요. 메모리 `project_learningx_architecture.md` 기준 LearningX 통합은 Canvas 측에서 처리. xnquiz 측은 LX URL 구성 및 학생 ID 전달 만 필요 → 간극 G-1.

**레이아웃 (목표 동작)**

```
[채점 대시보드 - 학생 상세 헤더]
  ├── 학생 이름 / 학번 / 상태
  └── 우측: "학습활동현황으로 이동" 진입점 (외부 링크 아이콘)
       └── 클릭 시 LX 화면 새 탭/창 또는 동일 컨텍스트 (Canvas iframe 환경에서)
```

**상태**

| 상태 | 표현 |
|---|---|
| 채점 권한 보유 + 학생 선택됨 | 진입점 노출 |
| 채점 권한 없음 | 진입점 숨김 (UX-COM-010, UX-P05-003) |
| 학생 미선택 | 진입점 비활성 또는 숨김 |
| LX 측 접근 거부 | 진입 후 LX 화면이 권한 거부. 사용자는 Quiz 측이 아닌 LX 접근 권한 문제임을 인지 (UX-P07 보강) |

---

### SCR-I-QUIZ-MODULE-LINK. 시험 목록/상세 모듈 연결 안내

**구현**: SSD-018 의 QuizList / QuizDetail 의 주차/차시 배지 영역.

**현재 프로토타입 동작**

```
[QuizList 카드 메타]
  └── 주차/차시 배지 (`n주차 m차시`) 또는 "미지정"
```

**상태**

| 상태 | 표현 |
|---|---|
| 모듈 연결 + 주차 매핑됨 | 정상 배지 표시 |
| 모듈 연결 + 주차 미매핑 | "주차 미연결" 카피 (UX-P07-020) — 현재 카피 명시 부족 → 간극 G-2 |
| 모듈 미연결 | "미지정" 또는 "주차/차시 미연결" 표시 (WEEK_OPTIONS 의 "미지정" 옵션) |
| 모듈 삭제됨 | Quiz 자체는 정상 운영. 별도 안내 카피 (UX-P07-021) — 현재 미명시 → 간극 G-3 |
