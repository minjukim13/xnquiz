## 3. 화면별 상세 설계

### SCR-I-QUESTION-TITLE-INPUT. 문항 제목 입력

**구현**: AddQuestionModal 의 `form.title` 필드. 선택 입력, 빈 값 허용 (UX-P02-001).

| 상태 | 표현 |
|---|---|
| 미입력 | textarea 빈 값. 저장 가능 |
| 입력값 | 다음 편집 시점에 동일한 값 표시 (UX-P02-002) |
| 응시 후 변경 시 안내 | 저장 시 안내 문구 ("이미 응시 시작한 학생의 시험지는 변경되지 않음, 이후 응시자에게만 반영") — 현재 카피 미구현 → 간극 G-1 |
| 기본값 수준 판별 안내 | 시스템 기본 라벨 수준이면 학생 비노출 사전 안내 (UX-P02-011) — 현재 판별 로직 미구현 → 간극 G-2 |

---

### SCR-L-QUESTION-TITLE-DISPLAY. 학생 응시 / 결과 화면 문항 헤더

**구현**: QuizAttempt 의 문항 헤더 영역.

**현재 프로토타입 동작 확인 필요**: 문항 헤더에 제목 결합 표시 (`번호 + 제목`) 또는 번호만 표시인지 코드 확인 → 메모리 `project_xnquiz_sprint_implementation.md` 의 Sprint 4 후보 "학생 응시 화면 헤더에 문항 제목 결합 표시" 가 미구현 가능성 → 간극 G-3.

| 상태 | 표현 |
|---|---|
| 제목 미입력 (또는 기본값 수준) | 번호만 표시 (예: "문제 1") |
| 제목 입력값 있음 | 번호 + 제목 표시 (예: "문제 1. 데이터베이스 정규화") |
| 응시 중 제목 유지 | 응시 시작 시점의 제목이 응시 종료 시까지 유지 (UX-P08-005) |
| 미리보기 / 결과 / 리뷰 | 응시 화면과 동일 표시 기준 (UX-COM-004) |
