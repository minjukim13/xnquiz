## 1. 역할별 네비게이션 구조

```
교수자 (instructor):
홈 → 사이드바 "문제은행" → /question-banks
                            ├── 현재 과목 문제모음 목록
                            ├── "새 문제모음" 진입 (이름 + 난이도 입력)
                            └── 카드 클릭 → 문제모음 상세 (/question-banks/:bankId)
                                ├── 문제모음명 편집 (인라인)
                                ├── 문항 리스트 (탐색/필터/검색)
                                ├── 문항 작성/편집 (AddQuestionModal, SSD-024)
                                ├── 일괄 업로드 (BankUploadModal, SSD-029)
                                ├── 자산 흐름 (복사 / 가져오기 / 내보내기 / 삭제)
                                └── "문항 수정은 기존 퀴즈에 자동 반영되지 않음" 안내

학생 (student):
사이드바에 "문제은행" 진입점 미노출. 직접 URL 접근 시 홈 리다이렉트

핵심 태스크 클릭 뎁스:
- 문제모음 진입: 사이드바 → 문제은행 (1단계)
- 문항 추가: 문제은행 → 모음 클릭 → 새 문항 (3단계)
- 다른 과목에서 가져오기: 문제은행 → 모음 클릭 → 가져오기 → 출처 선택 → 문항 선택 → 추가 (6단계)
```

**도달 원칙**

- 교수자 전용. 학생은 진입점 미노출 + 직접 URL 시 리다이렉트.
- 현재 과목 자산을 단일 목록으로 표시 (UX-P02-002). 다른 과목 자산은 가져오기 출처 / 내보내기 대상으로만 인지.
- 빈 상태 3종 분기 (문제모음 없음 / 조건 결과 없음 / 문항 없음).

## 3. 화면별 상세 설계

### SCR-I-BANK-LIST. 문제모음 목록

**구현 파일**: `src/pages/QuestionBankList.jsx`

**레이아웃**

```
[헤더] "문제은행" + "새 문제모음" 버튼
[필터/검색 행] 검색 입력 + 정렬 드롭다운
[목록]
  └── 문제모음 카드 × N
       ├── 문제모음명 (인라인 편집 가능)
       ├── 난이도 배지 (미지정 / 상 / 중 / 하)
       ├── 문항 수 (active 기준)
       └── 카드 메뉴 (복사 / 삭제 / 가져오기 진입 / 상세 진입)
[빈 상태] 문제모음 0개 → "첫 문제모음을 만들어보세요" CTA
[빈 상태] 조건 결과 0개 → "조건에 맞는 결과 없음" + 조건 해제 링크
```

**사용 컴포넌트**: `Button`, `Card`, `Badge`, `Input` (검색), `DropdownSelect` (정렬), `DropdownMenu` (카드 메뉴), `ConfirmDialog` (삭제), `Skeleton` (로딩), `Toast`

**인터랙션 / 상태**

| **#** | **트리거** | **동작** |
|---|---|---|
| I-1 | "새 문제모음" 클릭 | 신규 생성 모달 (이름 + 난이도 입력). 이름 빈 값 시 생성 불가 |
| I-2 | 카드 클릭 | `/question-banks/:bankId` 이동 |
| I-3 | 문제모음명 인라인 편집 | 편집 중 / 저장 완료 / 저장 실패 / 취소 상태 구분 (UX-P02-021) |
| I-4 | 카드 메뉴 - 복사 | 복사 실행 → 결과 안내 (복사된 모음 위치 + 후속 이동 링크) |
| I-5 | 카드 메뉴 - 삭제 | ConfirmDialog → 삭제 |

### SCR-I-BANK-DETAIL. 문제모음 상세

**구현 파일**: `src/pages/QuestionBank.jsx`

**레이아웃**

```
[헤더]
  ├── 좌측: 목록 복귀
  ├── 문제모음명 (인라인 편집)
  ├── 난이도 배지
  └── 우측: 액션 (가져오기 / 내보내기 / 복사 / 삭제 / 일괄 업로드)

[안내 박스] "문항 수정은 기존 퀴즈에 자동 반영되지 않음" (UX-P02-052)

[필터/검색 행]
  ├── 검색 (본문/제목)
  ├── 유형 필터 (12 유형)
  └── 난이도 필터

[문항 리스트]
  ├── 문항 카드 × N
  │    ├── 번호 + 유형 배지 + 난이도 배지 + 배점
  │    ├── 본문 요약 (HTML → 텍스트, line-clamp)
  │    └── 액션 (편집 Pencil / 삭제 Trash2)
  ├── 드래그 정렬 (필터/검색 미적용 시만)
  └── "새 문항 추가" 진입점 (AddQuestionModal, SSD-024)

[빈 상태] 문항 0개 → "첫 문항을 추가하세요" CTA
[빈 상태] 검색/필터 결과 0개 → 조건 해제 안내
```

**사용 컴포넌트**: `Button`, `Card`, `Badge`, `Input`, `DropdownSelect`, `AddQuestionModal` (SSD-024), `BankUploadModal` (SSD-029), `ConfirmDialog`, `Toast`

**인터랙션 / 상태**

| **#** | **트리거** | **동작** |
|---|---|---|
| I-1 | 인라인 편집 | 문제모음명 갱신, 상태 표시 |
| I-2 | "가져오기" / "내보내기" 클릭 | 각 모달 진입 (SCR-I-BANK-IMPORT / EXPORT) |
| I-3 | "일괄 업로드" 클릭 | BankUploadModal 진입 (SSD-029) |
| I-4 | 필터/검색 적용 | 결과 갱신. 드래그 정렬 비활성 (UX-P02-070) |
| I-5 | 문항 편집 (Pencil) | AddQuestionModal 수정 모드 (SSD-024). 유형 변경 차단 (UX-P02-081) |
| I-6 | 문항 키보드 진입 (Tab + Enter) | 편집 진입 가능 (UX-P02-082) |
| I-7 | "새 문항 추가" 클릭 | AddQuestionModal 신규 모드. 유형 선택 단계 → 작성 단계 (UX-P02-080) |
| I-8 | 난이도 지정 모음에 난이도 다른 문항 추가 시도 | 충돌 안내 (UX-COM-006). 일괄 업로드 흐름은 경고 후 진행 허용 (BankUploadModal, SSD-029) |

### SCR-I-BANK-IMPORT. 가져오기 모달

**구현 파일**: `src/components/QuestionBankModal.jsx` (QuizCreate/Edit 의 "문제모음에서 추가" 와 동일 컴포넌트, 진입 맥락이 다름) — 또는 QuestionBank 화면의 자체 가져오기 분기


## 5. 비정상 상태 UX

| **상태** | **현재 프로토타입 표현** |
|---|---|
| 로딩 | Skeleton (목록/상세 각각) |
| 문제모음 없음 (빈 상태) | "첫 문제모음을 만들어보세요" + CTA |
| 조건 결과 없음 | "조건에 맞는 결과 없음" + 조건 해제 링크 |
| 문항 없음 (상세) | "첫 문항을 추가하세요" + CTA |
| 자산 흐름 실패 | Toast 또는 결과 모달에 "실패" 안내 (UX-COM-004) |
| 자산 흐름 부분 실패 | "성공 N건 / 실패 M건" 분리 표시 + 실패 사유 |
| 권한 불가 출처 | 가져오기 모달에서 해당 출처 비노출 또는 사유 안내 (UX-P03-001) |
| 난이도 충돌 | 모음 난이도 ≠ 문항 난이도 시 경고 (UX-COM-006) |
| 학생 직접 진입 | Navigate to "/" |
| 존재하지 않는 모음 진입 | 회복 경로 (목록 페이지로 이동) (UX-P05-004) |
