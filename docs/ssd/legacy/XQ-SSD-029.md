## 1. 역할별 네비게이션 구조

```
교수자 (instructor):
홈 → 문제은행 (/question-banks) → 문제모음 카드 → 문제모음 상세 (/question-banks/:bankId)
                                                   └── "일괄 업로드" 버튼 → BankUploadModal
                                                          ├── Step 1: 양식 안내 / 다운로드
                                                          ├── Step 2: 파일 업로드 + 사전 검증
                                                          └── Step 3: 검증 결과 + 등록 / 보류

핵심 태스크 클릭 뎁스:
- 일괄 업로드: 문제은행 → 모음 → 일괄 업로드 (3단계)
```

**도달 원칙**

- 권한: 교수자 + 해당 모음에 일괄 업로드 가능 권한 보유. 학생 진입 차단.
- 차단 오류 1건 이상 → 전체 보류 (UX-P02-050). 어떤 문항도 등록되지 않음.
- 경고만 있는 경우 (난이도 불일치) → 사용자가 인지 후 등록 진행 (메모리 `project_xnquiz_oq_decisions_round2.md` BankUploadModal 난이도 경고 후 허용 구현).

## 3. 화면별 상세 설계

### SCR-I-BANK-UPLOAD. 일괄 업로드 모달 (3단계)

**구현 파일**: `src/components/BankUploadModal.jsx` (3단계 모달)

**목적**

대상 문제모음에 다량 문항을 양식 기반으로 등록. 사전 검증 결과를 행 단위로 안내하고, 차단 오류 시 전체 보류, 경고만 있는 경우 사용자 진행 허용.

**레이아웃 (3단계 분리)**

```
[Dialog]
[Header] 
  └── DialogTitle "{모음명} 에 일괄 업로드"

[Step 1: 양식 안내]
  ├── 표준 양식 다운로드 버튼 (.xlsx)
  ├── 필수 정보 안내 (질문 / 정답 / 유형 / 난이도(선택) / 배점(선택))
  ├── 1차 지원 유형 안내 (객관식 / OX / 단답형)
  ├── 고급 요소 1차 미포함 안내 (이미지, 수식, 복합 피드백)
  ├── 지원 파일 형식 (.xlsx / .xls / .csv, 동일 검증 기준)
  └── "다음 단계: 파일 선택" 버튼

[Step 2: 파일 업로드 + 사전 검증]
  ├── 대상 모음명 확인 ("{모음명} 에 등록됩니다")
  ├── 파일 선택 영역 (drag&drop 또는 파일 선택 버튼)
  ├── 선택된 파일명 + 파일 크기 표시
  ├── 검증 중 상태 (로딩 표시)
  └── "검증 시작" / "이전 단계" 버튼

[Step 3: 검증 결과]
  ├── 요약 카드
  │    ├── 등록 가능 후보 N문항 (success)
  │    └── 수정 필요 M문항 (warning)
  ├── 차단 오류 안내 (있는 경우)
  │    ├── 전체 보류 안내 (warning-bg + AlertTriangle)
  │    └── "어떤 문항도 모음에 등록되지 않음" 카피
  ├── 경고 안내 (있는 경우, 난이도 불일치 등)
  │    └── "확인 후 등록 진행 가능" 카피
  ├── 행 단위 오류 리스트
  │    ├── 행 번호 (양식 기준)
  │    ├── 열/정보 (필수 정보 명칭)
  │    ├── 오류 종류 (차단 / 경고 배지)
  │    └── 수정 안내 카피 (4 요소: 행 + 정보 + 오류 + 수정 방법)
  └── 액션
       ├── 차단 오류 있음: "이전 단계 (수정 후 재업로드)" 만 활성
       └── 경고만 있음 또는 오류 없음: "이전" + "등록 진행" 활성

[등록 완료]
  ├── AlertDialog "등록 완료" + N문항 등록됨
  └── 모달 닫힘 → 문제모음 상세에서 등록된 문항 확인 가능
```

**사용 컴포넌트**

| **컴포넌트** | **용도** |
|---|---|
| `Dialog` / `DialogContent` / `DialogHeader` / `DialogTitle` / `DialogDescription` | 모달 컨테이너 |
| `Button` | 단계별 이동 / 양식 다운로드 / 등록 진행 |
| native `<input type="file">` 또는 drag&drop 영역 | 파일 선택 |
| `parseExcelOrCsv` | `src/utils/excelUtils.js` 의 검증 함수 (행 단위 분리, 메모리 `project_xnquiz_sprint_implementation.md` 의 Sprint 1-1 구현) |
| `AlertDialog` | 등록 완료 안내 |
| `Badge` | 차단 / 경고 구분 |
| Lucide icons | `Upload` `FileText` `AlertTriangle` `CheckCircle2` 등 |

**인터랙션**

| **#** | **트리거** | **동작** |
|---|---|---|
| I-1 | Step 1 "다음 단계" 클릭 | Step 2 로 전환 |
| I-2 | 표준 양식 다운로드 | `.xlsx` 양식 파일 다운로드 (정해진 컬럼 구조) |
| I-3 | 파일 선택 (Step 2) | 파일명/크기 표시. 지원 형식 외 파일 거부 + "사용 가능한 형식" 안내 |
| I-4 | "검증 시작" 클릭 | `parseExcelOrCsv` 실행 → 행 단위로 차단 / 경고 분류 |
| I-5 | Step 3 진입 | 요약 + 행 단위 오류 리스트 표시 |
| I-6 | "이전" 클릭 | 이전 단계로 복귀 (파일 / 결과 보존) |
| I-7 | "등록 진행" (차단 0건) | 등록 실행 → 등록 완료 AlertDialog → 모달 닫힘 |
| I-8 | 차단 오류 있음 + "등록 진행" 시도 | 버튼 disabled, 클릭 불가 |

**상태**

| **상태** | **표현** |
|---|---|
| Step 1 양식 안내 | 양식 다운로드 + 1차 지원 유형 안내 |
| Step 2 파일 미선택 | "검증 시작" 버튼 disabled |
| Step 2 파일 선택됨 | 파일명/크기 표시 + 검증 시작 활성 |
| Step 2 검증 중 | 로딩 표시 ("검증 중") |
| Step 3 결과 - 오류 0건 | success 요약 + "등록 진행" 활성 |
| Step 3 결과 - 경고만 | warning 요약 + "확인 후 등록 진행 가능" 카피 + "등록 진행" 활성 (UX-P02-052) |
| Step 3 결과 - 차단 오류 1건 이상 | warning-bg 전체 보류 안내 + "등록 진행" disabled |
| 등록 완료 | AlertDialog "등록 완료" + 등록 수 + 모음 이동 안내 |
| 파일 형식 미지원 | "사용 가능한 형식: .xlsx / .xls / .csv" 안내 (UX-P02-010) |
| 모음 난이도 미설정 | 난이도 일치 검증 미수행 안내 (UX-COM-006) |
| 권한 없음 또는 정책 제한 | 일괄 업로드 진입 자체 차단. 사유 안내 (UX-P05-005) |

## 5. 비정상 상태 UX

| **상태** | **현재 프로토타입 표현** |
|---|---|
| 양식 다운로드 실패 | Toast 또는 에러 안내 |
| 파일 형식 미지원 | "사용 가능한 형식: .xlsx / .xls / .csv" 안내 |
| 파일 크기 / 문항 수 상한 초과 | 사전 안내 (UX-P02-053) - 현재 프로토타입 상한 정책 미확인 → 간극 G-2 |
| 검증 실패 (구조적 오류) | "양식이 일치하지 않습니다" 안내 + 양식 재다운로드 권장 |

| 난이도 불일치 (경고) | "확인 후 등록 진행 가능" 안내 + 사용자 판단 허용 |
| 재업로드 | 이전 검증 결과 폐기 + 새 파일 기준 재검증 (UX-P02-051) |
| 권한 없음 | 일괄 업로드 진입점 자체 비노출 또는 사유 안내 (UX-P05-005) |
| 학생 직접 URL | Navigate to "/" (문제은행 진입 차단) |
