## 4. 기능 분류

### 4-1. [A] Canvas 기존 기능 - 기존 Canvas 제공 기능 재구현

| **ID** | **기능** | **동작** | **구현 위치** |
|---|---|---|---|
| A-05-1 | 자동 채점 결과 조회 | 객관식/단답형 등 autoGrade=true 문항의 자동 채점 결과 표시 | AnswerCard |
| A-05-2 | 수동 채점 (서술형/파일) | 점수 직접 입력 + 정답/오답 마크 | StudentDetailPanel |
| A-05-3 | 학생 목록 + 응시 상태 분류 | 미채점/채점완료/미제출 섹션으로 자동 분류 | StudentListPanel |
| A-05-4 | 학생별 채점 진입 | 학생 클릭 → 학생 모드로 전환 + 답안 탭 활성 | 좌측 패널 클릭 |
| A-05-5 | 문항별 채점 진입 | 문항 클릭 → 문항 모드로 전환 + 응답 탭 활성 | 좌측 패널 클릭 |
| A-05-6 | 점수 수정 + 저장 | 직접 입력 후 저장 → updateAttempt | 점수 입력 필드 |

### 4-2. [B] 학교 요구사항 반영 - Canvas 에 없거나 부족하여 학교가 요구한 신규 기능

| **ID** | **라벨** | **기능** | **동작** | **구현 위치** |
|---|---|---|---|---|
| F-05-1 | [B-#13] | 성적 엑셀 업로드/다운로드 | 양식 다운로드 → 학생명/학번/점수 입력 → 검증 후 일괄 적용 | ExcelModal |
| F-05-2 | [B-#10] | 부분 점수 정책 적용 | 정답 변경 시 재채점 옵션 (이전 정답 유지/새 정답만/전원 만점) | QuestionDetailPanel 재채점 옵션 |
| F-05-3 | [B-#09] | 답안지 출력/다운로드 진입점 | 학생/문항 단위로 답안지 다운로드 (백엔드 Creator 영역 일부) | 다운로드 버튼 |

### 4-3. [C] 자체 도출 개선 - Canvas 기존 기능도 아니고 학교 요구도 아닌, 프로젝트 자체 발굴 개선

| **ID** | **라벨** | **기능** | **동작** | **구현 위치** |
|---|---|---|---|---|
| F-05-4 | [C] | 단일 화면 채점 대시보드 | Canvas SpeedGrader 처럼 별도 화면 분리하지 않고 목록+상세+통계 통합 | GradingDashboard 단일 라우트 |
| F-05-5 | [C] | 문항 ↔ 학생 이중 중심 모드 | 같은 데이터를 두 관점(문항 단위/학생 단위)로 토글하여 조회 | 모드 전환 토글 |
| F-05-6 | [C] | 미시작자 분모 포함 채점 | 마감 후 미시작자도 자동 0점 처리 + 채점 완료에 포함 (분모 일치 정책) | 채점 집계 로직 |
| F-05-7 | [C] | 응답 탭 - 다중 필터 (제출 상태 + 채점 상태) | 정상/지각/미제출 + 미채점/완료 등 다축 필터 + 페이지네이션 | ResponsesTab |
| F-05-8 | [C] | 통계 탭 - 문항별 점수 분포 | recharts 막대그래프로 문항별 점수 분포 + 평균/최고/최저 | StatsTab |
| F-05-9 | [C] | 학생 단위 일괄 채점 (모든 문항 정답/오답 + 가산점) | 한 학생의 모든 문항을 한 번에 정답 처리, 가산점 팝오버 별도 적용 | StudentDetailPanel |
| F-05-10 | [C] | 문항 단위 일괄 채점 (전체 정답/오답/미제출자 0점) | 한 문항의 모든 응답을 일괄 처리 | QuestionDetailPanel |
| F-05-11 | [C] | 활동 로그 타임라인 | 시작/이동/답변변경/포커스이탈/포커스복귀/자동저장/제출 시각화 (포커스 이탈은 amber 강조) | ActivityLogPanel |
| F-05-12 | [C] | 자동 제출 배지 | 시간 초과 자동 제출 시 활동 로그에 "자동" 배지 | ActivityLogPanel |
| F-05-13 | [C] | 코멘트 스레드 (정답/오답/중립 분류) | 교수자/학생 메시지 양방향 + 읽음 추적 + Shift+Enter 줄바꿈 | CommentThread |
| F-05-14 | [C] | 가산점 팝오버 | 학생별 가산점을 별도 저장 + 점수에 합산 표시 | xnq_fudge_points |
| F-05-15 | [C] | 재채점 로그 + 적용 안내 | 재채점 트리거 후 영향 범위 (n명) 토스트 표시 | xnq_regrade_log |
| F-05-16 | [C] | 모바일 - 목록/상세 탭 전환 | 데스크톱은 split-pane, 모바일은 탭으로 전환 | 반응형 레이아웃 |

## 5. 화면 상태 (State)

| **상태** | **트리거** | **표현** |
|---|---|---|
| 로딩 | 초기 fetch 중 | 스켈레톤 |
| 빈 상태 | 응시자 0명 | EmptyState 안내 ("아직 제출한 학생이 없습니다") |
| 문항 모드 | 좌측 = 문항 리스트 | 우측 응답 탭/통계 탭 |
| 학생 모드 | 좌측 = 학생 리스트 | 우측 답안 탭/활동 로그 탭/코멘트 탭 |
| 미채점 강조 | 학생 score=null | 좌측 리스트에 빨강 점/배지 |
| 채점 완료 | 학생 score≠null | 좌측 리스트에 회색 처리 |
| 일괄 처리 진행 | 일괄 정답/오답 트리거 | 토스트 + 영향 학생 수 표시 |
| 엑셀 업로드 검증 실패 | 배점 초과/학번 미존재 | ExcelModal 안에 오류 행 표시 + 템플릿 다운로드 링크 |

## 7. 예외/엣지 케이스

- 미시작자: startTime null → 자동 0점 처리되어도 학생 리스트에서 "미제출" 라벨 유지 (분모에는 포함)
- 마감 후 자동 제출: activityLog 에 "자동" 배지로 구분
- 포커스 이탈 다회 발생: ActivityLogPanel 에서 amber 강조 + 누적 카운트 표시
- 엑셀 업로드 시 배점 초과 점수: 검증 단계에서 거부, 오류 행만 별도 표시
- 학번 미존재 행: 거부 + "학번을 확인해 주세요" 안내
- 가산점 음수: 허용 (감점 용도)
- 재채점 정책이 mock 전용: api 모드에선 재채점 옵션 비활성 (서버 재채점 엔드포인트 추후 구현)
- 코멘트 빈 메시지: 전송 차단
