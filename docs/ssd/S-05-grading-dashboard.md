# S-05. 채점 대시보드

> **라벨 규칙**: `[A]` Canvas 기존 기능 / `[B-#NN]` 학교 요구사항 신규 / `[C]` 자체 도출 개선

## 1. 화면 정보

| **항목** | **내용** |
|---|---|
| 화면 ID | S-05 |
| 라우트 | `/quiz/:id/grade` |
| 진입 권한 | 교수자 (학생 접근 시 자동 리다이렉트) |
| 진입 경로 | 퀴즈 목록(S-01) 카드 메뉴 / 퀴즈 상세(S-03) 채점 버튼 / 통계(S-06) 답안 확인 |

## 2. 화면 목적

- 단일 화면에서 수강생 응답 + 문항별 분포 + 코멘트 + 활동 로그 + 엑셀 출력까지 모두 처리
- 자동 채점 결과 검토 + 수동 채점 (서술형/파일) + 일괄 채점 + 재채점 트리거

## 3. 영역 구성 (Region Map)

> **채점 진행 통계 분모 정책**: 헤더 정보 카드의 응시율 / 응시 인원 / 채점 완료 세 지표는 모두 **전체 수강생 수**를 분모로 계산함. 마감 후 미시작자는 자동 0점 처리되어 채점 완료 수치에 포함됨 (응시 학생만이 아닌 수강생 전체 대비 진행 현황을 보여주기 위함).

| **영역** | **설명** | **주요 컴포넌트** |
|---|---|---|
| 헤더 | 뒤로가기 + 퀴즈 정보 카드 (응시율 / 응시 인원 / 채점 완료) — 전체 수강생 기준 | QuizInfoCard |
| 모드 전환 | 문항 중심 ↔ 학생 중심 토글 | 모드 토글 |
| 좌측 패널 | (문항 모드) 문항 리스트 / (학생 모드) 학생 리스트 (검색 + 미채점/완료/미제출 분류) | StudentListPanel, StudentListItem |
| 우측 패널 - 문항 모드 | 응답 탭 (ResponsesTab) / 통계 탭 (StatsTab) | ResponsesTab, StatsTab |
| 우측 패널 - 학생 모드 | 답안 탭 / 활동 로그 탭 / 코멘트 탭 | StudentDetailPanel, AnswerCard, ActivityLogPanel, CommentThread |
| 모달 | 엑셀 업로드/다운로드 / 빈 상태 안내 | ExcelModal, EmptyState |

## 4. 기능 분류

### 4-1. [A] Canvas 기존 기능 - 기존 Canvas 제공 기능 재구현

| **ID** | **기능** | **동작** |
|---|---|---|
| A-05-1 | 자동 채점 결과 조회 | 객관식/단답형 등 autoGrade=true 문항의 자동 채점 결과 표시 |
| A-05-2 | 수동 채점 (서술형/파일) | 점수 직접 입력 + 정답/오답 마크 |
| A-05-3 | 학생 목록 + 응시 상태 분류 | 미채점/채점완료/미제출 섹션으로 자동 분류 |
| A-05-4 | 학생별 채점 진입 | 학생 클릭 → 학생 모드로 전환 + 답안 탭 활성 |
| A-05-5 | 문항별 채점 진입 | 문항 클릭 → 문항 모드로 전환 + 응답 탭 활성 |
| A-05-6 | 점수 수정 + 저장 | 직접 입력 후 저장 → updateAttempt |
| A-05-7 | 가산점 (Fudge Points) | 학생별 가산점을 별도 저장 + 점수에 합산 표시 (Canvas Fudge Points 와 동일 개념) |
| A-05-8 | 자동 제출 배지 | 시간 초과 자동 제출 시 활동 로그에 "자동" 배지 (Canvas 의 시간 만료 자동 제출 상태 표시) |

### 4-2. [B] 학교 요구사항 반영 - Canvas 에 없거나 부족하여 학교가 요구한 신규 기능

| **ID** | **라벨** | **기능** | **동작** |
|---|---|---|---|
| F-05-1 | [B-#13] | 성적 엑셀 업로드/다운로드 | 양식 다운로드 → 학생명/학번/점수 입력 → 검증 후 일괄 적용 |
| F-05-2 | [B-#10] | 부분 점수 정책 적용 | 정답 변경 시 재채점 옵션 (이전 정답 유지/새 정답만/전원 만점) |
| F-05-3 | [B-#09] | 답안지 출력/다운로드 진입점 | 학생/문항 단위로 답안지 다운로드 (백엔드 Creator 영역 일부) |

### 4-3. [C] 자체 도출 개선 - Canvas 기존 기능도 아니고 학교 요구도 아닌, 프로젝트 자체 발굴 개선

| **ID** | **라벨** | **기능** | **동작** |
|---|---|---|---|
| F-05-4 | [C] | 단일 화면 채점 대시보드 | Canvas SpeedGrader 처럼 별도 화면 분리하지 않고 목록+상세+통계 통합 |
| F-05-5 | [C] | 문항 ↔ 학생 이중 중심 모드 | 같은 데이터를 두 관점(문항 단위/학생 단위)로 토글하여 조회 |
| F-05-6 | [C] | 미시작자 자동 0점 + 전체 수강생 기준 진행률 | 마감 후 미시작자를 자동 0점으로 채점 완료 처리하여, 응시율/채점 완료 수치를 전체 수강생 대비로 일치시킴 |
| F-05-7 | [C] | 응답 탭 - 다중 필터 (제출 상태 + 채점 상태) | 정상/지각/미제출 + 미채점/완료 등 다축 필터 + 페이지네이션 |
| F-05-8 | [C] | 통계 탭 - 문항별 점수 분포 | 막대그래프로 문항별 점수 분포 + 평균/최고/최저 |
| F-05-9 | [C] | 학생 단위 일괄 채점 (모든 문항 정답/오답 + 가산점) | 한 학생의 모든 문항을 한 번에 정답 처리, 가산점 팝오버 별도 적용 |
| F-05-10 | [C] | 문항 단위 일괄 채점 (전체 정답/오답/미제출자 0점) | 한 문항의 모든 응답을 일괄 처리 |
| F-05-11 | [C] | 활동 로그 타임라인 | 시작 / 문항 이동 / 답변 변경 / 포커스 이탈 / 포커스 복귀 / 자동 저장 / 제출 시각화 (포커스 이탈은 주의 색상 강조) |
| F-05-12 | [C] | 코멘트 스레드 (정답/오답/중립 분류) | 교수자/학생 메시지 양방향 + 읽음 추적 + Shift+Enter 줄바꿈 |
| F-05-13 | [C] | 재채점 로그 + 적용 안내 | 재채점 트리거 후 영향 범위 (n명) 토스트 표시 |
| F-05-14 | [C] | 모바일 - 목록/상세 탭 전환 | 데스크톱은 좌우 분할 화면, 모바일은 상단 탭으로 전환 |

## 5. 화면 상태 (State)

| **상태** | **트리거** | **표현** |
|---|---|---|
| 로딩 | 초기 fetch 중 | 스켈레톤 |
| 빈 상태 | 응시자 0명 | EmptyState 안내 ("아직 제출한 학생이 없습니다") |
| 문항 모드 | 좌측 = 문항 리스트 | 우측 응답 탭/통계 탭 |
| 학생 모드 | 좌측 = 학생 리스트 | 우측 답안 탭/활동 로그 탭/코멘트 탭 |
| 미채점 강조 | 학생 score=null | 좌측 리스트에 빨강 점/배지 |
| 채점 완료 | 학생 score≠null | 좌측 리스트에 회색 처리 |
| 일괄 처리 진행 | 일괄 정답/오답 트리거 | 토스트 + 영향 학생 수 표시 |
| 엑셀 업로드 검증 실패 | 배점 초과/학번 미존재 | ExcelModal 안에 오류 행 표시 + 템플릿 다운로드 링크 |

## 7. 예외/엣지 케이스

- 미시작자(startTime null): 마감 후 자동 0점 처리되지만 학생 리스트에서는 "미제출" 라벨 유지. 응시율/채점 완료 수치 계산 시에는 전체 수강생 기준에 포함
- 마감 후 자동 제출: activityLog 에 "자동" 배지로 구분
- 포커스 이탈 다회 발생: ActivityLogPanel 에서 amber 강조 + 누적 카운트 표시
- 엑셀 업로드 시 배점 초과 점수: 검증 단계에서 거부, 오류 행만 별도 표시
- 학번 미존재 행: 거부 + "학번을 확인해 주세요" 안내
- 가산점 음수: 허용 (감점 용도)
- 재채점 정책이 mock 전용: api 모드에선 재채점 옵션 비활성 (서버 재채점 엔드포인트 추후 구현)
- 코멘트 빈 메시지: 전송 차단
