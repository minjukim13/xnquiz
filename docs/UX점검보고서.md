## 4. UX 측면 종합 평가

### 강점 (계속 유지할 것)
1. **응시 흐름의 견고함**: 자동저장, 세션 복원, lockDate 자동 제출, 미리보기 모드까지 Canvas 표준 모델 충실 구현
2. **점수 공개 정책의 정교함**: 6가지 차원(범위·시점·기간·1회조회 등) 모두 제어 가능
3. **활동 로그 인프라**: 분쟁 조사 시 강력한 자산
4. **다중 채점 모드**: 문항 중심/학생 중심 토글로 채점 패턴별 최적화
5. **상태 표시 일관성**: PM4 의 B-01~B-11 회귀 매트릭스 기반 11가지 케이스 정확 처리

### 약점 (개선 필요)
1. **재채점·재응시 영역의 정보 비대칭**: 학생/교수자 모두 "다음에 무슨 일이 일어날지" 예측 불가
2. **알림 시스템 전면 부재**: 점수 변동, 마감 임박, 채점 완료 모두 학생이 직접 들어와야 인지
3. **채점 효율성 도구 약함**: 단축키, 루브릭, 시뮬레이션 부재로 50명 단위 채점 시간 부담
4. **출제 화면 자동저장 부재**: 응시는 잘 되어 있으나 출제는 안 됨

### 즉시 효과 큰 작업 Top 3
1. **F-09 채점 단축키** — 1주 내 구현 가능, 채점 시간 30~50% 단축 가능
2. **F-18 점수 산정 정책 응시 전 안내** — 응시 카드에 한 줄 추가, 학생 불안 직접 해소
3. **F-02 출제 화면 자동저장** — 응시 화면 코드 재활용 가능, 데이터 손실 직접 방지
