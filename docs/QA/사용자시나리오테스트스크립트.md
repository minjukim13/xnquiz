# 사용자 시나리오 기반 테스트 스크립트

**작성일**: 2026-04-28
**대상 버전**: XN Quizzes Prototype (mock 모드 기준, api 모드 회귀 동일 적용)
**테스트 환경**: Chrome 최신, 데스크톱 1280px+ (반응형은 별도 시나리오)
**관련 문서**: `PM4_QA테스트보고서.md` (기능 단위 체크리스트), `PM2_기획검토보고서.md` (잔존 정책 이슈)

---

## 1. 개요

### 1.1 목적
기능 단위가 아닌 **실사용 흐름 단위**로 시나리오를 묶어, 역할(교수자/학생/운영자) 관점에서 끝-끝 동작과 데이터 일관성을 검증한다.

### 1.2 범위
- **포함**: 핵심 사용자 흐름 8종, 예외 흐름 6종, 역할 간 연계 시나리오 3종
- **제외**: 단일 컴포넌트 단위 테스트(PM4 영역), 실제 API 부하 테스트

### 1.3 사전 공통 조건
- 시드 데이터 적재 상태 (퀴즈 6개 이상, 학생 12명 이상, 문제은행 1개 이상)
- localStorage 초기화 후 시작 (mock 모드: `xnq_*` 키 일괄 삭제)
- `RoleToggle` 로 역할 전환 가능
- 테스트 시작 전 현재 시각, 데이터 모드(`VITE_DATA_SOURCE`) 기록

### 1.4 판정 기준 표기
- **PASS**: 기대 결과와 100% 일치
- **PARTIAL**: 동작하나 UX/표시 일부 불일치
- **FAIL**: 기대 결과와 다름 또는 에러 발생
- **BLOCKED**: 사전 조건 미충족 등으로 진행 불가

## 2. 교수자 시나리오

### S-T01. 신규 퀴즈를 처음부터 만들어 발행한다 [Critical]

**페르소나**: 김교수 (해당 학기 처음 퀴즈 출제)
**사전 조건**: 교수자 로그인, 임시저장/발행 이력 없음

| Step | 액션 | 입력값 | 기대 결과 | 판정 |
|---|---|---|---|:---:|
| 1 | 홈에서 "퀴즈 생성" 진입 | - | QuizCreate 페이지 진입, 1단계 표시 | |
| 2 | 기본정보 입력 | 제목, 설명, 분류 | 다음 버튼 활성화 | |
| 3 | 제목 미입력 상태로 다음 시도 | 제목 빈값 | 다음 버튼 비활성, 안내 메시지 | |
| 4 | 정책 설정 단계 진입 | 시간제한 30분, 재응시 1회, 지각허용 OFF | 다음 단계로 진행 | |
| 5 | 문항 추가 (4지선다 1문항) | 보기 4개, 정답 지정 | 문항 카드 1개 등록 | |
| 6 | 임시저장 클릭 | - | 토스트 노출, 목록에 "임시저장" 배지로 표시 | |
| 7 | 새로고침 후 목록 진입 | - | 임시저장 퀴즈 유지됨 (소실 X) | |
| 8 | 다시 편집 진입 → 발행 | - | 상태 "진행중" 으로 전환, 목록 노출 | |

**통과 기준**: 임시저장 → 새로고침 → 재편집 → 발행 흐름에서 데이터 손실/상태 오류 없음

### S-T02. 기존 퀴즈를 복사해 신규 회차로 만든다 [High]

**사전 조건**: 발행된 퀴즈 1개 이상 존재

| Step | 액션 | 기대 결과 | 판정 |
|---|---|---|:---:|
| 1 | 퀴즈 목록에서 복사 메뉴 클릭 | 복사 모달 열림 | |
| 2 | 새 제목 입력 후 확인 | 새 퀴즈가 "임시저장" 상태로 생성, 문항/정책 동일하게 복제 | |
| 3 | 복제본 편집 진입 | 원본 문항 그대로 표시, 정답/배점 일치 | |
| 4 | 복제본의 정답 1개 변경 | 원본 퀴즈 정답은 영향 없음 (독립성 검증) | |
| 5 | 복제본 발행 | 목록에 별개 퀴즈로 표시, 원본 응시 기록과 분리 | |

**통과 기준**: 복사본은 원본과 완전 분리된 ID/응시기록을 가짐

### S-T03. 발행 후 정답을 변경하고 재채점한다 [Critical]

**사전 조건**: 학생 응시 완료된 퀴즈 1개 (자동채점 포함)

| Step | 액션 | 기대 결과 | 판정 |
|---|---|---|:---:|
| 1 | 응시 기록이 있는 퀴즈 편집 진입 | 경고 배너 또는 안내 노출 | |
| 2 | 4지선다 1문항의 정답 변경 후 저장 | RegradeOptionsModal 표시 | |
| 3 | "이전+신규 인정" 옵션 선택 | 기존 정답자 점수 유지 + 신규 정답자도 가산 | |
| 4 | 채점 대시보드에서 재채점 로그 확인 | 재채점 이력(시각/옵션/대상 학생수) 표시 | |
| 5 | 학생 점수 합계 재계산 결과 확인 | 변경된 문항 외 점수는 동일 | |
| 6 | "신규 기준 / 만점 / 변경만" 각각도 동일 흐름 검증 | 각 옵션별 정책에 맞게 점수 변동 | |

**통과 기준**: 4가지 재채점 옵션 모두 점수 산출 정책과 일치, 로그 누락 없음

### S-T04. 채점 대시보드에서 수동채점을 완료한다 [High]

**사전 조건**: 서술형 포함 퀴즈, 학생 5명 이상 제출 완료

| Step | 액션 | 기대 결과 | 판정 |
|---|---|---|:---:|
| 1 | 채점 대시보드 진입 | 학생 목록, 통계 카드 표시 | |
| 2 | 학생 A 선택 | 답안 패널 표시, 자동채점 점수 인라인 노출 | |
| 3 | 서술형 답안에 점수/피드백 입력 | 저장 즉시 학생 합계 갱신 | |
| 4 | 학생 B 선택 후 동일 작업 | 학생 A 입력 점수 유지 (덮어쓰기 X) | |
| 5 | 미채점 필터 적용 | 서술형 채점 미완료 학생만 표시 | |
| 6 | "채점 완료" 토글 (정책 미정 시 SKIP) | E-08 미구현 — **현재 FAIL 예상** | |

**통과 기준**: 개별 문항 채점은 즉시 반영, 학생 간 데이터 충돌 없음
**알려진 실패**: 최종 점수 확정(commit) 흐름 부재 (PM2 E-08)

### S-T05. 마감 시각이 지난 퀴즈의 상태/표시를 확인한다 [Critical]

**사전 조건**: PM4 회귀 시나리오 B-01~B-11 매트릭스 전부

| Step | 액션 | 기대 결과 | 판정 |
|---|---|---|:---:|
| 1 | dueDate=과거, allowLateSubmit=false 퀴즈 목록 확인 | "마감" 배지, 응시 버튼 비활성 | |
| 2 | 동일 퀴즈 상세 진입 | 상세 헤더에도 "마감" 일관 표시 | |
| 3 | 채점 대시보드 진입 | 채점 분모 = 전체 수강생, 미시작자 자동 0점 처리 (2026-04-22 정책) | |
| 4 | dueDate 를 미래로 수정 (재오픈) | 배지 즉시 "진행중" 으로 전환 | |
| 5 | 지각 허용+lateSubmitDeadline 무제한 | dueDate 경과 후에도 영원히 "진행중" | |

**통과 기준**: 11개 케이스(B-01~B-11) 모두 표시 일치
**참조**: `PM4_QA테스트보고서.md` 회귀 시나리오 매트릭스

## 3. 학생 시나리오

### S-S01. 퀴즈를 처음부터 끝까지 응시·제출한다 [Critical]

**사전 조건**: 진행중 상태 퀴즈, 미응시 학생 계정

| Step | 액션 | 기대 결과 | 판정 |
|---|---|---|:---:|
| 1 | 학생 홈 → 진행중 퀴즈 진입 | "응시 시작" 버튼 노출 | |
| 2 | 시간제한 30분 안내 확인 후 시작 | 카운트다운 시작, 1번 문항 표시 | |
| 3 | 4지선다 1번 답안 선택 | 자동저장(localStorage) 즉시 반영 | |
| 4 | 새로고침 (의도된 이탈 시뮬레이션) | 카운트다운/답안 복원, 진행 가능 | |
| 5 | 마지막 문항까지 답안 입력 | 제출 버튼 활성화 | |
| 6 | 제출 클릭 | 결과 화면 노출, 자동채점 점수 즉시 표시 | |
| 7 | 결과 화면 새로고침 | 결과 유지, 재제출 불가 | |

**통과 기준**: 자동저장-복원-제출 흐름에서 답안 손실 0건

---

### S-S02. 시간 만료 시 자동 제출된다 [High]

**사전 조건**: 시간제한 1분 퀴즈

| Step | 액션 | 기대 결과 | 판정 |
|---|---|---|:---:|
| 1 | 응시 시작 후 1번 문항만 답안 입력 | 카운트다운 정상 감소 | |
| 2 | 1분 경과 대기 | 자동 제출, 결과 화면 진입 | |
| 3 | 미응답 문항 채점 결과 확인 | 0점 처리, 부분 점수 정책 적용 여부 확인 | |
| 4 | 다시 응시 시도 | 재응시 정책에 따라 응시 가능/불가 | |

**통과 기준**: 시간 만료 시 사용자 조작 없이 자동 제출, 점수 산출 일관

### S-S04. 동시 응시 (다중 탭) 시 데이터 충돌 [Medium]

## 자체 리뷰

**커버리지**: 핵심 흐름 8 + 예외 6 + 통합 3 = 17개 시나리오. PM4 의 회귀 매트릭스(B-01~B-11)와 연결되어 중복 없음.

**잠재 리스크**:
- S-S04(동시 응시), S-T04(채점 완료 확정), S-E03(마감 경합) 은 정책 미확정으로 FAIL 예상 — 정책 결정 후 재실행 필요

**개선 포인트**:
- 모바일/태블릿 시나리오는 현재 미포함 (반응형 별도 스크립트 권고)
- 접근성(키보드 only, 스크린리더) 별도 분리 필요
- 부하/동시성은 PM4 성능 영역으로 위임
