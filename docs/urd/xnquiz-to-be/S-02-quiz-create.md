## 1. UX 설계 목표

퀴즈 생성 화면의 1차 목표는 "정책 옵션이 많은 퀴즈 설정을 학습 부담 없이 단계적으로 입력" 하고, "검증 실패는 명확한 메시지로 알려 즉시 수정" 할 수 있도록 하는 것이다.

| **우선순위** | **UX 목표** | **설명** |
|---|---|---|
| 1 | 단계적 정보 입력 | 탭(기본 정보 / 문항 구성)으로 정보 영역 분리 |
| 2 | 옵션 의존성 자동 처리 | 마감일 없으면 지각 제출 비활성, 단일 응시면 재응시 옵션 숨김 등 |
| 3 | 검증 즉시 피드백 | 저장 시 첫 번째 실패 항목 메시지 표시, 작성 데이터 보존 |
| 4 | 임시저장 안전망 | 제목만 있으면 언제든 임시저장 가능, 비공개 보장 |
| 5 | 위험 옵션 안내 | 재응시 + 즉시 공개 같은 위험 조합은 저장 전 경고 |

|---|---|
| INSTRUCTOR | 화면 정상 렌더링 |
| ADMIN | 화면 정상 렌더링 (교수자 권한 포함) |
| STUDENT | `<Navigate to="/" replace />` 로 즉시 홈으로 리디렉션 |

### LTI 모드 프리필

LearningX 에서 LTI 런치로 진입한 경우 (localStorage `xnq_lti_active='1'`), 다음 값을 자동 입력한다:

| **필드** | **소스** | **유효 범위** |
|---|---|---|
| 주차 | `xnq_lti_week` | 과목 설정에 등록된 주차 범위 내 정수만 (예: 16주 강의면 1~16) |
| 차시 | `xnq_lti_session` | 해당 주차에 등록된 차시 범위 내 정수만 |

> 주차/차시 범위는 과목 설정에 따라 가변이다. 과목 메타 정보를 조회하여 유효성 검증을 수행하며, 범위를 벗어난 값은 `null` 로 처리한다.

## 4. [기본 정보 탭] 컴포넌트별 상세

### 4.1 퀴즈 유형 (Section: 퀴즈 유형)

| **요소** | **타입** | **기본값** | **동작** |
|---|---|---|---|
| 평가용 퀴즈 | 버튼 카드 | 선택됨 | `quizMode='graded'`, "성적에 반영됩니다" 설명 |
| 연습용 퀴즈 | 버튼 카드 | 미선택 | `quizMode='practice'`, 선택 시 황색 안내 박스 노출: "연습용 퀴즈는 성적에 반영되지 않습니다." |

### 4.2 기본 정보 (Section: 기본 정보)

| **필드** | **타입** | **필수** | **검증 규칙** |
|---|---|---|---|
| 퀴즈 제목 | text input | 예 | 빈 문자열 불가. 임시저장에도 제목 필수 |
| 설명 | textarea (min-h 176px, resize-y) | 아니오 | 제한 없음, HTML 미지원 (plain text) |
| 주차 | WeekSessionPicker (드롭다운) | 아니오 | 과목 설정에 등록된 주차 범위 또는 미지정 (`null`). 범위는 과목마다 가변 |
| 차시 | WeekSessionPicker (드롭다운) | 아니오 | 해당 주차에 등록된 차시 범위 또는 미지정. 차시 수는 과목/주차마다 가변. 주차 미지정 시 차시도 미지정 |

### 4.3 응시 기간 (Section: 응시 기간)

| **필드** | **타입** | **기본값** | **검증 규칙** |
|---|---|---|---|
| 시작 일시 | DateTimePicker | `''` | 비워두면 "응시 기간 제한 없음" 으로 처리 |
| 마감 일시 | DateTimePicker | `''` | `dueDate > startDate` 필수 (둘 다 있을 때) |
| 이용 종료 일시 | DateTimePicker | `''` | 권장: `lockDate >= dueDate`. 위반 시 황색 경고 박스 표시 (저장은 가능) |
| 마감 후 지각 제출 허용 | Toggle | `false` | 토글 활성 시 "지각 제출 마감 일시" 입력 노출 |
| 지각 제출 마감 일시 | DateTimePicker | `''` | `lateSubmitDeadline >= dueDate`. 비우면 "무제한 허용" |

#### 의존성 규칙

| **조건** | **결과** |
|---|---|
| `dueDate` 없음 + `allowLateSubmit=true` + `lateSubmitDeadline` 입력 | 저장 시 검증 오류: "지각 제출 마감 일시는 마감 일시가 설정되어 있을 때만 사용할 수 있습니다" |
| `lockDate < dueDate` | 황색 경고 박스 표시: "이용 종료 일시가 마감 일시보다 앞서 있습니다. 마감 전에 퀴즈 접근이 차단될 수 있습니다." (저장은 차단하지 않음) |

#### 툴팁 명세

| **필드** | **툴팁 내용** |
|---|---|
| 마감 일시 | "학생이 퀴즈를 제출해야 하는 기한입니다. 마감 이후에는 제출이 불가합니다." |
| 이용 종료 일시 | "퀴즈 페이지 자체에 접근할 수 없게 되는 시점입니다. 마감 이후에도 학생이 결과를 확인할 수 있도록 종료 일시는 마감 일시 이후로 설정하는 것을 권장합니다." |

### 4.4 추가 기간 설정 (Section: 추가 기간 설정)

`AssignmentOverrides` 컴포넌트로 특정 학생에게 다른 응시 기간을 부여한다.

| **요소** | **동작** |
|---|---|
| 학생 검색/선택 | 다중 선택 가능 |
| 기간 설정 | 학생 그룹별 startDate/dueDate 별도 지정 |
| 동일 학생 중복 검사 | 동일 학생이 여러 그룹에 포함되면 저장 시 검증 오류: "동일한 학생이 여러 추가 기간 설정에 포함되어 있습니다" |
| 저장 시 정제 | `sanitizeAssignments()` 로 빈 그룹 및 무효 데이터 제거 후 저장 |

### 4.5 응시 설정 (Section: 응시 설정)

#### 시간 제한

| **상태** | **표시 요소** |
|---|---|
| 시간 제한 사용 토글 OFF | `timeLimit=0`, `unlimitedTimeLimit=true`, "제한시간 없음" 으로 저장. 자동 제출 5분 유예 토글 자동 OFF |
| 시간 제한 사용 토글 ON (기본) | `timeLimit` 입력 (분, 정수, 1 이상). 자동 제출 5분 유예 토글 노출 |

#### 자동 제출 5분 유예

| **요소** | **동작** |
|---|---|
| 토글 | `disableAutoSubmit` |
| 설명 | "제한 시간이 끝난 후 5분간 직접 제출이 가능하고, 5분이 지나면 자동 제출됩니다" |
| 의존성 | 활성화 시 `lockDate` 필수. 미설정 상태에서 저장 시 검증 오류: "자동 제출 5분 유예 사용 시 이용 종료 일시를 반드시 설정해야 합니다" |
| 표시 조건 | 시간 제한 사용 ON 일 때만 노출 |

#### 재응시 허용

| **요소** | **동작** |
|---|---|
| 토글 | 활성 시 `allowAttempts=2` 로 초기화. 비활성 시 `allowAttempts=1` |
| 적용할 점수 | 드롭다운: 최고 점수 유지 (기본) / 최신 점수 유지 / 평균 점수 |
| 제출 횟수 제한 | 드롭다운: 2회 ~ 10회, 무제한 (`allowAttempts=-1`) |
| 자동 옵션 동기화 | 재응시 허용 OFF 시 `scorePolicy=null` 로 저장 |

### 4.6 문항 표시 설정 (Section: 문항 표시 설정)

| **토글** | **저장 키** | **동작** |
|---|---|---|
| 선지 순서 섞기 | `shuffleChoices` | 객관식 선지 순서가 학생마다 무작위 |
| 문제 순서 섞기 | `shuffleQuestions` | 문항 순서가 학생마다 무작위 |
| 한 번에 한 문항씩 표시 | `oneQuestionAtATime` | 활성 시 "응답 후 문항 잠금" 토글 노출. 비활성 시 잠금 자동 OFF |
| 응답 후 문항 잠금 | `lockAfterAnswer` | `oneQuestionAtATime` 가 ON 일 때만 의미 있음. OFF 로 돌아가면 자동으로 `false` 저장 |

### 4.7 성적 공개 정책 (Section: 성적 공개 정책)

| **요소** | **동작** |
|---|---|
| 성적 공개 토글 (`scoreRevealEnabled`) | 활성 시 하위 옵션 노출, 비활성 시 모든 하위 값 `null` 로 저장 |

#### 공개 범위 (`scoreRevealScope`)

| **값** | **라벨** | **설명** |
|---|---|---|
| `wrong_only` | 오답 여부만 | 정오답(O/X) + 점수 표시, 정답은 공개하지 않음 |
| `with_answer` | 정답까지 | 정오답(O/X) + 점수 + 정답 표시 |

#### 공개 시점 (`scoreRevealTiming`)

| **값** | **라벨** | **추가 입력** |
|---|---|---|
| `immediately` | 제출 즉시 | 없음 |
| `after_due` | 마감 후 | 없음 (`dueDate` 필요) |
| `period` | 기간 설정 | 공개 시작일/종료일 (DateTimePicker 2개) |

#### 응답 1회만 조회 허용 (`oneTimeResults`)

| **요소** | **동작** |
|---|---|
| 토글 | 활성 시 제출 직후 1회만 응답/정답 노출, 이후 재접근 시 비공개 |

#### 위험 조합 경고

| **조건** | **처리** |
|---|---|
| 재응시 허용 ON AND `scoreRevealEnabled=true` AND `scoreRevealTiming ∉ {period, after_due}` | "저장하기" 클릭 시 확인 모달 1단계 노출: "재응시가 허용된 퀴즈에서 점수 공개 기간이 설정되지 않으면, 1차 응시 마감 직후 점수(및 정답)가 공개되어 학생이 2차 응시 전에 정답을 확인할 수 있습니다." 사용자가 확인하면 저장 진행 |

### 4.8 퀴즈 접근 제한 (Section: 퀴즈 접근 제한 + 우측 Switch)

| **상태** | **표시** |
|---|---|
| Switch OFF (`accessControlEnabled=false`, 기본) | "접근 제한이 필요한 경우 우측 토글로 활성화하세요." 안내만 표시 |
| Switch ON | 액세스 코드 입력 + 접근 가능한 IP 주소 입력 노출 |

#### 액세스 코드

| **항목** | **명세** |
|---|---|
| 타입 | text input |
| placeholder | "코드를 입력하면 응시 시 코드 입력이 필요합니다" |
| 빈 값 처리 | "비워두면 액세스 코드 없이 응시 가능합니다." 안내. 저장 시 `accessCode=null` |

#### 접근 가능한 IP 주소

| **항목** | **명세** |
|---|---|
| 타입 | textarea (3 lines) |
| 형식 | 한 줄에 하나, CIDR 표기법 지원 (예: `192.168.1.0/24`) |
| 빈 값 처리 | "비워두면 모든 IP에서 접근 가능합니다." 안내. 저장 시 `ipRestriction=null` |

#### Switch OFF 시 저장 처리

`accessControlEnabled=false` 면 `accessCode`, `ipRestriction` 모두 `null` 로 저장 (입력 값이 남아있어도 무시).

### 4.9 퀴즈 안내사항 (Section: 퀴즈 안내사항 + 우측 Switch)

| **상태** | **표시** |
|---|---|
| Switch OFF (기본) | "안내사항을 표시하려면 우측 토글로 활성화하세요." 안내만 표시 |
| Switch ON | textarea 노출, 기본값으로 `DEFAULT_NOTICE` 채워짐 |

#### 기본 안내 문구 (`DEFAULT_NOTICE`)

```
- 제출 후에는 답안을 수정할 수 없습니다.
- 타인과의 협력 및 자료 공유는 금지됩니다.
- 부정행위 적발 시 해당 퀴즈 점수는 0점 처리됩니다.
```

저장 시 `noticeEnabled=false` 면 `notice=null` 로 저장.

### 4.10 퀴즈 공개 여부 (Section: 퀴즈 공개 여부)

| **요소** | **저장 키** | **기본값** |
|---|---|---|
| 학생에게 퀴즈 공개 토글 | `visible` | `true` |
| 설명 | - | "비공개 시 학생 화면에 퀴즈가 표시되지 않습니다. 임시저장 상태는 자동 비공개입니다." |

> 임시저장 (status=draft) 은 `visible` 값과 무관하게 학생 화면에 노출되지 않는다. 이 동작은 백엔드에서도 강제되어야 한다.

## 5. [문항 구성 탭] 컴포넌트별 상세

### 5.3 개별 문항 카드

| **요소** | **표시 내용** |
|---|---|
| 드래그 핸들 (GripVertical) | 좌측, cursor-grab |
| 순번 | 1부터 시작 |
| 유형 배지 | `QUIZ_TYPES[type].label` (객관식/단답형/서술형/...) |
| 점수 | `{points}점` |
| 수동채점 배지 | `QUIZ_TYPES[type].autoGrade === false` 인 경우 (서술형, 파일 업로드 등) 황색 "수동채점" 배지 |
| 편집 버튼 | 호버 시 노출 (현재 프로토타입에서는 동작 미연결, 향후 편집 모달 연결) |
| 삭제 버튼 | 호버 시 노출, 클릭 즉시 제거 (확인 없음) |
| 문제 본문 | `htmlToPlainText(q.text)` 로 HTML 제거 후 2줄 truncate (line-clamp-2) |
| 정답 영역 | 서술형/파일 업로드/주관식 단답형 외 유형은 `QuestionAnswer` 컴포넌트로 정답 표시 |

### 5.4 드래그 앤 드롭 정렬

| **단계** | **처리** |
|---|---|
| dragstart | 시작 인덱스 저장 |
| dragover | 대상 카드에 `border-primary bg-accent/30 shadow-sm` 시각 피드백 |
| drop | `onMove(fromIdx, toIdx)` 호출, 배열 순서 변경 |
| dragend | 시각 피드백 초기화 |

> 드래그 중 본인 카드는 `opacity-40` 으로 처리하여 어디서 출발했는지 표시한다.

## 6. 하단 액션

| **버튼** | **variant** | **위치** | **동작** |
|---|---|---|---|
| 취소 | ghost | 좌측 | 작성 내용 있으면 `ConfirmDialog`, 없으면 즉시 `/` 이동 |
| 임시저장 | outline | 우측 | 제목만 있으면 저장. 검증 통과 여부 무관 |
| 저장하기 | default | 우측 | 전체 검증 → 실패 시 AlertDialog, 성공 시 `status=open` 으로 저장 |

### 6.1 취소 동작

| **상태** | **동작** |
|---|---|
| 작성 내용 없음 (`title='', description='', questions=[]`) | 즉시 `/` 이동 |
| 작성 내용 있음 | `ConfirmDialog`: "작성 중인 내용이 있습니다. 저장하지 않고 나가시겠습니까?" |

### 6.2 임시저장 동작

| **조건** | **결과** |
|---|---|
| `title` 비어있음 | `AlertDialog` (variant=error): "임시저장 불가 - 퀴즈 제목을 입력해주세요." |
| `title` 입력됨 | `persistQuiz('draft')` 호출 → `AlertDialog`: "임시저장 완료 - 퀴즈가 임시저장되었습니다." |
| 저장 실패 | `AlertDialog` (variant=error): "임시저장 실패 - {에러 메시지}" |

### 6.3 저장하기 검증 (`getValidationErrors`)

저장 시 아래 순서로 검증하고, **첫 번째 실패 항목 메시지만** `AlertDialog` 로 표시한다.

| **순서** | **검증** | **실패 메시지** |
|---|---|---|
| 1 | 제목 입력 | "퀴즈 제목을 입력해주세요" |
| 2 | 시작 < 마감 | "마감 일시는 시작 일시 이후여야 합니다" |
| 3 | 이용 종료 >= 마감 | "이용 종료 일시는 마감 일시 이후로 설정해야 합니다" |
| 4 | 마감 없이 지각 마감 설정 불가 | "지각 제출 마감 일시는 마감 일시가 설정되어 있을 때만 사용할 수 있습니다" |
| 5 | 시간 제한 > 0 또는 무제한 | "제한 시간을 입력하거나 무제한으로 설정해주세요" |
| 6 | 자동 제출 5분 유예 시 lockDate 필수 | "자동 제출 5분 유예 사용 시 이용 종료 일시를 반드시 설정해야 합니다" |
| 7 | 문항 1개 이상 | "최소 1개 이상의 문항을 추가해주세요" |
| 8 | 추가 기간 학생 중복 없음 | "동일한 학생이 여러 추가 기간 설정에 포함되어 있습니다" |

### 6.4 저장 성공 시

| **단계** | **처리** |
|---|---|
| 1 | `createQuiz(buildQuizBody('open'))` 호출 |
| 2 | 응답으로 받은 `id` 로 `setQuizQuestions(id, questions)` 호출 |
| 3 | `/` 로 이동 (목록 화면) |
| 4 | 목록 화면에서 토스트는 별도 표시 안 함 (현재 명세) |

> 향후 sessionStorage `xnq_toast` 로 토스트 전달 옵션 고려 가능.

## 8. Edge Case

| **시나리오** | **처리** |
|---|---|
| 시간 제한 input 에 음수/0/문자열 입력 | 저장 시 검증 오류 (5번) |
| 시작/마감 둘 다 비어있음 | 검증 통과 (응시 기간 제한 없음) |
| 시작만 있고 마감 없음 | 검증 통과 (마감 제한 없음) |
| 마감만 있고 시작 없음 | 검증 통과 (시작 즉시 응시 가능) |
| 재응시 토글 ON → OFF | `allowAttempts=1`, `scorePolicy=null`, `unlimitedAttempts=false` 로 자동 복원 |
| 접근 제한 Switch OFF → ON → 입력 → OFF | 저장 시 `null` 처리되어 입력값 보존되지 않음 |
| 추가 기간 그룹 0개 | 저장 통과, `assignments=[]` |
| 문항 만들기 도중 다른 모달 진입 | 본 명세 범위 외 (모달 동시 오픈 방지는 각 모달 컴포넌트 책임) |
| createQuiz 성공 + setQuizQuestions 실패 | 현재 별도 보상 트랜잭션 없음. 백엔드 결합 시 결정 필요 |
| LTI 프리필 값이 범위 밖 (예: week=99) | `null` 로 처리하고 사용자가 직접 입력 |
| localStorage 접근 차단 환경 | LTI 프리필 skip, `week=null, session=null` |

## 9. 저장 데이터 모델 (buildQuizBody)

저장 시 백엔드로 전송되는 객체 구조 (status 는 `draft` 또는 `open`):

```ts
{
  title: string,
  description: string,
  course: string,                  // 현재는 'CS301 데이터베이스' 하드코딩, LTI 시 자동 매핑 필요
  quizMode: 'graded' | 'practice',
  status: 'draft' | 'open',
  visible: boolean,
  startDate: string | null,
  dueDate: string | null,
  lockDate: string | null,
  week: number | null,             // 1~16
  session: number | null,          // 1~4
  timeLimit: number,               // 0 = 무제한, 양수 = 분
  allowAttempts: number,           // -1 = 무제한, 1 = 단일 응시, 2~10 = 재응시
  scorePolicy: string | null,      // 재응시 허용 시에만 값. '최고 점수 유지' / '최신 점수 유지' / '평균 점수'
  shuffleChoices: boolean,
  shuffleQuestions: boolean,
  scoreRevealEnabled: boolean,
  scoreRevealScope: 'wrong_only' | 'with_answer' | null,
  scoreRevealTiming: 'immediately' | 'after_due' | 'period' | null,
  scoreRevealStart: string | null,
  scoreRevealEnd: string | null,
  oneTimeResults: boolean,
  accessCode: string | null,
  ipRestriction: string | null,
  allowLateSubmit: boolean,
  lateSubmitDeadline: string | null,
  gracePeriod: number,             // 분 단위, 0 또는 양수
  disableAutoSubmit: boolean,
  oneQuestionAtATime: boolean,
  lockAfterAnswer: boolean,
  assignments: Array<AssignmentOverride>,
  notice: string | null,
  totalStudents: 0,                // 생성 시점 초기값
  submitted: 0,
  graded: 0,
  pendingGrade: 0,
  questions: number,               // 문항 수 (자동 계산)
  totalPoints: number,             // 총점 (자동 계산)
}
```

## 10. 향후 백엔드 결합 시 확인 필요 사항 (Open Issue)

| **항목** | **결정 필요 사항** |
|---|---|
| `course` 필드 처리 | LTI 모드는 Canvas courseCode, 비 LTI 는 어떻게? 자동 매핑 필요 |
| `assignments` 데이터 모델 | 학생 ID 배열 vs. 그룹 객체 배열, 백엔드와 합의 필요 |
| `ipRestriction` 검증 | CIDR 형식 서버에서 한 번 더 검증할지 여부 |
| `accessCode` 보안 | 평문 저장 vs. 해싱. 응시 시 비교 방식 |
| `notice` 길이 제한 | 클라이언트는 제한 없음, 서버 정책 필요 |
| 동시 편집 충돌 | 본 화면은 신규 생성이라 영향 적으나, 임시저장 → 편집 전환 시 정책 필요 |
| createQuiz + setQuizQuestions 원자성 | 두 API 호출 사이 실패 시 보상 트랜잭션 필요 여부 |
| LTI 프리필 외 필드 자동 채움 | course, 강의명, 기관 ID 등 추가 자동 채움 검토 |
