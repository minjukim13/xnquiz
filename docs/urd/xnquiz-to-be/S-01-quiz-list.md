## 0. (AS-IS) 대비 개선 포인트

xnquiz 가 Canvas 표준 (AS-IS) 대비 본 화면에서 개선한 주요 사항을 정리한다. 각 항목은 [Canvas AS-IS 의 4절](../canvas-as-is/S-01-quiz-list.md#4-현재-ui-의-개선-필요-포인트) 에서 식별된 문제를 해결한다.

> LearningX custom.js 보강분은 본 화면(`/courses/:id/quizzes`) 에 직접 적용된 스크립트가 없어 별도 (AS-IS) 문서를 작성하지 않았다. 주차/차시 selector 등은 퀴즈 편집 화면(S-02) 에 한정되며, 해당 화면 URD 에서 다룬다.

## 1. UX 설계 목표

퀴즈 목록 화면의 1차 목표는 "내가 가르치는 과목에서 지금 어떤 퀴즈가 어떤 상태로 운영되고 있는지" 를 한눈에 파악하고, 신규 작성/편집/채점/통계로 빠르게 이동하는 것이다.

| **우선순위** | **UX 목표** | **설명** |
|---|---|---|
| 1 | 운영 현황 파악 | 퀴즈별 상태(예정/진행중/마감/임시저장), 응시율, 미제출 인원 확인 |
| 2 | 빠른 진입 | 카드 클릭 1회로 상세, 메뉴 1회로 편집/채점/통계 이동 |
| 3 | 자산 재사용 | 복사, 다른 과목 퀴즈 가져오기로 신규 작성 부담 절감 |
| 4 | 학생 노출 통제 | 공개/숨김 즉시 전환, 임시저장은 자동 비공개 |
| 5 | 정렬/필터 | 주차/차시, 마감임박, 최근생성 기준으로 빠른 좁히기 |

## 4. 컴포넌트별 상세

### 4.1 헤더 영역 (교수자 전용)

| **요소** | **표시 조건** | **동작** |
|---|---|---|
| 화면 제목 "퀴즈 관리" | 항상 | - |
| 전역 설정 버튼 (⚙) | 항상 | 클릭 시 `QuizSettingsDialog` 모달 오픈 (점수 표시 자릿수 등 사용자 환경설정) |
| 가져오기 버튼 | 항상 | 클릭 시 `QuizImportModal` 오픈 |
| 새 퀴즈 버튼 | 항상 | `/quiz/new` 로 이동 |

학생 뷰는 "내 퀴즈" 텍스트만 표시, 액션 버튼 없음.

### 4.2 필터/정렬 영역

#### 주차 드롭다운

| **옵션** | **값** | **동작** |
|---|---|---|
| 전체 주차 | `all` (기본) | 주차 필터 미적용 |
| 미지정 | `unassigned` | `week=null OR week=0` 인 퀴즈만 |
| N주차 | 과목 설정에 등록된 주차 번호 | `week=N` 인 퀴즈만 |

> **주차 범위는 과목 설정에 따라 가변이며, 고정값이 아니다.** 과목별로 정의된 총 주차 수만큼 옵션이 동적으로 노출된다 (예: 8주 강의는 1~8주차, 16주 강의는 1~16주차 노출). 과목 주차 정보는 LearningX/Canvas 과목 설정 또는 별도 과목 메타 API 에서 가져온다.

주차 변경 시 차시 값은 `all` 로 자동 초기화.

#### 차시 드롭다운

| **상태** | **표시** |
|---|---|
| 주차 = 전체 | 비활성화 (선택 불가) |
| 주차 = 특정 주차 | 해당 주차에 등록된 차시만 동적으로 노출 |
| 주차 = 미지정 | 미지정 퀴즈에 존재하는 차시만 |

> **차시 범위도 과목/주차 설정에 따라 가변이다.** 차시 수는 과목 정책에서 정의된 값을 따른다 (예: 주 1차시 강의 vs. 주 2~3차시 강의). 화면에서는 클라이언트 사이드 하드코딩 없이 과목 메타 API 응답에 따라 옵션을 구성한다.

#### 정렬 드롭다운

| **옵션** | **정렬 기준** |
|---|---|
| 최근생성순 (기본) | `createdAt DESC`, 없으면 `id DESC` |
| 주차 오름차순 | `week ASC, session ASC`, 미지정은 뒤로 |
| 주차 내림차순 | `week DESC, session DESC`, 미지정은 뒤로 |
| 마감임박순 | `dueDate ASC`, 마감 지난 항목과 마감 없는 항목은 뒤로 |

### 4.3 퀴즈 카드 (교수자 뷰)

#### 카드 좌측 영역

| **요소** | **표시 조건** | **표시 내용** |
|---|---|---|
| 공개 배지 | `status≠draft` | `공개` (visible=true) 또는 `비공개` (visible=false) |
| 상태 배지 | 항상 | 임시저장 / 예정 / 진행중 / 채점중 / 마감 |
| 주차/차시 배지 | `week>0 OR session>0` | 예: "3주차 1차시" |
| 퀴즈 제목 | 항상 | 한 줄 truncate |
| 응시 기간 | 항상 | `startDate ~ dueDate` 또는 "응시 기간 제한 없음" |
| 이용 종료 안내 | `lockDate` 존재 | "이용 종료: YYYY-MM-DD HH:mm" |
| D-day 배지 | `status=open` AND `dueDate` 존재 AND 미마감 | "D-0" (당일은 빨강), "D-7" (그 외는 황색) |
| 지각 제출 안내 | `allowLateSubmit=true` | "지각 제출: {lateSubmitDeadline}까지" 또는 "지각 제출: 무제한 허용" |

#### 카드 우측 통계 (status/scheduled에 따라 다른 4종)

**임시저장 OR 예정 상태**:

| **라벨** | **값** |
|---|---|
| 문항 수 | `{questions}개` |
| 총점 | `{totalPoints}점` |
| 제한시간 | `{timeLimit}분` 또는 "없음" |

**진행중/채점중/마감 상태**:

| **라벨** | **값** | **색상** |
|---|---|---|
| 응시율 | `round(submitted/totalStudents * 100)%` | primary |
| 응시인원 | `{submitted}명` | foreground |
| 미제출 | `{totalStudents - submitted}명` | 0 초과 시 red-500, 0 이면 muted |
| 평균점수 | `{avgScore}점` 또는 "-" | primary |

### 4.4 퀴즈 카드 (학생 뷰)

## 6. Edge Case 및 빈 상태

| **시나리오** | **처리** |
|---|---|
| 본인 과목에 퀴즈가 1개도 없음 | "해당 조건에 맞는 퀴즈가 없습니다." 빈 상태 표시. 새 퀴즈 버튼은 항상 헤더에 노출되어 있어 즉시 작성 가능 |
| 필터 조건에 맞는 퀴즈 0건 | 동일 빈 상태 |
| api 모드에서 `listQuizzes` 실패 | 콘솔 에러 로그, 빈 배열로 fallback, 빈 상태 표시 |
| 가져오기 모달에서 다른 과목 0건 | 좌측 패널에 "다른 과목이 없습니다" 표시 |
| 가져오기에서 과목 선택 후 공개 퀴즈 0건 | 우측 패널에 "공개된 퀴즈가 없습니다" 표시 |
| 학생 뷰에서 모든 퀴즈가 visible=false | "현재 응시 가능한 퀴즈가 없습니다." 빈 상태 |
| 학생 뷰에서 필터 결과 0건 | "해당 조건에 맞는 퀴즈가 없습니다." 빈 상태 |
| 카드 응시율 계산 시 totalStudents=0 | 응시율 0% 표시 (NaN 방지) |
| 평균점수 데이터가 없음 (`avgScore=null`) | "-" 표시 |
| 마감일이 없는데 D-day 요청 | 배지 미표시 |
| 지각 제출 허용이지만 데드라인 미설정 | "지각 제출: 무제한 허용" 표시 |

## 7. 갱신 및 데이터 처리 요구사항

| **항목** | **요구사항** |
|---|---|
| 초기 로드 | 페이지 마운트 시 `listQuizzes()` 1회 호출 |
| 갱신 트리거 | 생성/편집/삭제/복사/가져오기/공개 토글 성공 후 `reload()` 호출 |
| 자동 갱신 (폴링) | 본 화면에서는 미지원. 운영 통계가 실시간성을 요구하는 경우는 추후 결정 |
| 클라이언트 필터링 | 비 LTI 모드에서는 `course === CURRENT_COURSE` 로 클라이언트 필터링 |
| 서버 필터링 | LTI 모드에서는 서버가 `courseCode` 로 사전 필터링하여 반환 |
| 정렬/주차/차시 필터 | 클라이언트에서 useMemo 로 파생, 서버 호출 없음 |
| 가져오기 시 다른 과목 조회 | LTI 모드: `listTeacherCourses({ excludeCourseCode })`. 비 LTI: `listCourses()` |

## 8. 향후 백엔드 결합 시 확인 필요 사항 (Open Issue)

| **항목** | **결정 필요 사항** |
|---|---|
| 과목 주차/차시 메타 조회 API | 과목별로 정의된 총 주차 수, 주차별 차시 수를 가져오는 엔드포인트 정의 필요. LearningX/Canvas 과목 설정 동기화 방식 결정 필요 |
| 응시율/평균점수 집계 시점 | 실시간 계산인지, 미리 집계된 값을 반환하는지 명확화 필요 |
| `totalStudents` 계산 기준 | 수강생 전체 vs. `assignments` 로 추가된 학생 포함 여부 |
| `submitted` 카운트 정의 | 최신 attempt 만 카운트 vs. 모든 응시 합산 (재응시 허용 시) |
| 채점 대기 (`pendingGrade`) | 수동채점 문항 수 기반인지, 응시별 카운트인지 정의 |
| LTI 모드 다른 과목 가져오기 권한 | Canvas API 가 교수자 본인 과목만 반환하는지 검증 필요 |
| draft 퀴즈 보호 | 다른 교수자가 본인 임시저장 퀴즈를 볼 수 없도록 서버에서 강제 |
