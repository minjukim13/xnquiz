# S-01. 퀴즈 목록 (Canvas 표준 AS-IS)

- **상위 문서**: [URD 통합 README](../README.md)
- **관점**: Canvas LMS 원본 동작. LearningX custom.js / xnquiz 개선 사항 포함하지 않음.
- **참고 소스**: Canvas LMS `app/views/quizzes/quizzes/index.html.erb`, `ui/features/quizzes_index/`
- **경로 (Canvas 원본)**: `/courses/:course_id/quizzes`
- **상태**: Draft

## 1. 화면 개요

Canvas 표준 퀴즈 목록은 한 과목 안의 퀴즈를 **유형별로 묶어서** 한 페이지에 펼쳐 보여주는 단순 목록 화면이다. 주차 필터나 정렬 옵션 없이, 그룹 안에서 백엔드 기본 순서대로 나열된다. 교수자는 카드 오른쪽의 점 세 개 메뉴(⋮, 이하 "더보기 메뉴")로 편집, 삭제, 보내기 같은 동작을 수행한다.

### 진입 위치

| **경로** | **조건** |
|---|---|
| `/courses/:course_id/quizzes` | 과목 좌측 메뉴에서 "퀴즈(Quizzes)" 클릭 |
| 과목 메뉴 노출 여부 | `Tabs::TAB_QUIZZES` 가 활성화된 경우에만 보임 (관리자가 숨길 수 있음) |

### 권한 분기

같은 컴포넌트(`IndexView`)를 쓰지만, 사용자 권한에 따라 보이는 요소가 달라진다.

| **권한 플래그** | **노출 요소** |
|---|---|
| `permissions.create` (퀴즈 생성 권한) | "퀴즈 추가(Create Quiz)" 버튼 |
| `permissions.manage` (퀴즈 관리 권한) | 헤더 더보기 메뉴(⋮), 각 카드의 공개/잠금 아이콘과 더보기 메뉴 |
| `permissions.read_question_banks` | "문제 은행 보기(View Question Banks)" / "문제 은행 관리(Manage Question Banks)" |
| 권한 없음 (학생) | 제목과 날짜만 표시. 액션 버튼은 일체 없음 |

## 2. 화면 구성

### 2.1 와이어프레임 (교수자 화면)

```
┌──────────────────────────────────────────────────────────────┐
│ Quizzes                                                      │
│ [🔍 Search for Quiz                ]   [⋮]   [+ Create Quiz] │
├──────────────────────────────────────────────────────────────┤
│ ▼ Assignment Quizzes                                         │
│ ┌──────────────────────────────────────────────────────────┐ │
│ │ 📝  Midterm: DB Design                           [✓][⋮] │ │
│ │     Available until May 17 | Due May 17 | 100 pts | 30Q │ │
│ └──────────────────────────────────────────────────────────┘ │
│ ┌──────────────────────────────────────────────────────────┐ │
│ │ 📝  Week 4 Pop Quiz                               [○][⋮] │ │
│ │     Not yet published | 20 pts | 10Q                    │ │
│ └──────────────────────────────────────────────────────────┘ │
├──────────────────────────────────────────────────────────────┤
│ ▼ Practice Quizzes                                           │

├──────────────────────────────────────────────────────────────┤

```

> 위에서 아래로: 화면 제목과 검색창/메뉴 영역, 과제 퀴즈 그룹(공개된 카드 1건과 비공개 카드 1건), 연습 퀴즈 그룹(빈 상태), 설문 그룹(빈 상태) 순서다. 각 카드 오른쪽의 ✓/○ 아이콘은 공개 여부, ⋮ 아이콘은 카드별 더보기 메뉴를 의미한다.

### 2.2 와이어프레임 (학생 화면)

```
┌──────────────────────────────────────────────────────────────┐
│ Quizzes                                                      │
│ [🔍 Search for Quiz                ]                         │
├──────────────────────────────────────────────────────────────┤
│ ▼ Assignment Quizzes                                         │
│ ┌──────────────────────────────────────────────────────────┐ │
│ │ 📝  Midterm: DB Design                                   │ │
│ │     Available until May 17 | Due May 17 | 100 pts | 30Q │ │
│ └──────────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────────┘
```

> 학생 화면은 검색창만 있고 더보기 메뉴와 추가 버튼은 없다. 카드 오른쪽의 공개/잠금/더보기 아이콘도 사라지고, 제목과 응시 정보만 노출된다.

## 3. 컴포넌트별 상세

### 3.1 헤더 영역

| **요소** | **표시 조건** | **동작** |
|---|---|---|
| 화면 제목 "퀴즈(Quizzes)" | 항상 | 정적 텍스트 |
| 검색창 ("Search for Quiz") | 항상 | `data-view="inputFilter"` 에 바인딩되어, 입력하는 즉시 클라이언트 사이드에서 제목 부분 일치로 필터링됨 (서버 호출 없음) |
| 퀴즈 설정 버튼 (⋮) | `permissions.create` 또는 `permissions.manage` 가 있을 때 | 드롭다운 메뉴를 연다 (아래 3.2 참고) |
| 퀴즈 추가 버튼 ("Create Quiz", +) | `permissions.create` 가 있을 때 | 클릭 즉시 빈 퀴즈를 생성(POST)하고 편집 화면으로 이동 |
| 문제 은행 보기 버튼 ("View Question Banks") | 학생이면서 `permissions.read_question_banks` 가 있을 때 | 문제 은행 페이지로 이동 |

### 3.2 퀴즈 설정 메뉴 (헤더의 ⋮)

| **메뉴 항목** | **표시 조건** |
|---|---|
| 문제 은행 관리 ("Manage Question Banks") | `permissions.read_question_banks && flags.question_banks` |
| 여러 퀴즈 일괄 공개 ("Publish Multiple Quizzes") | `permissions.manage && flags.publish_multiple` |
| 외부 도구 (LTI) | `placement = quiz_index_menu` 로 설치된 LTI 도구가 동적으로 추가됨 |

### 3.3 퀴즈 그룹 (Item Group)

Canvas는 퀴즈를 4가지 그룹으로 나눠서 보여준다. 그룹 단위로 접고 펼 수 있다.

| **그룹** | **포함 조건** | **표시 라벨 (원문)** |
|---|---|---|
| 과제 퀴즈 | `quiz_type = assignment` (점수가 기록되는 퀴즈) | "Assignment Quizzes" |
| 연습 퀴즈 | `quiz_type = practice_quiz` | "Practice Quizzes" |
| 설문 | `quiz_type` 이 `graded_survey` 또는 `survey` | "Surveys" |
| 빈 상태 그룹 | 위 세 그룹이 모두 비어있을 때만 단독으로 표시됨 | "Course Quizzes" / "No quizzes available" |

> 한 그룹에 퀴즈가 0건이라도 그룹 영역은 항상 보이며, "No quizzes found" 텍스트가 안에 들어간다.

### 3.4 퀴즈 카드 (QuizItemView)

#### 카드 왼쪽 영역

| **요소** | **표시 내용** |
|---|---|
| 아이콘 | `icon-quiz` |
| 제목 링크 | `title_label` (퀴즈명, 클릭하면 퀴즈 상세 화면으로 이동) |
| 상세 정보 (한 줄, 파이프(`|`) 구분) | 이용 시작일(Available date), 마감일(Due date), 배점(Points possible), 문항 수(Question count) |

> 표시할 데이터가 없는 항목은 아예 렌더링되지 않는다. 예를 들어 응시 기간이 설정되지 않으면 "Available" 자체가 화면에서 사라진다.

#### 카드 오른쪽 영역 (교수자만 보임)

| **요소** | **표시 조건** | **동작** |
|---|---|---|
| 마스터리 경로 라벨 ("Mastery Paths") | `cyoe.isTrigger` 또는 `cyoe.isReleased` | Conditional Release 기능과 연동 |
| SIS 버튼 | `postToSISEnabled && postToSIS !== null && published` | 학사 정보 시스템(SIS) 전송 토글 |
| 잠금 아이콘 | `can_update` | 클릭하면 잠금/잠금 해제 토글 |
| 공개 아이콘 (✓ 공개 / ○ 비공개) | `can_update` | 클릭하면 공개/비공개 토글 |
| 더보기 메뉴 (⋮) | `canOpenManageOptions` | 아래 3.5 참고 |

#### 학생 화면

학생은 카드의 왼쪽 영역(아이콘, 제목, 상세 정보)만 본다. 오른쪽 액션 영역은 아예 렌더링되지 않는다. 본인의 응시 결과(점수, 응시 여부)는 카드에 노출되지 않고, 카드를 클릭해 상세 화면으로 들어가야 확인할 수 있다.

### 3.5 카드 더보기 메뉴 (교수자, 카드 ⋮)

| **메뉴 항목** | **표시 조건** | **동작** |
|---|---|---|
| 편집 ("Edit") | `can_update` | 퀴즈 편집 화면으로 이동 |
| SpeedGrader | `assignment_id && showSpeedGraderLink && published` | SpeedGrader 를 새 창으로 연다 |
| 할당 대상 지정 ("Assign To") | `canManageAssignTo` | Assign-To 트레이를 연다 |
| 마스터리 경로 ("Mastery Paths") | `can_update && cyoe.isCyoeAble` | 편집 화면의 Mastery Paths 탭으로 이동 |
| 삭제 ("Delete") | `permissions.delete && !is_locked` | 브라우저 `confirm` 띄운 뒤 DELETE API 호출 |
| 보내기 ("Send to") | `DIRECT_SHARE_ENABLED` | 다른 사용자에게 직접 전송하는 모달을 연다 |
| 복사 ("Copy to") | `DIRECT_SHARE_ENABLED` | 다른 과목으로 복사하는 모달을 연다 |
| 외부 도구 (LTI) | `placement = quiz_menu` | LTI 외부 도구 호출 |

### 3.6 빈 상태 (NoQuizzesView)

| **요소** | **내용** |
|---|---|
| 그룹 헤더 | "Course Quizzes" |
| 본문 | "No quizzes available" |
| CTA | 없음. 안내 문구나 버튼 없이 텍스트만 노출 |

### 3.7 검색/필터/정렬

| **항목** | **지원 여부** |
|---|---|
| 제목 검색 | 지원 (클라이언트 사이드 부분 일치) |
| 주차/차시 필터 | **미지원** |
| 정렬 옵션 (마감 임박순, 최근 생성순 등) | **미지원**. 그룹 안의 표시 순서는 백엔드 기본 순서(주로 `position` 필드)를 따른다 |
| 상태 필터 (공개/비공개) | **미지원** |

## 4. 현재 UI 의 개선 필요 포인트

이 절은 Canvas 표준 퀴즈 목록을 한국 대학 LMS 운영자/교수자/학생 관점에서 사용했을 때 드러나는 UI/UX 개선 포인트를 정리한다.

### 4.1 정보 표시 측면

| **이슈** | **현재 상태** | **개선 필요 이유** |
|---|---|---|
| 운영 통계가 안 보임 | 카드에 응시율, 미제출 인원, 평균 점수가 노출되지 않는다 | 교수자가 한 학기 운영 상태를 파악하려면 매 퀴즈마다 SpeedGrader 에 들어가야 한다 |
| 상태 라벨이 단순함 | 공개/비공개(Published/Unpublished) 두 가지만 있다 (작은 아이콘 토글) | "예정 / 진행중 / 마감 / 채점중 / 임시저장" 같은 운영 단계별 상태가 카드에서 한눈에 구분되지 않는다 |
| D-day 시각화 없음 | 마감일은 텍스트로만 표시된다 (예: `Due May 17`) | 마감 임박 퀴즈를 색상이나 배지로 강조하는 시각 신호가 없다 |
| 주차/차시 표시 없음 | Canvas 원본에는 주차 개념 자체가 없다 | 한국 대학 LMS 는 주차 단위 운영이 필수인데, 카드에서 어느 주차 퀴즈인지 알 수 없다 |
| 학생 응시 결과가 카드에 없음 | 학생 카드에 본인 점수나 응시 여부 라벨이 없다 | 학생이 응시 여부를 확인하려면 매 퀴즈 상세로 들어가야 한다 |
| 지각 제출 정책이 안 보임 | 지각 허용 여부가 카드에서 보이지 않는다 | 학생과 교수자 모두 지각 허용 여부를 카드에서 즉시 인지할 수 없다 |

### 4.2 분류/필터 측면

| **이슈** | **현재 상태** | **개선 필요 이유** |
|---|---|---|
| 4개 그룹 강제 분류 | 과제/연습/설문/빈 상태(Assignment/Practice/Surveys/No Quizzes) 그룹이 항상 노출된다 | xnquiz 는 설문을 별도 화면으로 분리해서 운영 중이라 설문 그룹은 중복이다. 평가용/연습용 두 가지만 남기는 게 합리적 |
| 정렬 옵션 없음 | 정렬은 백엔드 기본 순서로 고정 | 마감 임박순, 최근 생성순, 주차순 같은 동선이 막힌다 |
| 주차/차시 필터 없음 | 검색은 제목 부분 일치만 가능 | 100건 이상 퀴즈가 쌓이면 원하는 퀴즈를 빠르게 좁힐 수단이 없다 |
| 상태 필터 없음 | 공개/비공개로 거를 수 없다 | "임시저장만 보기", "진행중만 보기" 같은 운영 동선이 막힌다 |

### 4.3 액션/동선 측면

| **이슈** | **현재 상태** | **개선 필요 이유** |
|---|---|---|
| 다른 과목에서 가져오기 미지원 | 직접 공유(Direct Share, "Send to"/"Copy to") 는 보내는 쪽 기반이다 | 받는 입장에서 "지난 학기 퀴즈를 이번 학기로 가져오기" 가 어렵다. 동선이 다단계(이전 과목 진입 → "Copy to" → 대상 과목 선택)로 길어진다 |
| 일괄 공개가 메뉴 안에 숨겨짐 | "여러 퀴즈 일괄 공개(Publish Multiple Quizzes)" 가 헤더 더보기 메뉴(⋮) 안에 있다 | 신학기 일괄 공개 시점에 동선 노출이 부족하다 |
| 삭제 확인이 브라우저 기본 다이얼로그 | `confirm("Are you sure?")` 네이티브 다이얼로그 | 디자인 시스템과 안 맞고, 모바일/태블릿에서 UX 일관성이 깨진다 |
| Optimistic Update 미적용 | 공개/비공개 토글 후 서버 응답을 기다리는 동안 UI 가 멈춘다 | 반응성이 떨어지고, 여러 건 공개할 때 체감 지연이 크다 |
| 마스터리 경로(Mastery Paths) 메뉴 잔존 | 카드 오른쪽 라벨과 더보기 메뉴에 마스터리 경로(Conditional Release) 가 표시된다 | 한국 대학에서 조건부 학습경로 기능은 거의 사용되지 않는다. xnquiz 에서는 메뉴 제거 가능 |

### 4.4 디자인/일관성 측면

| **이슈** | **현재 상태** | **개선 필요 이유** |
|---|---|---|
| 모달/툴팁 스타일 불일치 | 직접 공유 등 일부는 React 모달, 삭제 확인은 브라우저 네이티브 다이얼로그 | 같은 화면 안에서 모달 위계와 닫기 버튼 위치가 통일되지 않는다 |
| 빈 상태 안내가 약함 | "No quizzes available" 텍스트만 있고, 아이콘이나 다음 액션 버튼이 없다 | 신규 과목에 처음 들어왔을 때 다음에 뭘 해야 할지 유도가 없다 |
| 카드 정보 밀도가 낮음 | 한 카드에 한 줄(제목, 날짜, 배점, 문항 수)만 있다 | 운영 통계, 주차, 상태 같은 핵심 정보가 카드에서 한눈에 안 보인다 |

### 4.5 접근성/반응형 측면

| **이슈** | **현재 상태** | **개선 필요 이유** |
|---|---|---|
| 모바일 헤더 부담 | 검색창, 더보기 메뉴(⋮), 퀴즈 추가 버튼이 한 줄에 몰려 있다 | 좁은 뷰포트에서 줄바꿈되거나 잘린다 |
| 그룹 접기 상태가 저장되지 않음 | 새로고침하면 모든 그룹이 펼침 상태로 초기화된다 | 자주 닫는 그룹을 매번 다시 닫아야 한다 |
| 학생 화면의 정보가 부족함 | 본인 응시 결과가 어디에 나오는지 일관성이 부족하다 | 결과 화면에 들어가기 전까지 본인 진행 상태를 파악할 수 없다 |

## 5. Edge Case 및 데이터 처리

| **시나리오** | **Canvas 동작** |
|---|---|
| 한 과목에 퀴즈가 0건 | "Course Quizzes" 헤더와 "No quizzes available" 텍스트만 표시 |
| 연습 퀴즈 그룹만 0건 | 그룹 영역은 그대로 보이고 안에 "No quizzes found" 만 표시 (그룹 자체가 사라지지는 않음) |
| 마감일이 지난 공개 퀴즈 | 카드에 별도 변화 없음. 마감 텍스트만 과거 날짜 그대로 표시 |
| 외부 도구(LTI) 메뉴 | `placement` 값에 따라 헤더 더보기 메뉴와 카드 더보기 메뉴에 동적으로 항목이 추가됨 |
