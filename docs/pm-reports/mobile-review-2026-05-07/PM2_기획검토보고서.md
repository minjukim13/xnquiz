# PM2 기획 검토 보고서 — 모바일 사용자 시나리오 점검

**작성일**: 2026-05-07
**관점**: 교수자/학습자/운영자 시나리오에서 모바일 환경의 사용성 정합성 검증

---

## 1. 모바일 사용 빈도 가설

| 사용자 | 모바일 사용 시나리오 | 빈도 가설 |
|--------|-------------------|----------|
| **학습자** | 출퇴근/통학 중 응시, 마감 임박 알림 후 즉시 응시 | **높음** (전체 응시의 30~40% 추정) |
| **교수자** | 외부 이동 중 채점 진행상황 확인, 응시 현황 모니터링 | 중간 (보조 화면 위주) |
| **교수자** | 모바일에서 문항 직접 추가/편집, 채점 작성 | **낮음** (지양해야 할 시나리오) |
| **운영자** | 모바일에서 정책 변경, 권한 관리 | 매우 낮음 |

> **핵심 결론**: 학습자 응시 + 교수자 모니터링이 모바일의 1순위. 문항 작성/대규모 채점은 모바일 보조 수준이면 충분.

## 2. 시나리오별 검토

### 2-1. 학습자 — 모바일 응시

| 단계 | 점검 결과 | 비고 |
|------|----------|------|
| 퀴즈 목록 진입 → 응시 시작 | 양호 | QuizList의 주차/차시 필터가 flex-wrap으로 줄바꿈 처리됨 |
| 응시 화면 — 객관식/주관식 | 양호 | QuestionAnswer 컴포넌트 라디오/체크박스 터치 타겟 충분 |
| 응시 화면 — 연결 문제(matching) | **수정 후 개선** | 좌-↔-우 가로 배치 → 모바일에서 stack 처리 (수정됨) |
| 응시 화면 — 수치형(numerical) | **수정 후 개선** | 정답/허용오차 grid-cols-2 → 모바일 stack (수정됨) |
| 결과 화면 — 메타 3열 (시간/문항/만점) | 양호 | 짧은 정보라 grid-cols-3 유지 OK |
| 결과 화면 — 결과 자세히 보기 | 양호 | 풀스크린 진입, 별도 이슈 없음 |

### 2-2. 교수자 — 모바일 모니터링/채점

| 단계 | 점검 결과 | 비고 |
|------|----------|------|
| 채점 대시보드 진입 | 양호 | 모바일 탭 전환(`문항 목록` ↔ `Q? 상세`) 잘 구현됨 |
| 학생 활동 로그 패널 (5요소 통계) | **수정 후 개선** | grid-cols-5 → 모바일 grid-cols-2 stack 처리 (수정됨) |
| 통계 탭 4요소 카드 | **수정 후 개선** | grid-cols-4 → 모바일 grid-cols-2 stack 처리 (수정됨) |
| 조건부 재응시 부여 모달 — Step 2 학생 목록 | **수정 후 개선** | 5컬럼 그리드 → 모바일 카드형 단일행 압축 (수정됨) |

### 2-3. 교수자 — 모바일 문항 편집(보조)

| 단계 | 점검 결과 | 비고 |
|------|----------|------|
| 문항 직접 추가 모달 | **수정 후 개선** | max-w-4xl 모달 + 유형 선택 grid-cols-2 → 모바일 stack 처리 |
| 문제모음에서 추가 모달 | **수정 후 개선** | w-[240px] 사이드바 → 모바일 상단 collapsed 가로 스크롤 |
| 복수 문제모음 랜덤 출제 | **수정 후 개선** | 미리보기 테이블 가로 스크롤, 난이도별 배점 stack 처리 |
| 가져오기/내보내기 위저드 | **수정 후 개선** | BankWizardShared의 사이드바 모바일 분기 처리 |

## 3. 정책/예외 케이스 — 모바일 관점 추가 점검

### 3-1. 응시 자동저장 + 포커스 이탈 감지

- 모바일 브라우저는 백그라운드 전환 빈도 ↑ → `focusLossCount` 가 데스크톱 대비 비정상적으로 높을 수 있음
- **권고**: 모바일에서는 포커스 이탈 임계치를 조정하거나, 자동저장 주기를 짧게 하는 정책 검토 필요 (현 기획 미정)

### 3-2. 마감 임박 시 모바일 진입 + 액세스 코드/IP 제한

- IP 제한이 걸린 퀴즈는 모바일 데이터(LTE/5G)에서 차단될 수 있음 → 학생 안내 부족 가능성
- **권고**: 액세스 코드/IP 제한 활성 퀴즈는 응시 시작 전 안내 메시지 강화 (별도 이슈)

### 3-3. 조건부 재응시 모달 모바일 사용 가능성

- 교수자가 모바일에서 "미응시자 일괄 재응시 부여" 시 학생 50명 리스트를 카드형으로 보게 됨 → 스크롤 길어짐
- 현 수정안: 모바일에서는 학생 카드를 단일행 압축 표시 + max-h-80 overflow-y-auto 유지
- **추가 권고(후순위)**: 모바일에서는 "전체 선택 후 일부만 제외" UI 외에 "필터+일괄 부여" 단축 UI 검토

## 4. MVP 범위 vs Out-of-scope

| 기능 | 모바일 우선순위 | 비고 |
|------|---------------|------|
| **Must-have**: 응시, 결과 확인, 채점 모니터링 | 1순위 | 이번 수정으로 모두 양호 |
| **Should-have**: 모달 기반 문항 편집, 문제모음 추가 | 2순위 | 이번 수정으로 사용 가능 수준 도달 |
| **Nice-to-have**: 모바일 전용 통계 차트, 포커스 이탈 정책 | 후순위 | MVP 이후 |
| **Out-of-scope**: 모바일에서 문항 60+ 일괄 import, 복잡한 채점 작성 | 제외 | 데스크톱 권장 |
