## 2. Acceptance Criteria (Jira 형식)

### AC-1: 학습자 모바일 응시 (Must-have)
```
Given: 학습자가 모바일(375px 이하) 브라우저로 응시 페이지에 진입
When: 연결(matching) 또는 수치형(numerical) 문항을 본다
Then: 입력 필드가 stack 형태로 배치되어 가로 잘림 없음
And: 모든 라디오/체크박스/입력의 터치 영역이 ≥ 36px
```

### AC-2: 모바일 모달 너비 (Must-have)
```
Given: 모바일(375px) 환경
When: AddQuestionModal / QuestionBankModal / RandomQuestionBankModal /
      ExportBankModal / ImportBankModal / ConditionalRetakeModal 중 하나 열기
Then: 모달이 화면 좌우 ≤ 12px 여백으로 표시되고
And: 본문 내 모든 가로 정보가 화면 내에 보이거나 명시적 가로 스크롤로 표시됨
And: max-h-[90vh] 이내로 세로 overflow 처리
```

### AC-3: 조건부 재응시 학생 카드 (Must-have)
```
Given: 교수자 모바일에서 채점 대시보드 → 조건부 재응시 부여 → Step 2 진입
When: 대상 학생 N명이 표시됨
Then: 각 학생이 모바일에서 단일 행 카드(이름/학번/학과/사유)로 압축 표시되고
And: 데스크톱(sm:640px+)에서는 5컬럼 그리드 표시 (회귀 없음)
And: 학생 카드 탭으로 제외/포함 토글 가능
```

### AC-4: 채점 통계/활동 로그 (Should-have)
```
Given: 모바일에서 채점 대시보드 → 통계 탭 또는 학생 활동 로그 진입
When: 요약 카드(4~5개)가 표시됨
Then: 모바일에서는 grid-cols-2로 2열 stack 표시
And: 태블릿(sm:)에서는 grid-cols-3 또는 grid-cols-4
And: 데스크톱(lg:) 에서는 원래 컬럼 수 유지
```

## 3. 정책 결정 사항 (확정)

| 정책 | 확정 사항 |
|------|----------|
| **모바일 우선순위 사용자** | 학습자(응시) > 교수자(모니터링) > 교수자(편집/채점 작성) |
| **모바일 최소 지원 너비** | 360px (Galaxy S 기본) |
| **터치 타겟 최소 크기** | 36px (이상적 44px, 일부 컴포넌트 예외 허용) |
| **모달 모바일 너비 패턴** | `max-w-* w-[calc(100vw-24px)] sm:w-auto max-h-[90vh]` 페어 |
| **사이드바 모달 모바일 처리** | 상단 collapsed 영역(max-h-[180px]) + 가로 스크롤 + 검색 메인 진입 |

## 4. 정책 결정 보류 사항 (PM6 안건 후보)

| 안건 | 쟁점 |
|------|------|
| **모바일 포커스 이탈 자동 제출 임계치** | 데스크톱과 동일(현재) vs 모바일 별도 임계치(예: 3회→5회) — 학생 항의 vs 부정 응시 방지 균형 |
| **모바일 응시 IP 제한** | LTE/5G 환경에서 IP 화이트리스트 적용 시 모바일 응시 차단 — 정책 안내 강화 또는 제외 옵션 |
| **모바일 일괄 작업(50명+ 재응시) UI** | 카드형 vs 필터+일괄 부여 vs 데스크톱 권장 안내 |

> **권고**: 위 3개는 PM6 별도 안건 등록 검토 (사용자 승인 시)

## 5. KPI 지표

### [현재 측정 가능 — 프로토타입/목업 단계]

| 지표 | 측정 방법 | 목표 |
|------|----------|------|
| 모바일 반응형 회귀 이슈 수 | 코드 정적 분석 + 시각 회귀 | 0건 (Critical/High) |
| 모바일 모달 가로 스크롤 발생 페이지 수 | DevTools 수동 점검 | 0건 |
| Tailwind 반응형 prefix 일관성 | grep으로 grid-cols 미분기 사용 검출 | 0건 |

### [실데이터 연동 후 측정 가능]

| 지표 | 측정 방법 | 목표 |
|------|----------|------|
| 모바일 응시 비율 | useragent 기반 응시 로그 분석 | ≥ 30% (가설) |
| 모바일 응시 자동 제출 발생률 | focus loss 로그 + 자동제출 로그 | ≤ 2% |
| 모바일 응시 평균 풀이 시간 vs 데스크톱 | attempts.timeTaken 비교 | 데스크톱 대비 ±20% 이내 |
| 모바일 응시 환경 에러 발생률 | 클라이언트 에러 로그 | ≤ 1% |
| 채점 대시보드 모바일 진입율 | 페이지뷰 + useragent | ≥ 15% (교수자 보조 사용) |
