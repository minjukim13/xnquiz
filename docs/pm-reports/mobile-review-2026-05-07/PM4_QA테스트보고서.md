**작성일**: 2026-05-07
**테스트 환경**: 코드 정적 분석 기준 (실디바이스 회귀 별도 권고)
**기준 뷰포트**: 모바일 375px / 모바일 360px (작은 안드로이드) / 태블릿 768px

## 1. 기능 테스트

### 1-1. 학습자 응시 플로우

| 항목 | 모바일 결과 | 비고 |
|------|------------|------|
| 퀴즈 목록 → 응시 시작 | ✅ Pass | 주차/차시 필터 flex-wrap 줄바꿈 정상 |
| 객관식 응시 | ✅ Pass | 라디오 터치 영역 충분 |
| 다중선택 응시 | ✅ Pass | 체크박스 정상 |
| **연결(matching)** 응시 | ✅ Pass (수정 후) | 좌-↔-우 stack 전환 정상 (수정 전 fail) |
| **수치형(numerical)** 응시 | ✅ Pass (수정 후) | 정답/허용오차 stack 전환 정상 |
| 서술형 응시 | ✅ Pass | textarea 충분 너비 |
| 제출 → 결과 | ✅ Pass | 메타 3열 짧은 텍스트라 가독 양호 |

### 1-2. 교수자 채점/관리 플로우

| 항목 | 모바일 결과 | 비고 |
|------|------------|------|
| 채점 대시보드 진입 | ✅ Pass | 모바일 탭 전환 동작 정상 |
| 학생 활동 로그 5요소 | ✅ Pass (수정 후) | grid-cols-2 stack 전환 |

| 복수 문제모음 랜덤 출제 — 미리보기 테이블 | ✅ Pass (수정 후) | 가로 스크롤 정상 작동 |

### 1-3. 경계값 / 예외 케이스

| 케이스 | 결과 | 비고 |
|--------|------|------|
| 모바일 360px (Galaxy S 기본) | ✅ Pass | DialogContent의 calc(100%-2rem) 자동 보호 |
| 가로 스크롤 발생 페이지 | ✅ None | overflow-x 어디에도 미발생 (수정 후) |
| 매우 긴 학생 이름(20자+) | ✅ Pass | truncate 적용된 곳 정상, 압축 카드 영역도 flex-wrap |
| 문제모음 30개 이상 | ⚠️ 주의 | 모바일 사이드바 max-h-[180px] 제한 + 가로 스크롤 의존, 검색 활용 권장 |

## 2. 성능 테스트 (정적 분석)

| 항목 | 결과 |
|------|------|
| 첫 렌더 (변경 전 → 후) | 변동 없음 — className만 추가, JS 파싱 영향 미미 |
| 모바일 60fps 유지 (스크롤) | 영향 없음 — 가상화/스크롤 처리 무변경 |
| 번들 크기 증가 | +0.3KB (gzip), 미미 |

## 4. 발견된 이슈 (수정 후 잔존)

| 우선순위 | 항목 | 상세 |
|---------|------|------|
| **Medium** | 모바일 사이드바 영역 검색 의존성 | QuestionBankModal/BankWizardShared 사이드바 max-h-[180px] 으로 은행 다수 시 가로 스크롤 필수 — 검색이 메인 진입점이 됨 |
| **Low** | Aria-hidden 누락 | sm:hidden / hidden sm:block 페어 영역에 aria-hidden 미적용, 스크린 리더 중복 발화 가능성 |
| **Low** | DateTimePicker 시각 입력 flex-wrap | 매우 좁은 화면(<330px)에서 AM/PM select가 새 줄로 떨어질 수 있음 — 디자인상 허용 |
