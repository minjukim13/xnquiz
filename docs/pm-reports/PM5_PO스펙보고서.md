## 1. 제품 방향성 요약

XN Quizzes는 Canvas LMS 기반의 교수자-학습자 퀴즈/시험 관리 시스템이다. 현재 프로토타입은 문제은행 관리, 퀴즈 채점, 통계 출력이라는 핵심 교수자 워크플로우를 구현한 상태다.

**핵심 가치 명제**: 교수자가 문항 자산을 쌓고, 퀴즈를 구성하고, 채점하고, 결과를 분석하는 전 과정을 Canvas 외부 도구 없이 완결할 수 있게 한다.

**우선순위 판단 기준**:
- 교수자 워크플로우 단절 여부 (채점 완료까지의 흐름)
- 고객사 인제대 특정 요구사항 (주차별 보기, 2-4) 포함 여부
- 개발 Effort 대비 Impact 비율

## 3. Acceptance Criteria (핵심 기능)

### AC-01. 문항 난이도/그룹 메타데이터 (1-1)

**Given** 교수자가 문항을 생성하거나 편집할 때
**When** 난이도(상/중/하)와 그룹명을 입력하고 저장하면
**Then**
- [ ] 문항 목록에 난이도 배지가 표시된다
- [ ] 문항 목록에서 난이도/그룹 기준 필터링이 동작한다
- [ ] 엑셀 업로드 시 difficulty, group 컬럼을 인식하여 저장한다
- [ ] 기존 문항(메타데이터 미입력)은 난이도 "미설정"으로 표시된다

**예외**: 난이도 미입력 문항도 랜덤 출제 풀에 포함 가능 (단, 난이도 필터 적용 시 제외)

---

### AC-02. 복수 은행/그룹 선택 랜덤 출제 (2-1)

**Given** 교수자가 퀴즈 생성 화면에서 랜덤 출제를 선택할 때
**When** 복수의 문제은행 및 그룹을 선택하고 그룹별 출제 수와 배점을 입력하면
**Then**
- [ ] 선택한 은행/그룹에서 지정 수만큼 문항이 무작위 추출된다
- [ ] 난이도별 배점 설정 시 해당 배점이 출제 문항에 적용된다
- [ ] 출제 풀에 충분한 문항이 없으면 "문항 부족" 경고를 표시한다
- [ ] 추출된 문항 목록을 미리보기로 확인할 수 있다
- [ ] 재추출(셔플) 버튼으로 다른 조합을 확인할 수 있다

**예외**: 선택한 그룹의 문항 수가 출제 수보다 적을 경우 → 최대 출제 가능 수를 안내하고 저장 차단

---

### AC-03. 엑셀 업로드 난이도/그룹 컬럼 추가 (1-3)

**Given** 교수자가 문항 업로드 템플릿을 다운로드할 때
**When** difficulty, group 컬럼이 추가된 템플릿을 작성하여 업로드하면
**Then**
- [ ] 템플릿에 difficulty(상/중/하), group(문자열) 컬럼이 포함된다
- [ ] difficulty 값이 상/중/하 외의 값이면 행 단위 오류 메시지를 표시한다
- [ ] difficulty 미입력 시 "미설정"으로 저장하고 업로드는 계속 진행된다
- [ ] 업로드 결과 요약에 "난이도 미설정 N건" 안내를 포함한다

---

### AC-04. 부분 점수 정책 설정 (4-1)

**Given** 교수자가 퀴즈를 생성하거나 문항을 편집할 때
**When** 복수선택형(multiple_answers) 문항에서 부분 점수 정책을 설정하면
**Then**
- [ ] 정책 옵션: "전체 정답 시에만 만점" / "정답 선택 비율만큼 부분 점수"
- [ ] 부분 점수 선택 시: (정답 선택 수 / 전체 정답 수) x 배점으로 계산된다
- [ ] 오답 선택 시 감점 여부를 추가 옵션으로 설정할 수 있다 (Nice-to-have)
- [ ] 채점 결과 화면에서 부분 점수 적용 기준이 표시된다

**예외**: 부분 점수 정책은 multiple_answers 유형에만 표시. 기타 유형에는 미노출.

---

### AC-05. 단답형 정답 판정 옵션 (1-6)

**Given** 교수자가 단답형(short_answer) 문항을 설정할 때
**When** 정답 판정 옵션을 선택하면
**Then**
- [ ] 옵션 A: 대소문자 구분 (기본값: 미구분)
- [ ] 옵션 B: 알파벳 전체 일치 필요 여부 (기본값: 트림 후 비교)
- [ ] 교수자가 복수 정답(허용 답안 목록)을 콤마 구분으로 입력할 수 있다
- [ ] 채점 결과에서 판정 기준이 툴팁으로 표시된다

**현재 코드 현황**: `autoGradeAnswer()`에서 `toLowerCase()` 일괄 적용 중 → 대소문자 구분 옵션 추가 시 해당 함수 수정 필요

---

### AC-06. 주차별 퀴즈 목록 (2-4, 인제대 특화)

**Given** 교수자가 퀴즈 목록 화면에 진입할 때
**When** "주차별 보기" 탭 또는 토글을 선택하면
**Then**
- [ ] 퀴즈 목록이 week 필드 기준으로 그룹화되어 표시된다
- [ ] 각 주차 내에서 session 순으로 정렬된다
- [ ] week 미설정 퀴즈는 "주차 미설정" 섹션에 분리 표시된다
- [ ] 주차별 보기에서도 퀴즈 상태(초안/게시/마감) 필터가 동작한다

**현재 코드 현황**: mockData에 `week`, `session` 필드 존재. 구현 Effort 낮음.

---

### AC-07. 시험 복제/이관 (2-3)

**Given** 교수자가 퀴즈 목록 또는 퀴즈 상세 화면에서 복제를 실행할 때
**When** 복제 대상 과목과 제목을 입력하고 확인하면
**Then**
- [ ] 원본 퀴즈의 설정(시간제한, 배점, 응시 횟수, 점수 정책)이 복제된다
- [ ] 원본 퀴즈의 문항 구성(문제은행 참조 또는 직접 문항)이 복제된다
- [ ] 복제된 퀴즈의 상태는 "초안"으로 초기화된다
- [ ] 시작일/마감일은 복제 시 비워진 상태로 생성된다
- [ ] 복제 완료 후 새 퀴즈의 편집 화면으로 이동한다

## 4. 정책 결정 사항 및 예외 처리 룰

### P-01. 복수 응시 점수 정책

| **정책명** | **동작 규칙** | **현재 구현 상태** |
|-----------|-------------|----------------|
| 최고 점수 유지 | 전체 응시 중 최고 점수를 최종 점수로 채택 | 구현 완료 |
| 평균 점수 | 전체 응시 점수의 평균을 최종 점수로 채택 | 구현 완료 |
| 최신 점수 유지 | 마지막 응시 점수를 최종 점수로 채택 | 구현 완료 |

**미결 정책**: 채점 완료 후 학생이 재응시 요청 시 처리 기준 (→ 미결 사항 P-01 참조)

### P-02. 지각 제출 정책

**현재 상태**: 마감 시간(dueDate) 초과 여부 검증 로직 미구현

**확정 룰**:
- 마감 시간 초과 제출은 시스템에서 차단 (기본값)
- 교수자가 퀴즈별로 "지각 제출 허용" 옵션을 설정 가능 (Nice-to-have)
- 지각 제출 허용 시 감점 비율(%) 입력 가능 (Nice-to-have)

### P-03. 자동채점 단답형 판정 기준 (현재 정책 불투명)

**현재 코드**: `autoGradeAnswer()`에서 `answer.trim().toLowerCase() === c.toLowerCase().trim()` 완전 일치 비교
**확정 룰**:
- 기본 정책: 대소문자 무시 + 앞뒤 공백 제거 후 완전 일치
- 교수자가 문항별로 대소문자 구분 옵션 변경 가능 (AC-05 구현 후)
- 복수 정답 허용: 교수자가 허용 답안 목록 콤마 구분 입력

### P-04. 채점 완료 기준

**확정 룰**:
- 자동채점 문항: 제출 즉시 채점 완료
- 수동채점 문항(essay, short_answer): 교수자가 해당 문항에 점수를 입력해야 채점 완료
- 퀴즈 전체 채점 완료 조건: 모든 응시 학생의 모든 수동채점 문항에 점수 입력 완료
- 일부 채점 완료 상태: `grading` 상태로 표시 (현재 구현 일치)

### P-05. 문항 변경 시 기출제 문항 반영 정책 (1-5)

**문제**: 문제은행 문항을 수정했을 때 이미 출제된 퀴즈의 응시 화면에 반영할지 여부

**확정 룰 (권고)**:
- 응시 중인 퀴즈: 문항 변경 즉시 반영 (진행 중 응시자 혼란 방지 불가)
- 초안 상태 퀴즈: 즉시 반영
- 마감된 퀴즈: 반영하지 않음 (이미 제출 완료)
- **단, 정답 변경 시**: 교수자에게 "재채점 여부" 확인 모달 노출 필요

**미결**: 게시 중(open) 퀴즈의 정답 변경 시 처리 기준 (→ 미결 사항 P-02 참조)

### P-06. 퀴즈 상태 전이 규칙

```
초안(draft) → 게시(open): 교수자가 명시적으로 게시 실행
게시(open) → 마감(closed): dueDate 도래 시 자동 전환 (현재 미구현, 수동 처리)
마감(closed) → 채점 중(grading): 수동채점 문항 존재 시 자동 진입
채점 중(grading) → 마감(closed): 전체 채점 완료 시 자동 전환
```

**미결**: 게시 취소(open → draft) 허용 여부, 조건 (→ 미결 사항 P-03 참조)

| 평균 문항분석 다운로드 소요 시간 | downloadItemAnalysisXlsx() 실행~완료 | < 2초 | 브라우저 Performance API |

### [실데이터 연동 후 측정 가능] API/DB 연동 이후에만 수집 가능

| **지표명** | **정의** | **목표값(참고)** | **비고** |
|-----------|---------|--------------|--------|
| 실제 응시율 | 실제 응시 학생 / 등록 학생 x 100 | > 90% | 학생 인증 + 응시 이력 DB 필요 |
| 재응시율 | allowAttempts > 1 퀴즈에서 2회 이상 응시 학생 비율 | 모니터링 지표 | 응시 이력 로그 필요 |
| 평균 풀이 시간 | 제출 시각 - 시작 시각 (문항별, 전체) | 모니터링 지표 | 서버 타임스탬프 필요 |
| 교수자 채점 소요 시간 | 채점 시작 ~ 완료까지 경과 시간 | < 48시간 | 채점 이벤트 로그 필요 |
| 문항 오답률 상위 문항 | correctRate 하위 20% 문항 목록 | 모니터링 지표 | 실 답안 데이터 필요 |
| 점수 분포 왜도/첨도 | 실제 학생 점수 분포의 통계적 형태 | 정규분포 근사 | 실 점수 데이터 필요 |
| 문항 변별도 실측값 | 상위 27% vs 하위 27% 평균 점수 차이 | D >= 0.3 | 실 점수 데이터 필요 |
| Cronbach 알파 실측값 | 문항 내적 일관성 신뢰도 | >= 0.7 | 실 점수 데이터 필요 |
| 세션 이탈률 | 퀴즈 시작 후 미제출 학생 비율 | < 5% | 세션/이벤트 로그 필요 |
| IP 중복 접속 발생률 | 동일 IP에서 2회 이상 응시 시도 | 0% (목표) | 서버 IP 로그 필요 |

## 8. MVP 로드맵 요약

### MVP1 (현재 완료)
- 문제은행 CRUD, 문항 CRUD
- 은행 간 문항 복사
- 검색/필터링
- 채점 대시보드 (자동 + 수동 채점)
- 퀴즈 통계 (recharts)
- 엑셀 다운로드 4종 (성적, 문항분석, 답안지, 채점양식)

### MVP2 (다음 스프린트 권고)
우선순위 순:
1. 문항 난이도/그룹 메타데이터 (1-1) — 2-1의 선행 조건
2. 엑셀 업로드 컬럼 확장 (1-3) — 1-1 연동
3. 복수 은행/그룹 랜덤 출제 (2-1) — 핵심 요구사항
4. 단답형 정답 판정 옵션 (1-6) — 현재 정책 불투명 해소
5. 주차별 퀴즈 목록 (2-4) — 낮은 Effort, 인제대 대응
6. 부분 점수 정책 (4-1) — 채점 정확도
7. 시험 복제/이관 (2-3) — 반복 생성 비용 절감

### MVP3 (후속 검토)
- PDF 출력 (시험지, 답안지)
- 멀티미디어 첨부 (이미지, 음향)
- 지각 제출 허용 옵션
- 응시 IP 로그 (서버 구축 후)
- TrustLock 연동 (계약 후)
