# XN Quiz UX 개선 포인트 — Canvas LMS 대비 비교

> **목적**: XN Quiz 프로토타입이 기존 Canvas LMS 퀴즈 기능 대비 어떤 UI/UX 관점에서 개선되었는지 정리합니다.
> **작성일**: 2026-04-24
> **비교 기준**: Canvas LMS Classic Quizzes 및 New Quizzes 기준 동작
> **대상 독자**: 내부 기획, 디자인, 개발 팀

---

## 0. 한 줄 요약

Canvas가 전세계 범용 LMS로서 **옵션 최대화**와 **표/엑셀 기반 수치 제공**을 택했다면, XN Quiz는 **한국 대학 교수자가 가장 자주 쓰는 플로우**에 집중하고, 통계 수치를 **그래프 중심으로 시각화**하여 결정 피로와 해석 부담을 동시에 낮추는 방향을 택했습니다.

---

## 1. 정보 구조 및 네비게이션

| **영역** | **Canvas (기존)** | **XN Quiz (개선)** |
| --- | --- | --- |
| 퀴즈 버전 | Classic Quizzes와 New Quizzes 병존으로 교수자가 어떤 빌더를 써야 할지 혼란 | 단일 UX, 버전 선택 개념이 없음 |

| 역할 전환 | 교수자/학생 뷰 전환이 별도 진입(Student View) | `RoleToggle` 한 번 클릭으로 교수자 학생 뷰 즉시 전환 |

## 2. 퀴즈 생성 및 편집

**Canvas**
- 문항 추가 UI가 Rich Content Editor 기반이라 단순 객관식/단답형에도 에디터가 과하게 노출됩니다.
- 문항 유형 드롭다운의 깊이가 깊어 선택까지 2단계 이상 걸립니다.
- 저장(Save)과 게시(Publish) 상태 구분이 모호하고, 게시 후 수정 시 재게시 흐름이 불명확합니다.

**XN Quiz**
- 문항 유형별 **배지 색상 시스템**을 적용해 목록에서 유형을 즉시 판별할 수 있습니다.
- 스텝 모달의 푸터 레이아웃을 이전(좌측), 취소+다음(우측)으로 통일해 조작 패턴이 예측 가능합니다.
- 문제은행에서 **가져오기** 동선을 생성 흐름과 통합해 별도 탭 이동이 필요 없습니다.
- 저장과 게시 동작을 명확히 분리하고, 게시 중 편집 시 회귀 영향 범위를 명시합니다.

---

## 3. 상태 시각화

**Canvas**
- 게시/미게시(Published/Unpublished) 이분법만 존재합니다.
- 마감 여부는 별도 안내 문구로만 노출되어, 목록에서 한눈에 파악하기 어렵습니다.

**XN Quiz**
- **초안 / 진행중 / 마감** 3단계 상태 배지를 제공합니다.
- 마감일이 경과하면 자동으로 "마감" 배지로 전환되며, 교수자가 수동 마감할 필요가 없습니다.
- 로딩/진행 표현에 말줄임표 대신 "진행중"과 같은 확정형 레이블을 사용합니다.

## 4. 채점 및 통계 시각화 (핵심 차별점)

Canvas는 "수치는 제공하되 해석은 교수자 몫"이라는 철학에 가깝습니다. XN Quiz는 **"수치를 보는 순간 문제가 있는 문항과 학생이 보이게"** 설계했습니다.

### 4-1. 채점 화면 구조

| **영역** | **Canvas** | **XN Quiz** |
| --- | --- | --- |
| 채점 도구 | SpeedGrader가 별도 화면으로 분리되어 전체 현황과 개별 채점 동선이 단절됨 | 채점 대시보드 단일 화면에서 수강생 목록, 문항별 분포, 재채점까지 처리 |
| 집계 방식 | 제출자 기준 집계가 기본 | 전체 수강생 분모 기반, 마감 후 미시작자 자동 0점 처리로 운영자 관점 일치 |
| 재채점 | 문항 단위 재채점 시 학생별 반복 조작 필요 | 정책 변경/오답 처리 시 일괄 재채점 트리거 제공 |

### 4-2. 통계 제공 방식: 표/엑셀 vs 그래프 시각화

**Canvas의 Quiz Statistics**
- 평균, 최고점, 최저점, 표준편차를 숫자로만 상단에 나열합니다.
- 점수 분포는 단순 높이 바 차트로 제공되나, 평균선이나 해석 가이드가 없습니다.
- 문항별 정답률은 수치와 단색 가로 바로 표시되며, 난이도에 대한 색상 코딩이 없습니다.
- 상세 분석은 **CSV/엑셀 다운로드 후 직접 피벗** 방식이 전제입니다. Item Analysis 리포트도 텍스트 수치 기반입니다.

**XN Quiz의 통계 화면 (실제 구현 기준)**

**(1) 점수 분포 차트**
- 점수 구간별 인원을 BarChart로 표시합니다.
- **평균점에 점선 ReferenceLine과 라벨을 표시**하여 "우리 반 평균이 어디인지"를 즉시 인지할 수 있게 했습니다.
- 평균점에 해당하는 막대를 primary 색상으로 강조(그 외 구간은 연한 accent 색상)하여 분포 중심이 시각적으로 드러납니다.
- "응시 25명 중 채점 완료 20명 기준"처럼 **분모 기준 설명을 차트 상단에 병기**하여 수치 오해를 방지합니다.

**(2) 응시 현황 패널**
- 수강 인원, 응시 완료, 미제출, 채점 완료, 채점 대기를 각각 **비율 기반 progress bar**로 표현합니다.
- 인원수와 비율(%)을 동시 표기하여 "미제출 5명(11.1%)"처럼 규모감 파악이 즉시 가능합니다.
- Canvas는 "Submitted: 40/45"처럼 분수로만 제공합니다.

**(3) 점수 분포 구간 분할(상위/중위/하위 27%)**
- 자동으로 상위 27%, 중위 46%, 하위 27% 구간 경계 점수를 계산해 표시합니다.
- 교수자는 별도 계산 없이 "하위 27%는 몇 점 미만인가"를 확인할 수 있습니다.
- Canvas는 해당 구간 통계를 별도 Item Analysis 리포트에서 수치로만 제공합니다.

**(4) 문항별 득점률 차트**
- **가로 BarChart + 난이도 색상 코딩**을 적용했습니다.
  - 70% 이상: 녹색 (쉬움)
  - 40~69%: 주황 (보통)
  - 40% 미만: 빨강 (어려움)
  - 채점 전: 회색
- **70%와 40% 기준선을 점선 ReferenceLine**으로 표시하여, 각 문항이 어느 난이도 구간에 속하는지 한눈에 보입니다.
- Canvas는 문항별 정답률을 단색 가로 바와 숫자로만 제공하며, 난이도 판별은 교수자 몫입니다.

**(5) 문항별 상세 통계 테이블**
- 문항번호, 유형(배지), 배점, 평균 점수, 득점률(inline bar + 수치), 난이도(배지), 채점 현황을 한 행에 통합했습니다.
- 난이도 배지는 쉬움/보통/어려움 색상 구분으로 표시되어 개선 필요 문항을 즉시 필터링할 수 있습니다.
- **xlsx 다운로드 버튼을 표 헤더에 병설**하여, 시각 분석 후 필요 시 엑셀 반출도 가능합니다(Canvas는 엑셀이 기본, XN Quiz는 시각화가 기본).

**(6) 개별 학생 상세 화면 (StatsTab)**
- 학생별 응답 스크린에서 문항별 점수 분포 히스토그램과 정답률 progress bar를 제공합니다.
- "이 학생이 받은 점수가 전체에서 어느 위치인지"를 그래프로 즉시 비교할 수 있습니다.

### 4-3. 통계 시각화 비교 요약

| **통계 영역** | **Canvas 제공 방식** | **XN Quiz 제공 방식** |
| --- | --- | --- |
| 평균/최고/최저 | 숫자 텍스트 | 숫자 + 평균선이 차트에 반영 |
| 점수 분포 | 단순 막대 (평균선 없음) | 막대 + 평균 ReferenceLine + 강조 색상 |
| 상/중/하위 구간 | Item Analysis 리포트에서 숫자로 | 화면 내 자동 구간 분할 + 경계 점수 즉시 노출 |
| 응시 현황 | "Submitted: 40/45" 분수 | 항목별 progress bar + 비율% |
| 문항별 정답률 | 단색 바 + 수치 | 색상 코딩(쉬움/보통/어려움) + 기준선 점선 |
| 난이도 판별 | 교수자 직접 해석 | 자동 난이도 배지 부여 |
| 엑셀 반출 | 기본 전제 | 시각화가 기본, 필요 시 xlsx 버튼 제공 |

## 6. UX 라이팅

**Canvas**
- 한국어 UI가 기계 번역 기반이라 "이 퀴즈를 다시 시도하도록 허용"과 같이 의미 파악에 시간이 걸리는 문구가 다수 존재합니다.
- 오류 메시지가 기술 용어 그대로 노출되는 경우가 있습니다.

**XN Quiz**
- 한국 대학의 교수자 및 학습자 어휘에 맞춘 자연스러운 한글 라이팅을 적용했습니다.
- 버튼, 안내 문구, 에러 메시지의 톤을 일관되게 통제합니다.
- 마감일 임박, 채점 완료, 점수 공개 등 **상태 전환 시점의 안내 문구를 맥락 기반**으로 정리했습니다.

## 8. 의사결정 피로 감소

Canvas의 Quiz Options에는 한국 대학 운영에서 거의 사용되지 않는 항목(One Question at a Time, Access Code, IP Filter 등)이 포함되어 있어 설정 화면이 길어집니다. XN Quiz는 한국 대학 교수자가 실제로 결정하는 **4가지 핵심 정책(재응시, 지각 제출, 공개 시점, 채점 방식)만 노출**하여 설정 화면의 인지 부하를 낮췄습니다.

## 10. 참고 사항 및 한계

- 본 비교는 주로 Canvas Classic Quizzes 기준이며, Canvas New Quizzes 최신판에서는 일부 UI가 개선된 영역이 있습니다. 외부 제안 시 비교 기준을 명시할 필요가 있습니다.
- LearningX가 Canvas 위에 얹힌 래퍼라는 점을 감안하면, 내부 설득 시 "Canvas 대비"와 "LearningX 현재 퀴즈 UX 대비" 중 주 비교축 선정이 필요합니다.
- 정량 지표(채점 시간 단축 수치, 통계 해석 시간 단축 수치, 설정 완료 시간 등)는 사용성 테스트 이후 보강 예정입니다.
