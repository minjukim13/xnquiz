## 4. 이슈 목록

> 에이전트 1차 진단에서 Critical(QTI 파일 크기)/High(자동 폴링)로 분류된 2건은, 현 단계가 mock 프로토타입(교수자 본인이 자기 파일 업로드, 실시간 데이터 소스 없음)임을 고려해 심각도를 재조정함.

### M-01 [Medium] 재응시 기한 미설정 + 마감 경과 시 부여 무효화 가드 부재

- **위치**: 응시 모니터링 화면 > 학생 행 > 응시 기회 추가 부여 모달 (GrantAttemptModal)
- **파일**: QuizMonitor.jsx L348-414 (모달), L145-160 (handleGrant)
- **현황(정정)**: 모달에 "재응시 기한"(retakeDeadline) 입력이 이미 존재하며, "미설정 시 퀴즈 마감일 적용" 안내가 있음 (L396-402). 즉 마감 후 부여 시 기간 연장을 동반하는 메커니즘이 부분 구현되어 있음. 초판의 "마감 검증 전면 부재" 표현은 부정확하여 정정함.
- **남은 문제**: 재응시 기한을 미설정으로 둘 경우 이미 지난 퀴즈 마감일이 그대로 적용되어, 부여해도 학생이 응시할 수 없는 모순이 발생. ("기한 미설정 + 마감 경과" 조합 한정)
- **정책 판단**: 신규 정책 결정 불필요. 현 구현이 "기간 연장 동반" 방향으로 사실상 확정되어 있음.
- **권고**: 코드 가드 추가. 마감 경과 상태에서 기한 미설정 시 경고 노출 또는 재응시 기한 입력 필수화. 제출 완료 학생 부여 후 학생 측 재진입 동작은 별도 확인 권장.

### M-02 [Medium] QTI 업로드 파일 크기 제한 부재

- **파일**: QuizList.jsx L1435-1440
- **현상**: `<input accept=".zip,.imscc,.xml">` 에 크기 제한 없음. 대용량/압축 폭탄(ZipBomb) 방어 없음.
- **영향**: 현재는 교수자 본인이 자기 내보내기 파일을 올리는 구조라 위협은 낮으나, 실데이터/외부 사용자 연동 시 클라이언트 메모리 과다 점유 위험.
- **권고**: 업로드 시 파일 크기 상한(예: 50MB) 검증 후 초과 시 에러 메시지. 실데이터 전환 시 High로 격상.

### L-01 [Low] 검색 입력 aria-label 누락

- **파일**: QuizList.jsx L185-197 (SearchInput)
- **현상**: input에 placeholder만 있고 aria-label 없음. 스크린리더 용도 인식 저하.
- **권고**: `aria-label="퀴즈 제목 검색"` 추가. (접근성 WCAG)

### L-02 [Low] QTI 파싱 실패 메시지가 일반적

- **파일**: qti.js L508
- **현상**: 가져올 퀴즈를 못 찾으면 일반 메시지만 표시되어 원인(형식 불일치/빈 파일/미지원 버전) 구분 불가.
- **권고**: 실패 사유별 구체적 메시지 분기.

### L-03 [Low] 재채점 로그 localStorage 용량 관리 부재

- **파일**: QuestionDetailPanel.jsx L66-71 (`xnq_regrade_log`)
- **현상**: 로그가 무한 누적, 용량 초과 시 저장 실패/데이터 손실 가능.
- **권고**: 최근 N건만 유지하는 롤링 또는 용량 가드. (mock 전용이므로 우선순위 낮음)

### L-04 [Low] 빌드 경고: mockData 정적/동적 import 혼용

- **파일**: src/data/mockData.js
- **현상**: `students.js`는 동적 import, 다수 컴포넌트는 정적 import → 청크 분리 무효(INEFFECTIVE_DYNAMIC_IMPORT).
- **권고**: import 방식 일원화. 동작 영향은 없으나 청크 최적화 저해.

## 6. 결론

최근 피드백 해결분은 기능 구현 완성도가 높고, 빌드/Lint 무오류, XSS 및 권한 가드 등 핵심 보안도 양호하다. 즉시 배포를 막을 Critical/High 결함은 발견되지 않았다. 다만 실데이터/API 연동 전환 전에 M-01(마감 경과 시 재응시 기한 가드)과 M-02(파일 크기 제한)는 보완이 필요하며, Low 4건은 품질 개선 차원에서 백로그 반영을 권고한다.
