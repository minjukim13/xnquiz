# XN Quiz 생성 스토리보드 빌더 (Figma Plugin)

퀴즈 생성 화면 스토리보드 22장의 슬라이드 레이아웃과 캡쳐 이미지 배치를 Figma 파일에 자동으로 처리하는 플러그인입니다. 두 개의 명령으로 구성되어 있습니다.

| **명령** | **기능** |
|---|---|
| 1. 스토리보드 골격 생성 | 22 슬라이드 프레임을 격자로 배치 + 텍스트 전용 5장은 본문 자동 채움 |
| 2. 캡쳐 이미지 일괄 배치 | PNG 파일 18장을 한 번에 받아 해당 placeholder 위치에 자동 배치 |

## 사용 방법

### 1단계 - 플러그인 임포트 (최초 1회만)

1. Figma 데스크톱 앱을 실행하고 새 Design 파일을 엽니다.
2. 메뉴: `Plugins → Development → Import plugin from manifest...`
3. 본 폴더의 `manifest.json` 을 선택합니다.

### 2단계 - 골격 생성

1. `Plugins → Development → "XN Quiz 생성 스토리보드 빌더" → "1. 스토리보드 골격 생성 (22 슬라이드)"`
2. 22장의 슬라이드 프레임이 4열 격자로 배치됩니다.
3. 텍스트 전용 5장 (C-01 커버, C-02 목차, C-03 플로우, M-01 매핑표, E-01 검증오류 + M-02 메모) 은 본문이 자동으로 채워져 있습니다.

### 3단계 - 캡쳐 일괄 배치

선행 조건: 프로젝트 루트에서 `npm run capture-storyboard` 실행하여 PNG 18장을 미리 생성해두세요.

1. `Plugins → Development → "XN Quiz 생성 스토리보드 빌더" → "2. 캡쳐 이미지 일괄 배치 (18장)"`
2. 다이얼로그가 열리면 PNG 파일을 드래그하거나 클릭하여 선택합니다.
   - 경로: `~/Downloads/QUIZ참고/스토리보드_캡쳐/`
   - 18개 파일 한 번에 선택 가능 (Ctrl/Cmd + A)
3. "배치 실행" 클릭 → 파일 이름 앞 2자리 번호로 슬라이드를 매칭하여 자동 배치됩니다.
4. placeholder 의 점선 테두리/안내문은 제거되고 이미지가 채워집니다.

### 4단계 - Figma Slides 로 이관 (선택)

1. 본 플러그인은 Figma Design 파일에 프레임을 그립니다.
2. Figma Slides 로 옮기려면 각 프레임을 우클릭 → Copy → Slides 파일에서 슬라이드 추가 후 붙여넣기.

## 자동 채워지는 텍스트 전용 슬라이드

| **ID** | **자동 채움 내용** |
|---|---|
| C-01 커버 | 제목, 부제, 대상, 작성자, 작성일, 버전 (중앙 정렬) |
| C-02 목차 | 22장 슬라이드 전체 목록 (번호/ID/제목/카테고리 표) |
| C-03 플로우 | 사용자 플로우 ASCII 다이어그램 |
| M-01 매핑표 | 19행의 컴포넌트 ↔ 코드 위치 매핑 표 |
| M-02 메모 | 범위 외/제약/후속 논의 항목 (3 섹션 불릿 리스트) |
| E-01 검증오류 | 8개 검증 규칙 표 + 직렬 검증 방식 설명 |

## 슬라이드 구성

| **카테고리** | **장수** | **포함 슬라이드 ID** |
|---|---|---|
| 표지 (cover) | 2 | C-01, C-02 |
| 개요 (overview) | 1 | C-03 |
| 본문 (main) | 15 | S-01 ~ S-10 (시험 설정 토글 ON 등 분할 포함) |
| 부록 (extra) | 4 | E-01, E-02, M-01, M-02 |
| **총합** | **22** | |

## 디자인 토큰

플러그인이 사용하는 색상은 XN Quiz 디자인 시스템 (Toss 스타일) 기준입니다.

- Primary: `#3182F6`
- Accent: `#E8F3FF`
- Foreground: `#191F28`
- Secondary FG: `#4E5968`
- Muted FG: `#8B95A1`
- Placeholder BG: `#F2F4F6`
- Border: `#E5E8EB`

## 폰트 자동 선택

플러그인은 환경에 설치된 폰트를 자동 탐지하여 사용합니다.

- 후보 (선호 순): Pretendard → Inter → Noto Sans KR → Noto Sans → Roboto → Arial
- 후보 중 Bold/Medium/Regular 모두 갖춘 첫 번째 폰트 선택
- 일부 스타일이 없는 경우 Regular 로 자동 폴백

## 트러블슈팅

### 플러그인 실행 실패 메시지가 떴을 때

플러그인이 실패하면 자동으로 빨간 에러 창이 표시됩니다 (자동 사라지지 않음). 창 내용:

- 실패 지점: 어떤 단계에서 막혔는지
- 메시지: 정확한 에러 내용

더 자세한 스택 추적은 `Plugins → Development → Open Console` 에서 확인 가능합니다. `[XN Storyboard]` prefix 로 시작하는 로그를 찾으세요.

### 캡쳐 일괄 배치가 매칭되지 않을 때

파일 이름은 반드시 `NN_` (2자리 숫자 + 언더스코어) 로 시작해야 합니다.

- `01_S-01_진입점.png` 정상
- `S-01.png` 매칭 실패 (번호 prefix 없음)
- `1_S-01.png` 매칭 실패 (1자리)

`npm run capture-storyboard` 가 생성하는 파일명은 자동으로 올바른 형식입니다.

### 슬라이드를 잘못 만들어서 다시 만들고 싶을 때

1. Figma 에서 22 프레임을 모두 선택 (Ctrl/Cmd + Click 으로 다중 선택)
2. Delete 키로 삭제
3. 플러그인 "1. 스토리보드 골격 생성" 다시 실행
