## 잔여 개발 우선순위

### 우선순위 1 — 재채점 고도화 (UI 완료 → 실 로직 착수)

현재 채점 대시보드에 재채점 버튼 UI와 수정 전/후 필터가 구현된 상태.  
실제 재채점 로직 구현 필요.

| **문항 유형** | **재채점 여부** |
|---|:---:|
| 객관식 단일/복수, 참/거짓, 단답형, 수치형, 연결형, 다중빈칸 | 재채점 가능 |
| 서술형, 파일 업로드 | 재채점 불가 (수동채점 유지) |
| 수동 점수 조정 문항 | 재채점 제외 |

- 정답 변경 후 "재채점 실행" → 지원 유형 전체 일괄 재계산
- 수동 조정 문항 제외 후 교수자에게 별도 안내
- 재채점 완료 알림은 MVP 이후 과제

---

### 우선순위 2 — 랜덤 출제 고도화

경북대 최우선 요청. 난이도 메타데이터 완성으로 착수 조건 충족됨.

| **기능** | **비고** |
|---|---|
| 복수 문제은행 다중 선택 (전체 선택/해제 모달) | 공수 L |
| 문제은행 그룹별 출제 문항 수 + 배점 설정 | 공수 L |
| 난이도별 랜덤 출제 비율 + 차등 배점 | 공수 L |
| 문항 미리보기 (문제은행 선택 시 팝업) | 선행 작업 |

> 복수 문제은행 랜덤 출제는 공수가 커서 1단계(그룹 선택) → 2단계(출제 옵션 설정) 순으로 분리 착수 권고.

---

### 우선순위 2 — 출력/내보내기 기능 완성

| **기능** | **비고** |
|---|---|
| 시험지 PDF 출력 | 현재 UI만 존재 |
| 학생별 답안지 PDF 출력 / 일괄 다운로드 | 현재 UI만 존재 |
| 파일 업로드 제출물 문항별 분리 다운로드 | 현재 전체 묶음 다운로드만 지원 |
| 응시자 기준 출석부 다운로드 | 경북대 요청 |

> PDF 출력 범위(문제, 답안, 점수, 제출 정보 등 포함 항목)는 정책적으로 확정 후 구현.

### 우선순위 3 — 채점 세부 기능

| **기능** | **비고** |

| 교수자 문항 제목 → 학생 화면 표시 | 현재 기본값(문제1, 문제2…)으로 표시 |

| 엑셀 채점지 업로드 실 로직 완성 | UI 및 파싱 로직 구현됨, 성적 반영 흐름 미완 |

### 우선순위 4 — 실서비스 전환 (현재 프로토타입 범위 외)

| **기능** | **비고** |
|---|---|
| Canvas LMS API 연동 | 파트너십 계약 선행 필요 |

| 채점 완료 → 학습자 점수 공개 흐름 | 학습자 화면 완성 후 착수 |

| 운영 로그 / 이용현황 통계 | 운영 인프라 필요 |
| 동일 IP 응시 제한 / TrustLock 연동 | 외부 계약 선행 필요 |

## 정책 확정 항목

논의필요사항에서 확정된 정책. 이하 항목은 재논의 불필요.

| **#** | **항목** | **확정 내용** |
|---|---|---|
| 1 | 재채점 가능/불가 유형 | 객관식·참거짓·단답형·수치형·연결형·다중빈칸 가능 / 서술형·파일업로드 불가 |
| 2 | 복수선택 채점 방식 | 전역 설정(조직 단위)으로 통합. 채점 방식(`all_correct`/`partial`) + 감점 방식(`none`/`right_minus_wrong`/`formula_scoring`) |
| 3 | 지각 제출 감점 | Canvas Gradebook Late Policies에서 처리 (XN 범위 외) |
| 4 | 퀴즈-문제은행 연동 | 퀴즈에 문항 추가 시 독립 복사 — 이후 은행 수정/삭제 미반영 |
| 5 | 엑셀 업로드 지원 유형 | 객관식·참거짓·단답형·서술형 (행 단위 구조 표현 가능 유형만) |
| 6 | 엑셀 채점지 업로드 정책 | 학번 미매칭·배점 초과 1건이라도 → 전체 취소 / 점수 미입력 행 → 건너뜀 허용 |
| 7 | 응시 중 문항 수정 | 수정 허용, 변경 시점(`questionsModifiedAt`) 기록, 수정 전/후 제출 필터 + 일괄 재채점 |
| 8 | 영문 대소문자 구분 정답 판정 | 전역 설정(조직 단위). 기본값 OFF(대소문자 무시) |
