# XN Quizzes — MVP 우선순위 정리

**작성일**: 2026-04-03
**인풋**: 고객사 요구사항(경북대/인제대) + 기능 리스트 + PM2/PM4/PM5 보고서 + Canvas LMS 소스 분석

## 우선순위 1 — MVP1 필수 (지금 당장)

데이터가 저장되고 상태가 정상 전환되어야 실사용 가능한 단계.

| **기능** | **근거** |
|---|---|
| 퀴즈 생성/편집 저장 영속성 | 새로고침 시 데이터 소실 — 기본 전제 |
| 문제은행 저장 영속성 | 동일 |
| 채점 저장 + 총점 자동 계산 | 채점 워크플로우 완성 |
| 퀴즈 상태 전이 트리거 | "채점 완료" 버튼 → closed 전환 |
| 날짜 역전 / 문항 0개 발행 차단 | 잘못된 퀴즈 발행 방지 |
| 임시저장 (draft) | 발행 전 초안 보관 |
| 점수 공개 시점 제어 | 채점 중 학생 점수 노출 방지 |
| 성적 CSV 다운로드 완성 | 현재 부분 구현 → 완료 |

## 우선순위 1.5 — 재채점 고도화 (MVP1 직후)

현재 객관식(단일) 수준의 재채점만 지원. 정답 오류 발견 시 전체 학생에게 소급 적용하는 기능이 없으면 운영 불가 수준의 리스크.

Canvas LMS 소스(`quiz_regrader/regrader.rb`, `submission_grader.rb`) 분석 결과, Canvas는 정답 변경 시 자동채점 문항 전체를 일괄 재계산하는 구조를 사용함.

**재채점 지원 확장 대상 (Canvas 기준)**

| **문항 유형** | **현재** | **목표** |
|---|:---:|:---:|
| 객관식 단일 선택 (multiple_choice) | 지원 | 유지 |
| 객관식 복수 선택 (multiple_answers) | 미지원 | **지원** |
| 참/거짓 (true_false) | 미지원 | **지원** |
| 단답형 (short_answer) | 미지원 | **지원** |
| 수치형 (numerical) | 미지원 | **지원** |
| 연결형 (matching) | 미지원 | **지원** |
| 다중빈칸 (fill_in_multiple_blanks) | 미지원 | **지원** |
| 서술형 (essay) | 미지원 | 재채점 불가 (수동채점 유지) |
| 파일 업로드 (file_upload) | 미지원 | 재채점 불가 (수동채점 유지) |

**구현 방향**

- 정답 변경 후 "재채점 실행" 시 → 위 지원 유형 전체 일괄 재계산
- 재채점 결과로 총점 자동 갱신
- 수동채점 문항(서술형, 파일업로드)은 재채점 대상에서 제외하고 교수자에게 별도 안내

## 우선순위 2 — MVP1.5 (문제은행 자산화)

경북대 최우선 요청 집중 구간. 난이도/그룹 메타데이터가 선행되어야 랜덤 출제 고도화 착수 가능.

| **기능** | **근거** |
|---|---|
| 난이도 (상/중/하) 메타데이터 | 경북대 핵심 요청 + 기능 리스트 신규 항목 |
| 그룹 태그 (단원/챕터) 메타데이터 | 동일 |
| 은행 간 문항 이동/복사 단순화 | 경북대 — 현재 흐름 복잡 |
| 문항 다중 선택 삭제 | 기능 리스트 신규 |
| 복수 문제은행 × 그룹별 출제 수 설정 | 경북대 핵심 요청 (공수 L) |
| 난이도별 랜덤 출제 비율 + 차등 배점 | 경북대 핵심 요청 (공수 L) |
| 엑셀 업로드 난이도/그룹 컬럼 반영 | 경북대 + 기능 리스트 |

> 복수 문제은행 랜덤 출제는 공수가 커서 메타데이터 완성 후 별도 착수 권고.

## 우선순위 3 — MVP2 (채점 고도화 + 출력)

"다수 학교 요청"으로 명시된 출력 기능 포함. 채점 완결성 높이는 구간.

| **기능** | **근거** |
|---|---|
| 단답형 대소문자/공백 판정 옵션 | 경북대 요청 + 기능 리스트 신규 |
| 복수선택형 부분 점수 | 경북대 요청 + 기능 리스트 신규 |
| 엑셀 채점지 업로드 실제 구현 | 현재 UI만 존재 |
| 재채점 실제 구현 | 현재 UI만 존재 |
| 시험지 PDF 출력 | 다수 학교 요청 |
| 학생별 답안지 출력 / 일괄 다운로드 | 다수 학교 요청 |
| 응시자 기준 출석부 다운로드 | 경북대 요청 |
| 퀴즈 복제 | 경북대 요청 |
| 주차별 퀴즈 목록 뷰 | 인제대 요청 + 기능 리스트 신규 |

## 우선순위 4 — MVP3 (학습자 + 연동)

실서비스 전환 시 필요. 현재 프로토타입 범위 외.

| **기능** | **근거** |
|---|---|
| 학습자 응시 화면 (12종 유형, 타이머) | 실서비스 필수, 공수 XL |
| 채점 완료 → 학습자 점수 공개 흐름 | 학습자 화면 선행 필요 |
| 재응시 흐름 (횟수 제한, 점수 정책) | 동일 |
| 멀티미디어 첨부 (이미지/음향) | 파일 스토리지 인프라 선행 필요 |
| Canvas LMS API 연동 | 파트너십 계약 선행 필요 |
| 운영 로그 / 이용현황 통계 | 운영 인프라 필요 |
| 동일 IP 응시 제한 / TrustLock 연동 | 외부 계약 선행 필요 |
