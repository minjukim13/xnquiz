# 미결

## 1-1. 정답 변경 소급 적용 범위 [상태: 미결]

문제 정정 및 복수 정답 추가 시 소급 적용을 기본으로 하되, 풀이 중 세션은 응시 시작 시점 스냅샷 기준으로 채점 유지

| **학생 그룹** | **처리 기준** |
|---|---|
| 현재 풀이 중 | 변경 미반영 (응시 시작 시점 기준 정답) |
| 이미 제출 완료 | 소급 재채점 자동 반영 (수동 조정 문항 제외) |
| 이후 응시 예정 | 변경된 정답 기준 자동 채점 |

복수 정답 추가 = 정답 변경과 동일 플로우, 수동 조정 문항 제외, 감점 발생 시에도 동일 처리

---

## 2-3. 복수 응시 점수 정책 소급 변경 [상태: 미결]

최고점/최신/평균 중 선택한 채점 방식을 교수자가 중간에 변경할 경우 기존 응시자에 소급 적용할지.

**Canvas 방식**: 소급 미적용 (변경 이후 응시분부터 적용)

---

## 2-4-감점. 복수선택 감점 방식 [상태: 미결]

`multiple_answers` 유형에서 오답 선택 시 감점 적용 여부 및 감점 하한(최소 0점) 설정 여부.

- 연관 결정: 2-4 (채점 방식 결정됨), 1-2 (재채점 가능 유형)

---

## 4-5. 응시 중 상태에서 문항 수정 범위 [상태: 미결]

### Canvas LMS 리서치 결과

**Canvas Classic Quizzes**:
- 퀴즈 발행 후에도 문항 편집 가능
- 응시 중 세션: 응시 시작 시점 스냅샷 기준 유지, 편집 내용 미반영
- 이미 제출된 응답: 편집만으로는 재채점 미발생, 교수자가 수동으로 Regrade 트리거해야 함
- 이후 응시자: 편집된 버전 즉시 노출

**Canvas New Quizzes (현재 표준)**:
- 첫 응시자 발생 후 편집 기능 일부 잠금
- 문항 텍스트 오탈자 수정: 허용
- 정답 변경, 선택지 추가/삭제, 문항 삭제, 배점 변경: 제한 또는 경고 후 재채점 필요

### 권고안

Canvas New Quizzes 접근 방식 채택을 권장.

| **수정 항목** | **권고** | **비고** |
|---|---|---|
| 문항 문구 오탈자 수정 | 허용 | 텍스트만, 의미 변경 없는 수정 |
| 선택지 텍스트 수정 | 허용 (주의) | 기존 응시자 스냅샷 기준 유지 |
| 정답 변경 | 허용 + 재채점 트리거 | 제출 완료 응답에 소급, 진행 중 세션 제외 |
| 배점 변경 | 차단 | 응시 발생 후 변경 불가 |
| 문항 삭제 | 차단 | 응시 기록 있으면 삭제 불가 |
| 문항 추가 | 차단 | 응시 중 추가 불가 |
| 선택지 추가/삭제 | 차단 | 응시 중 구조 변경 불가 |

**결정이 필요한 쟁점**: 정답 변경 + 소급 재채점 자동화 여부 (자동 vs 교수자 수동 트리거)

# 결정됨

## 1-2. 재채점 가능/불가 유형 [상태: 결정됨 — 2026-04-08]

| **재채점 가능** | **재채점 불가 (수동채점 유지)** |
|---|---|
| 객관식 단일/복수, 참/거짓, 단답형, 수치형, 연결형, 다중빈칸 | 서술형, 파일 업로드 |

- 수동 조정 문항은 재채점에서 제외
- 재채점 완료 알림: 알림 기능 미포함 (MVP 이후 과제)

---

## 2-1. 소수점 점수 허용 [상태: 결정됨 — 2026-04-08 / 구현 완료]

소수점 허용, 최소 단위 0.5점. 저장은 소수 둘째 자리까지. 표시 규칙: 합산은 소수점 첫째 자리까지 반올림.

- 적용: 문항 배점 입력(AddQuestionModal), 채점 점수 입력(GradingDashboard) → `step={0.5}`

---

## 2-2. 미채점 문항 총점 처리 [상태: 결정됨 — 2026-04-08 / 구현 완료]

미채점(수동채점 대기) 문항은 0점으로 계산하여 총점에 반영. 제출 직후 ResultModal 및 학생뷰 종료 카드에서 `총 점수 / 퀴즈 전체 배점` 형식으로 표시.

- 수동채점 미완료 안내 배너는 별도 표시 유지

---

## 2-4. 복수선택 채점 방식 [상태: 결정됨 — 2026-04-08 / 구현 완료]

`multiple_answers` 유형 문항 생성 시 교수자가 채점 방식을 직접 선택.

| **옵션** | **설명** | **기본값** |
|---|---|---|
| 전체 정답 시에만 만점 (`all_correct`) | 오답 하나라도 포함 시 0점 | 기본값 |
| 정답 비율 배점 (`partial`) | 정답 개수 / 전체 정답 수 × 배점 | 선택 |

- 문항 데이터에 `scoringMode` 필드 추가
- 감점 방식은 미결로 유지 (§2-4-감점)

---

## 2-5. 재응시 후 기존 채점 기록 확인 [상태: 결정됨 — 2026-04-08 / 구현 완료]

재응시 후에도 기존 응시 회차별 채점 기록을 학생이 확인할 수 있어야 한다. 새로운 응시 결과 점수 정책(`scorePolicy`: 최고점/최신/평균)에 따른 최종 점수 계산은 유지.

- 학생 뷰 종료 카드: 복수 응시 이력이 있는 경우 "응시 기록 N회" 표시 및 회차별 점수 확인 가능

---

## 3-1. 문제은행 문항 삭제 시 처리 [상태: 결정됨]

문제은행-퀴즈 연동이 독립 복사 방식으로 확정됨에 따라, 문제은행 문항 삭제는 이미 퀴즈에 복사된 문항에 영향 없음.

---

## 3-2. 문제은행-퀴즈 연동 방식 [상태: 결정됨: 독립 복사]

문제은행에서 퀴즈로 문항 추가 시 해당 시점에 데이터를 복사. 이후 문제은행 수정은 퀴즈에 반영되지 않음.

---

## 4-1. 정오답/정답 공개 설정 [상태: 결정됨: 구현 완료]

퀴즈 생성 시 교수자가 직접 설정 가능. 구현된 설정 구조:

| **설정 항목** | **옵션** |
|---|---|
| 정오답 여부 공개 | 공개 / 비공개 |
| 정답 공개 | 정오답과 함께 정답 표시 / 정오답 여부만 |
| 정오답 공개 시기 | 제출 즉시 / 마감 후 / 기간 설정 |

## 4-3. 퀴즈 상태 전이 자동화 정책 [상태: 결정됨 — 2026-04-08]

| **전이** | **결정** |
|---|---|
| `open` → `grading` | `dueDate` 경과 시 자동 전환 |
| `grading` → `closed` | 성적공개 정책 설정 기준으로 자동 전환 (수동 "성적 확정" 버튼 없음) |

---

## 4-4. 성적 공개(Release) 정책 [상태: 결정됨 — 2026-04-08]

별도 "성적 공개" 버튼이나 단계를 두지 않음. 퀴즈 생성 시 "성적 공개 정책" 섹션에서 설정한 값에 따라 자동 제어.

**데이터 모델 (결정됨)**

| **필드** | **값** | **설명** |
|---|---|---|
| `scoreRevealEnabled` | `true / false` | 공개 여부 |
| `scoreRevealScope` | `wrong_only / with_answer` | 공개 범위 |
| `scoreRevealTiming` | `immediately / after_due / period` | 공개 시점 |
| `scoreRevealStart` | datetime | 기간 설정 시 시작일 (`period`일 때만) |
| `scoreRevealEnd` | datetime | 기간 설정 시 종료일 (`period`일 때만) |

**공개 범위별 학생 표시**

| **`scoreRevealScope`** | **학생에게 표시되는 것** |
|---|---|
| `wrong_only` (오답 여부만) | 점수 + 정오답(✓/✗). 정답 비공개 |
| `with_answer` (정답까지) | 점수 + 정오답(✓/✗) + 정답 |

- `scoreRevealEnabled = false`이면 아무것도 표시 안 함
- 수동채점 미완료 문항은 0점으로 계산 후 표시 (→ §2-2)
- 적용 범위: QuizCreate "성적 공개 정책" 섹션, QuizEdit, QuizAttempt 제출 모달, QuizList 학생뷰 종료 카드, GradingDashboard 퀴즈 정보 카드 배지

## 5-1. 엑셀 문항 업로드 지원 유형 [상태: 결정됨]

행 단위 구조로 표현 가능한 유형만 지원.

| **유형** | **코드** | **지원 여부** |
|---|---|---|

| 단답형 | short_answer | 지원 |
| 서술형 | essay | 지원 (문항 텍스트만) |

## 5-2. 엑셀 일괄 채점 업로드 정책 [상태: 결정됨]

| **항목** | **결정 내용** |
|---|---|
| 학번 미매칭 처리 | 1건이라도 미매칭 시 전체 업로드 취소 |

## 6-1. timeLimit=0 응시 화면 동작 [상태: 결정됨: 구현 완료]

`timeLimit=0`이면 타이머 UI 미표시 + "제한 없음" 배지 표시 + 무제한 응시.

---

## 6-2. 총점 0점 퀴즈 발행 [상태: 결정됨: 구현 완료]

발행 허용. 응시 결과 화면에서 "점수 없음" 표시, 진행 바 미표시.

---

## 6-3. 응시 횟수 제한 초과 처리 [상태: 결정됨: 구현 완료]

`allowAttempts` 초과 시 버튼 비활성화. 호버 시 "응시 가능 횟수(N회)를 초과했습니다" 툴팁.

---

## 6-4. draft 퀴즈 URL 직접 접근 응시 차단 [상태: 결정됨: 구현 완료]

`quiz.status !== 'open'`이면 응시 차단. 상태별 안내 메시지 + "퀴즈 목록으로" 버튼.

---

## 6-5. 재채점(Regrade) 기능 범위 [상태: 결정됨 — 2026-04-08 / 구현 완료]

- 재채점 대상: **특정 문항**만 허용 (전체 문항 일괄 재채점 없음)
- 재채점 가능 유형: 자동채점 가능 문항만 (→ §1-2)
- 재채점 방식: 특정 문항의 정답 변경 후 소급 적용
- GradingDashboard 및 QuizEdit 문항 목록에서 자동채점 문항에만 "재채점" 버튼 표시

---

## 7-1. 지각 제출 허용 기간 [상태: 결정됨 — 2026-04-08 / 구현 완료]

허용 기간은 교수자가 직접 설정. QuizCreate/QuizEdit "응시 기간" 섹션에서 `allowLateSubmit = true` 시 허용 기간(시간 단위) 입력 필드 표시.

- 감점 방식은 미결 유지 (→ §7-1-감점)

---

## 7-2. 주차/차시 미선택 처리 [상태: 결정됨 — 2026-04-08 / 구현 완료]

미선택 허용. 주차/차시 없이도 퀴즈 생성 가능. 선택 안 하면 null로 저장, 카드에서 주차/차시 태그 미표시.

- QuizCreate/QuizEdit 주차·차시 드롭다운에 "선택 안함" 옵션 추가 (기본값)
- 기존 "연결 안함(0)" 옵션 제거 및 null 처리로 통일
