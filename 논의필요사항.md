# XN Quizzes — 논의 필요사항

**최종 업데이트**: 2026-04-08
**용도**: 스펙 확정 전 의사결정이 필요한 안건 목록. 결정된 항목은 상태를 "결정"으로 변경.
**단일 소스**: PM2 §8, PM5 §7의 미결 정책 항목이 모두 이 파일로 통합됨. 중복 관리하지 않음.

---

## 1. 재채점

### 1-1. 정답 변경 소급 적용 범위 [상태: 미결]

문제 정정 및 복수 정답 추가 시 소급 적용을 기본으로 하되, 풀이 중 세션은 응시 시작 시점 스냅샷 기준으로 채점 유지

| **학생 그룹** | **처리 기준** |
|---|---|
| 현재 풀이 중 | 변경 미반영 (응시 시작 시점 기준 정답) |
| 이미 제출 완료 | 소급 재채점 자동 반영 (수동 조정 문항 제외) |
| 이후 응시 예정 | 변경된 정답 기준 자동 채점 |

복수 정답 추가 = 정답 변경과 동일 플로우, 수동 조정 문항 제외, 감점 발생 시에도 동일 처리

---

### 1-2. 재채점 가능/불가 유형 [상태: 미결]

| **재채점 가능** | **재채점 불가 (수동채점 유지)** |
|---|---|
| 객관식 단일/복수, 참/거짓, 단답형, 수치형, 연결형, 다중빈칸 | 서술형, 파일 업로드 |

- 수동 조정 문항은 재채점에서 제외
- 재채점 완료 알림 시 문구로만 고지

---

### 1-3. 재채점 후 학생 알림 [상태: 미결]

| **옵션** | **장점** | **단점/리스크** |
|---|---|---|
| 점수 상승만 알림 (즉시) | 긍정 경험, 문의 감소 | 편향 소통, 하락 시 불신 가능 |
| 상승/하락 모두 알림 (공개 후) | 투명성, 혼선 최소화 (공개와 동기화) | 알림 폭주 가능, 공개 지연 시 체감 저하 |

**권고**: MVP는 알림 비포함. 후속 릴리스에서 "공개 후 상승/하락 모두 알림"을 기본값으로 제공, 채널은 인앱 우선, 이메일은 선택.

---

### 1-4. 재채점 이력 관리 [상태: 미결 / MVP 범위 외]

- 재채점 이력 보관 여부 (필수 vs 선택)
- 이력 조회 권한: 교수자만 / 운영자 포함
- 보관 기간 정책

## 2. 채점 정책

### 2-1. 소수점 점수 허용 [상태: 미결]

| **옵션** | **장점** | **단점/리스크** |
|---|---|---|
| 정수만 허용 | 간단, 합산 오차 없음 | 세밀 채점 불가, 공정성 이슈 |
| 소수점 (0.5/0.1) 허용 | 세밀 채점 가능, 국제 관행 부합 | 표시·반올림 규칙 필요, UI 제약 |

**권고**: 소수점 허용 (최소 0.5 스텝), 저장은 소수 둘째 자리까지, 표시 규칙 명시 (합계는 반올림)

---

### 2-2. 미채점 문항 총점 처리 [상태: 미결]

**권고**: 총점은 '임시(미확정)' 상태로 표시하고 평균·성적 반영 계산 제외. 교수자 화면에서 진행률 배지로 미채점 건수 안내. 설정으로 "미채점=0 계산"도 선택 가능하되 기본값은 보류.

---

### 2-3. 복수 응시 점수 정책 소급 변경 [상태: 미결]

최고점/최신/평균 중 선택한 채점 방식을 교수자가 중간에 변경할 경우 기존 응시자에 소급 적용할지.

**Canvas 방식**: 소급 미적용 (변경 이후 응시분부터 적용)

---

### 2-4. 복수선택 부분 점수 및 감점 정책 [상태: 미결]

`multiple_answers` 유형 문항에서 일부만 정답 선택 시 점수 처리 방식.

| **옵션** | **설명** |
|---|---|
| 전체 정답 시에만 만점 | 오답 하나라도 포함 시 0점 |
| 정답 비율 배점 | 정답 개수 / 전체 정답 수 × 배점 (예: 3개 중 2개 = 2/3점) |
| 교수자 선택 | 문항 생성 시 직접 선택 |

추가 쟁점: 오답 선택 시 감점 적용 여부, 감점 하한(최소 0점) 설정 여부.

- 연관 안건: 1-2(재채점 가능 유형), PM5 AC-04

---

### 2-5. 채점 완료 후 재응시 처리 [상태: 미결]

채점이 완료된 퀴즈에서 학생이 재응시를 요청하거나 `allowAttempts` 범위 내 재응시가 가능한 경우 처리 기준.

- 재응시 허용 시 기존 채점 기록 유지 여부
- 재응시 후 점수 정책(`scorePolicy`) 재적용 기준
- 교수자 채점이 완료된 수동채점 문항의 재채점 여부

## 3. 문제은행

### 3-1. 문제은행 문항 삭제 시 처리 [상태: 결정: 퀴즈 영향 없음]

문제은행-퀴즈 연동이 독립 복사 방식으로 확정됨에 따라, 문제은행 문항 삭제는 이미 퀴즈에 복사된 문항에 영향 없음.

---

### 3-2. 문제은행-퀴즈 연동 방식 [상태: 결정: 독립 복사]

문제은행에서 퀴즈로 문항 추가 시 해당 시점에 데이터를 복사. 이후 문제은행 수정은 퀴즈에 반영되지 않음.

| **항목** | **독립 복사** | **동기화 연동** |
|---|---|---|
| 설명 | 가져오는 시점 복사, 이후 영향 없음 | 문제은행 수정이 퀴즈에 자동 반영 |
| 장점 | 퀴즈 안정성, 예기치 않은 변경 차단 | 오류 정정 일괄 반영 편리 |
| 단점 | 정답 오류는 퀴즈별 수동 수정 필요 | 진행 중 문항이 예고 없이 바뀜 |
| Canvas 방식 | 독립 복사 | |

---

## 4. 발행 / 공개 정책

### 4-1. 정오답/정답 공개 설정 [상태: 결정: 구현 완료]

퀴즈 생성 시 교수자가 직접 설정 가능. 구현된 설정 구조:

| **설정 항목** | **옵션** |
|---|---|
| 정오답 여부 공개 | 공개 / 비공개 |
| 정답 공개 | 정오답과 함께 정답 표시 / 정오답 여부만 |
| 정오답 공개 시기 | 제출 즉시 / 마감 후 / 기간 설정 |
| 정답 공개 시기 | 정오답 공개와 동시 / 마감 후 / 기간 설정 |

---

### 4-2. 게시 취소(open → draft) 허용 여부 [상태: 미결]

이미 응시한 학생이 있는 상태에서 교수자가 퀴즈를 게시 취소(open → draft)할 수 있는지 여부.

- 옵션 A: 응시 기록이 없는 경우에만 허용
- 옵션 B: 조건 없이 허용 (응시 기록은 숨김 처리)
- 옵션 C: 게시 취소 전면 차단 (open → draft 전이 없음)

응시 기록이 있을 때 허용하는 경우, 해당 응시 기록 보존/삭제 기준도 함께 정의 필요.

---

### 4-3. 퀴즈 상태 전이 자동화 정책 [상태: 결정됨 — 2026-04-08]

| **전이** | **결정** |
|---|---|
| `open` → `grading` | `dueDate` 경과 시 자동 전환 |
| `grading` → `closed` | 성적공개 정책 설정 기준으로 자동 전환 (수동 "성적 확정" 버튼 없음) |

- 연관 안건: §4-4 (성적공개 정책)와 통합 결정됨

---

### 4-4. 성적 공개(Release) 정책 [상태: 결정됨 — 2026-04-08]

별도 "성적 공개" 버튼이나 단계를 두지 않음. 퀴즈 생성 시 설정한 **성적공개 정책**에 따라 자동 제어.

| **옵션 (`scoreReleasePolicy`)** | **학생에게 표시되는 것** |
|---|---|
| `null` (비공개) | 아무것도 표시 안 함 |
| `wrong_only` (오답여부만 공개) | 점수 + 정오답(✓/✗) 표시. 정답 비공개 |
| `with_answer` (정답까지 공개) | 점수 + 정오답(✓/✗) + 정답 표시 |
| `after_due` (마감 후 자동공개) | `dueDate` 경과 후 점수 + 정오답 + 정답 표시 |
| `period` (기간 설정) | `scoreRevealStart~scoreRevealEnd` 기간 내에만 표시 |

- 수동채점 미완료 문항은 0점으로 계산 후 표시 (결정됨)
- `isReleased` 플래그 또는 `published` 상태 별도 불필요 — `scoreReleasePolicy` 필드로 대체
- 적용 범위: QuizCreate 기본정보 탭, QuizEdit 표시 설정, QuizAttempt 제출 모달, QuizList 학생뷰 종료 카드

---

### 4-5. 응시 중 상태에서 문항 수정 범위 [상태: 미결]

| **수정 항목** | **허용 여부** |
|---|---|
| 문항 문구 오탈자 수정 | 허용 권고 |
| 선택지 내용 수정 | 논의 필요 |
| 정답 변경 | 논의 필요 |
| 배점 변경 | 차단 권고 (형평성 문제) |
| 문항 삭제 | 차단 권고 |
| 문항 추가 | 논의 필요 |

## 5. 엑셀 업로드 정책

### 5-1. 엑셀 문항 업로드 지원 유형 [상태: 결정: 4개 유형만 지원]

행 단위 구조로 표현 가능한 유형만 지원. numerical 포함 시 오차 범위 컬럼이 필요해 과도한 스펙으로 판단하여 제외.

| **유형** | **코드** | **지원 여부** |
|---|---|---|
| 객관식 | multiple_choice | 지원 (보기 2개 이상 필수, 정답 필수) |
| 참/거짓 | true_false | 지원 (정답 필수) |
| 단답형 | short_answer | 지원 (정답 필수, 없으면 에러) |
| 서술형 | essay | 지원 (정답 없음, 문항 텍스트만) |
| 수치형 이하 나머지 | — | 미지원 (UI 에디터에서만 생성) |

해설 컬럼: 제거 확정. 현재 문항 직접 추가 시에도 해설 입력 기능 없음.

---

### 5-2. 엑셀 일괄 채점 업로드 정책 [상태: 결정]

| **항목** | **결정 내용** |
|---|---|
| 학번 미매칭 처리 | 1건이라도 미매칭 시 전체 업로드 취소. 해당 학번을 에러 메시지에 명시 |
| 점수 미입력 행 | 건너뜀 허용 (보류 상태로 간주) |
| 배점 초과 | 1건이라도 배점 초과 시 전체 업로드 취소 |
| 오류 발생 시 | 부분 저장 없음. 전체 취소 후 재업로드 유도 |

## 6. PM4 QA — 스펙 미정의 항목 (2026-04-03 도출)

### 6-1. timeLimit=0 (제한 없음) 응시 화면 동작 [상태: 결정: 구현 완료]

`timeLimit=0`이면 타이머 UI 미표시 + "제한 없음" 배지 표시 + 무제한 응시로 처리. 응시 결과 화면에서 "시간 제한 없음"으로 표시.

---

### 6-2. 총점 0점 퀴즈 발행 [상태: 결정: 발행 허용 + 점수 없음 처리]

발행 허용. 응시 결과 화면에서 자동채점 결과 영역에 "점수 없음" 표시, 진행 바 미표시.

---

### 6-3. 응시 횟수 제한 초과 처리 [상태: 결정: 구현 완료]

`allowAttempts` 초과 시 응시하기 버튼 비활성화(회색). 호버 시 "응시 가능 횟수(N회)를 초과했습니다" 툴팁 표시.

---

### 6-4. draft 퀴즈 URL 직접 접근 응시 차단 [상태: 결정: 구현 완료]

`quiz.status !== 'open'`이면 응시 차단. 상태별 안내 메시지 표시 (draft: 미발행, grading: 채점 중, closed: 종료). "퀴즈 목록으로" 버튼으로 홈 이동.

---

### 6-5. 재채점(RegradeModal) 기능 범위 [상태: 미결]

재채점 모달 UI는 존재하나 내부 로직 미구현.

- 재채점 대상: 전체 문항 vs 선택 문항
- 재채점 방식: 자동채점 재실행 / 수동채점 초기화 / 특정 문항 정답 변경 후 소급
- 연관 안건: 1-1(정답 변경 소급 적용 범위)

---

### 6-6. 채점 종료 후 채점 대시보드 재진입 허용 여부 [상태: 미결]

퀴즈 status가 'closed'인 경우에도 `/quiz/:id/grade` 직접 접근 시 채점 UI 그대로 표시.

- 옵션 A: 채점 종료 후 읽기 전용 모드로 표시 (수정 불가)
- 옵션 B: 채점 가능 상태 유지 (현행)

---

### 6-7. 유효하지 않은 quizId/bankId 접근 시 폴백 정책 [상태: 미결]

현재 존재하지 않는 ID로 접근 시 첫 번째 데이터로 자동 폴백. 잘못된 데이터가 표시될 수 있음.

- 옵션 A: 에러 UI 표시
- 옵션 B: 홈으로 리다이렉트
- 옵션 C: 현행 폴백 유지

---

## 7. 기타

### 7-1. 지각 제출 감점 정책 세부 [상태: 미결]

- 허용 기간: 마감 후 몇 시간까지 허용할 것인지 (교수자 직접 설정 vs 고정값)
- 감점 방식: 감점 없음 / 일정 비율 감점 / 교수자 직접 설정

---

### 7-2. 주차/차시 미선택 시 기본값 처리 [상태: 미결]

퀴즈 생성 시 주차/차시를 선택하지 않으면 현재 1주차 1차시로 자동 저장됨. 의도된 동작인지 확인 필요.

- 현행 유지 (미선택 시 1/1 기본값)
- 미선택 허용 (주차/차시 없는 퀴즈 생성 가능)
- 주차/차시를 필수 항목으로 변경

---

### 7-3. 주차별 보기 기본 표시 여부 [상태: 미결]

퀴즈 목록 화면의 기본 뷰를 "전체 목록"으로 할지 "주차별 그룹핑 보기"로 할지 결정 필요.

- **배경**: 인제대는 주차별 보기를 선호하나, 타 기관은 전체 목록이 더 직관적일 수 있음
- 옵션 A: 전체 목록 기본, 주차별 보기 토글
- 옵션 B: 주차별 보기 기본, 전체 목록 토글
- 옵션 C: 기관별 설정값으로 제어
