# PRD — XN Quizzes

> 작성일: 2026-03-27
> 버전: v1.0
> 상태: 초안

---

## 1. Summary

XN Quizzes는 기존 Canvas LMS 퀴즈 기능의 한계를 해결하기 위해 독립적으로 개발된 웹 기반 퀴즈 관리 도구다. Canvas 퀴즈의 UI·기능 제약에서 벗어나, 교수자가 채점·문항 관리·성적 분석을 하나의 도구에서 효율적으로 처리할 수 있도록 하는 것이 목표다.

---

## 2. Background

대학 교수자는 퀴즈, 형성평가, 중간/기말고사 등 다양한 평가를 운영하지만, 기존 Canvas LMS 퀴즈는 아래 한계를 가진다:

- **채점 비효율**: 학생별 페이지 전환이 필요해 대규모 채점에 부적합
- **문제은행 재사용 불편**: 이전 문항을 찾아 복붙하는 작업 반복
- **통계 부재**: 문항 난이도 분석·성적 분포 시각화 없음
- **서술형 채점 UI 조악**: SpeedGrader 외 대안 없고 학습 곡선 높음

XN Quizzes는 Canvas의 이러한 제약에서 완전히 독립하여, UI와 기능을 자유롭게 설계할 수 있는 별도 도구로 개발한다.

---

## 3. Objective

교수자가 퀴즈 생성부터 채점, 결과 공유까지 하나의 도구에서 끝낼 수 있도록 하여, 반복 업무 시간을 줄이고 학습자에게 빠른 피드백을 제공한다.

### 비즈니스 가치

- **교수자**: 채점 시간 단축 → 교육 품질 개선에 집중 가능
- **학습자**: 빠른 피드백 → 학습 효과 향상

### Key Results (KR)

| 지표 | 목표 | 측정 시점 |
|------|------|----------|
| 채점 완료율 | 90% 이상 | 파일럿 운영 후 |
| 퀴즈 응시율 | 85% 이상 | MVP2 이후 |
| 평균 채점 소요 시간 | 기존 대비 30% 단축 | 실데이터 연동 후 |
| 문제은행 재사용률 | 30% 이상 | MVP2 이후 |
| 교수자 채점 이탈률 | 10% 이하 | 실데이터 연동 후 |

## 6. Solution

### 6.1 사용자 흐름

**교수자**
```
퀴즈 목록 → 생성(기본정보·문항 추가·발행) → 채점 대시보드(문항/학생 중심) → 채점 완료 확정 → 점수 공개 → 통계 확인
```

**학습자** (MVP2)
```
응시 가능 퀴즈 확인 → 응시 시작 → 답안 입력 → 제출 → 결과 확인
```

### 6.2 핵심 기능

#### MVP1 — 교수자 도구

**기능 1. 퀴즈 생성 및 편집**
- 기본정보: 제목, 설명, 과목, 시작/마감 일시, 응시 제한 시간, 총점
- 문항 추가: 11가지 유형 (객관식, 진위형, 복수 선택, 단답형, 서술형, 수치형, 연결하기, 빈칸, 다중 빈칸, 드롭다운, 순서 배열, 파일 첨부)
- 문제은행에서 검색·선택·추가
- 발행 유효성 검증: 문항 1개 이상, 총점 1점 이상, 시작 일시 ≤ 마감 일시
- 임시저장(draft) 가능

**기능 2. 채점 대시보드**
- 문항 중심 모드: 한 문항에 대한 모든 학생 답안 나열, 점수 입력
- 학생 중심 모드: 한 학생의 전체 문항 답안, 점수 입력
- 자동채점 문항 결과 자동 표시 (수정 가능)
- 채점 진행률 표시 (문항별/학생별)
- 채점 완료 확정 → 상태 grading → closed 전환

**기능 3. 퀴즈 통계**
- 성적 분포 막대 차트, 문항별 난이도 분석, 학생별 성적표, CSV 다운로드

**기능 4. 문제은행**
- 문항 CRUD, 유형·과목·키워드 검색 필터, 퀴즈 생성 시 문항 가져오기

**기능 5. 퀴즈 목록 및 상태 관리**
- 상태: draft / open / grading / closed, 상태별 필터링, 채점 대기 건수 표시

#### MVP2 — 학습자 응시 + 정책 완성

**기능 6. 퀴즈 응시**: 응시 시작, 시간 제한, 자동 제출, 중간 저장

**기능 7. 재응시 및 결과 확인**: 교수자 설정에 따른 재응시 허용, 최고 점수 유지, 피드백 확인

**기능 8. 지각 제출 정책**: 교수자 설정 (기본값: 불허 / 허용 시 감점 0·10·50%)

**기능 9. 점수 공개 정책**: 기본값 교수자 수동 공개, 자동 공개 옵션

#### MVP3 — 외부 연동 (검토 중)

Canvas Gradebook 성적 반영, 수강생 명단 동기화, LTI 통합 등 — 범위 및 방식은 추후 확정

### 6.3 기술 스택

| 항목 | 내용 |
|------|------|
| 프론트엔드 | React 19 + Vite 8 + Tailwind CSS v4 |
| 차트 | recharts |
| 라우팅 | react-router-dom v7 |
| 데이터 | REST API 연동 (백엔드 미정) |
| 인증 | 자체 JWT 또는 외부 연동 (추후 결정) |

### 6.4 주요 가정

| 가정 | 검증 방법 |
|------|----------|
| 교수자는 문항 중심 채점 모드를 더 자주 사용할 것이다 | 파일럿 후 탭 클릭 비율 추적 |
| 재응시 허용이 학습 효과를 높일 것이다 | 재응시 전후 점수 변화 비교 |
| 문제은행 재사용률이 30% 이상 될 것이다 | 실데이터 연동 후 집계 |

## 7. Release

### MVP1 — 교수자 도구

- 퀴즈 생성·편집·발행 (저장, 유효성 검증 포함)
- 채점 대시보드 (데이터 영속성 포함)
- 퀴즈 통계, 문제은행 CRUD
- 상태 전이 자동화

제외: 학습자 응시 화면, 알림, 외부 연동

### MVP2 — 학습자 응시 + 정책

- 학습자 응시 화면 (시간 제한, 자동 제출, 재응시)
- 지각 제출·점수 공개·단답형 자동채점 정책
- 역할 기반 접근 제어 (교수자/조교/학습자)
- 알림 (마감 임박, 채점 완료)

### MVP3 — 외부 연동 (범위 추후 확정)

Canvas Gradebook 반영, 수강생 동기화, LTI/SSO 통합
