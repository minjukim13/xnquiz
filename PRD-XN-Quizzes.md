# PRD — XN Quizzes

> 작성일: 2026-03-27
> 버전: v1.0
> 상태: 초안

---

## 1. Summary

XN Quizzes는 대학 교수자가 Canvas LMS 환경에서 퀴즈를 생성·배포·채점하고, 학습자가 응시 및 결과를 확인할 수 있도록 설계된 웹 기반 퀴즈 관리 시스템이다. 교수자의 반복적인 채점 작업을 줄이고, 학습자에게 즉각적인 피드백을 제공하는 것을 핵심 목표로 한다. 현재 MVP1 프로토타입이 완성된 단계이며, 이 문서는 MVP1 완성 및 MVP2 이후 개발 방향을 정의한다.

## 3. Background

### 배경

대학 교수자는 학습 효과를 측정하기 위해 퀴즈, 형성평가, 중간/기말고사 등 다양한 평가 도구를 운영한다. 그러나 기존 Canvas LMS의 기본 퀴즈 기능은 아래 한계를 가진다:

- 채점 인터페이스가 비효율적 — 학생별 페이지 전환이 필요해 대규모 채점에 부적합
- 문제은행 재사용이 불편 — 이전 문항을 다시 찾아 복붙하는 작업 반복
- 통계 기능 부재 — 문항 난이도 분석, 성적 분포 시각화가 없어 교육 개선에 활용 어려움
- 서술형/파일 제출 채점 UI가 조악 — SpeedGrader 외 대안이 없고 학습 곡선이 높음

### 왜 지금인가

한국 대학 시장에서 Canvas LMS 도입이 급격히 확산되고 있다. 교수자들이 기본 기능의 한계를 인지하기 시작했고, 보조 도구에 대한 수요가 생기고 있다. 또한 React 19 + Vite 8 기술 스택이 성숙 단계에 접어들어, 가볍고 유지보수가 쉬운 SPA 형태의 퀴즈 도구를 빠르게 만들 수 있는 환경이 갖춰졌다.

## 4. Objective

### 목표

교수자가 퀴즈 생성부터 채점, 결과 공유까지 하나의 도구 안에서 끝낼 수 있도록 하여, 반복 업무에 소모되는 시간을 줄이고 학습자에게 더 빠른 피드백을 제공한다.

### 비즈니스 가치

- **교수자**: 채점 시간 단축 → 교육 품질 개선에 집중 가능
- **학습자**: 빠른 피드백 → 학습 효과 향상
- **운영자(대학)**: Canvas 연동 도구로 LMS 만족도 향상 → 계약 유지율 상승

### 전략적 정렬

Canvas LMS 생태계 내 부가 도구(LTI) 시장 진입. 운영사(학교/기업 교육)에 번들 판매 가능한 모듈형 제품 구조 지향.

### Key Results (KR)

| 지표 | 목표 | 측정 시점 |
|------|------|----------|
| 채점 완료율 (채점 시작 후 완료까지) | 90% 이상 | MVP1 완성 후 파일럿 운영 |
| 퀴즈 응시율 (배포 대비 제출 수) | 85% 이상 | MVP2 이후 실데이터 |
| 평균 채점 소요 시간 (학생 1인당) | 기존 대비 30% 단축 | 실데이터 연동 후 |
| 문제은행 재사용률 (기존 문항 재활용 비율) | 30% 이상 | MVP2 이후 |
| 교수자 세션 이탈률 (채점 도중 이탈) | 10% 이하 | 실데이터 연동 후 |

## 5. Market Segments

### 주요 대상

**1. 대학 교수자 (주 사용자)**

- Canvas LMS를 도입한 대학의 교수 및 강사
- 수강생 30명~200명 규모 강좌 운영
- 주 1~3회 형성평가, 학기당 중간/기말고사 각 1회
- 서술형/파일 첨부 문항을 포함한 혼합형 시험을 출제함
- 기술 친숙도: 중간 수준 (Excel 사용 가능, 코드 작성 불가)

**2. 대학원 조교 (부 사용자)**

- 교수자 위임을 받아 채점을 대행
- 학부 수업 채점의 50% 이상을 실질적으로 담당
- 채점 기준(루브릭)에 따라 점수를 입력하는 역할

**3. 학습자 (최종 수혜자)**

- 수강생 (대학 학부생, 대학원생)
- 모바일 및 PC 환경 모두 사용
- 퀴즈 응시, 결과 확인, 재응시 여부 확인이 주 니즈

**4. 운영자 (간접 이해관계자)**

- 대학 교학처 또는 LMS 담당자
- 전체 퀴즈 현황 모니터링, 부정 응시 감지 수요

### 제약 조건

- 실명/실학번 등 개인정보는 시스템 내부에서 익명 처리 또는 마스킹 필요
- Canvas LMS API 연동 전까지는 독립 실행(standalone) 방식으로 운용
- 모바일 대응 필요 (학습자 응시 화면 최소 375px)

## 7. Solution

### 7.1 UX / 사용자 흐름

#### 교수자 핵심 플로우

```
[퀴즈 목록]
    │
    ├─ [신규 생성] → 기본정보 입력 → 문항 추가 → 발행 설정 → 발행(open)
    │
    ├─ [편집] → 문항 수정 → 저장 (응시 중이면 경고)
    │
    ├─ [채점] → 채점 대시보드
    │             ├─ 문항 중심 탭: 문항별 전체 학생 답안 열람 + 점수 입력
    │             └─ 학생 중심 탭: 학생별 전체 문항 점수 입력
    │           → 채점 완료 확정 → 상태 closed 전환 → 점수 공개
    │
    └─ [통계] → 성적 분포 차트 + 문항 난이도 분석 + CSV 다운로드
```

#### 학습자 핵심 플로우 (MVP2)

```
[응시 가능 퀴즈 확인] → [응시 시작] → [문항별 답안 입력]
    → [제출] → [자동채점 문항 즉시 확인] → [서술형은 교수자 채점 후 확인]
```

### 7.2 핵심 기능

#### [MVP1] 교수자 도구 — 현재 UI 완성, 저장 로직 미구현

**기능 1. 퀴즈 생성 및 편집**

- 기본정보: 제목, 설명, 과목, 시작/마감 일시, 응시 제한 시간, 총점
- 문항 추가: 11가지 유형 지원 (객관식, 진위형, 복수 선택, 단답형, 서술형, 수치형, 연결하기, 빈칸, 다중 빈칸, 드롭다운, 순서 배열, 파일 첨부)
- 문제은행에서 가져오기: 검색 → 선택 → 추가
- 발행 유효성 검증 (필수):
  - 문항 1개 이상
  - 총점 1점 이상
  - 시작 일시 ≤ 마감 일시
- 임시저장 (draft) 가능

**기능 2. 채점 대시보드**

- 문항 중심 모드: 한 문항에 대한 모든 학생 답안을 세로로 나열, 점수 입력
- 학생 중심 모드: 한 학생의 모든 문항 답안을 한 화면에서 채점
- 자동채점 문항: 채점 결과 자동 표시 (수정 가능)
- 수동채점 문항: 빈 점수 입력 필드, 저장 버튼
- 채점 진행률 표시: 문항별/학생별 완료 비율
- 채점 완료 확정 버튼: 상태를 grading → closed로 전환
- 점수 공개 제어: 교수자가 수동으로 공개 시점 결정

**기능 3. 퀴즈 통계**

- 성적 분포 막대 차트 (10점 단위 구간)
- 문항별 난이도 분석 (평균 득점률, 표준편차)
- 학생별 성적표 (이름, 점수, 제출 시간, 소요 시간)
- CSV 다운로드

**기능 4. 문제은행**

- 문항 CRUD (생성, 조회, 수정, 삭제)
- 유형별/과목별/키워드 검색 필터
- 퀴즈 생성 시 문제은행에서 문항 선택 및 추가

**기능 5. 퀴즈 목록 및 상태 관리**

- 상태: draft(초안) / open(진행) / grading(채점 중) / closed(완료)
- 상태별 필터링
- 채점 대기 건수 표시 (뱃지)
- 퀴즈 복사, 삭제, 상태 전환 액션

#### [MVP2] 학습자 응시 화면

**기능 6. 퀴즈 응시**

- 진행 중인 퀴즈 목록 표시
- 응시 시작 → 문항별 답안 입력
- 남은 시간 표시 (제한 시간 설정 시)
- 시간 초과 시 자동 제출
- 중간 저장 (페이지 이탈 후 복귀 시 이어서 응시)

**기능 7. 재응시 및 결과 확인**

- 재응시 허용 여부: 교수자가 퀴즈 설정에서 결정
- 재응시 시 최고 점수 유지 (기본값)
- 결과 화면: 점수, 정답 여부 (공개 설정에 따라), 교수자 피드백

#### [MVP2] 알림 및 정책

**기능 8. 지각 제출 정책**

- 교수자가 지각 허용 기간 설정 가능 (기본값: 불허)
- 허용 시 감점 옵션: 0% / 10% / 50% 중 선택

**기능 9. 점수 공개 정책**

- 기본값: 교수자 수동 공개
- 옵션: 채점 완료 즉시 자동 공개 가능
- 학습자는 공개 전까지 점수 미확인

#### [MVP3] Canvas LMS 연동

**기능 10. Canvas LMS 통합**

- Canvas API를 통한 수강생 명단 자동 동기화
- 성적을 Canvas Gradebook에 직접 반영
- LTI 방식 또는 OAuth2 통합 로그인
- 퀴즈 데이터를 Canvas Assignments와 양방향 동기화

### 7.3 Technology

| 항목 | 현재 | 목표 (MVP2 이후) |
|------|------|----------------|
| 프론트엔드 | React 19 + Vite 8 + Tailwind CSS v4 | 유지 |
| 데이터 | mockData.js (로컬) | REST API 연동 (백엔드 미정) |
| 저장소 | localStorage (임시) | DB 연동 (PostgreSQL 또는 MongoDB 예상) |
| 인증 | 없음 (역할 선택 UI만) | Canvas OAuth2 또는 자체 JWT |
| 차트 | recharts | 유지 |
| 라우팅 | react-router-dom v7 | 유지 |

**주요 기술 제약**:
- `manualChunks`는 함수 형식 사용 (Vite 8 rolldown 요구사항)
- Tailwind v4: `@apply`는 `@layer utilities` 내에서만 사용
- 실명/실학번 사용 금지, 익명 처리 필수

### 7.4 Assumptions

| 가정 | 검증 방법 | 우선순위 |
|------|----------|---------|
| 교수자는 문항 중심 채점 모드를 학생 중심보다 더 자주 사용할 것이다 | 파일럿 운영 후 탭 클릭 비율 추적 | High |
| 재응시 허용이 학습 효과를 높일 것이다 | 재응시 전후 점수 변화 비교 | Medium |
| 문제은행 재사용률이 30% 이상 될 것이다 | 실데이터 연동 후 재사용 문항 수 집계 | Medium |
| 자동채점 즉시 피드백이 학습자 만족도를 높일 것이다 | 설문 또는 NPS | Low (MVP2) |
| Canvas LMS API가 필요한 데이터를 모두 제공할 것이다 | Canvas API 문서 사전 검토 | High (MVP3) |

## 8. Release

### MVP1 — 교수자 단독 채점 도구 완성

**포함 범위**:
- 퀴즈 생성/편집/발행 (저장 로직 포함)
- 채점 대시보드 (API 연동 또는 안정적인 영속성 구현)
- 퀴즈 통계
- 문제은행 CRUD
- 발행 유효성 검증 (마감/시작 일시, 문항 0개 방지)
- 상태 전이 자동화 (grading → closed)
- Critical 이슈 8건 해결

**제외 범위**:
- 학습자 응시 화면
- 실시간 알림
- Canvas LMS 연동

**예상 기간**: 현재 대비 2~3주 추가 개발

---

### MVP2 — 학습자 응시 + 정책 완성

**포함 범위**:
- 학습자 응시 화면 (시간 제한, 자동 제출)
- 재응시 흐름
- 지각 제출 정책 (교수자 설정)
- 점수 공개 제어 (자동/수동)
- 단답형 키워드 자동채점
- 재채점 및 이의 신청 (기본)
