## 1. 핵심 사용자 시나리오 (Happy Path) 정합성

### 1-1. 교수자 시나리오

| 단계 | 구현 경로 | 정합성 | 이슈 |
|---|---|---|---|
| **퀴즈 생성** | `/quiz/new` QuizCreate | 정상 | 임시저장 버튼 UI 존재하나 실제 저장 로직 없음 (버튼만 렌더링) |
| **문항 구성** | QuizCreate > 문항 구성 탭 | 정상 | 문제은행 불러오기, 직접 입력 모두 동작 |
| **설정 구성** | QuizCreate > 기본 정보 탭 | 정상 | 점수 공개 기간, 접근 코드, IP 제한, 지각 제출 허용 옵션 존재 |
| **발행** | QuizCreate > 발행하기 | 조건부 정상 | 발행 시 status='open' 고정. 예약 발행(scheduled) 흐름 없음 |
| **목록 확인** | `/` QuizList | 정상 | 주차/차시 필터, 카드뷰 정상 |
| **채점** | `/quiz/:id/grade` GradingDashboard | 정상 | 문항 중심/학생 중심 전환, localStorage 채점 저장 동작 |
| **통계 확인** | `/quiz/:id/stats` QuizStats | 정상 | 학생별 성적 조회, 퀴즈 통계 탭 존재 |
| **편집** | `/quiz/:id/edit` QuizEdit | 부분 구현 | 기본 정보 편집 동작. 단 편집 후 저장 시 mockData 반영 여부 불명확 |

**주요 누락: 임시저장 기능이 버튼만 존재하고 동작하지 않음.**

QuizCreate의 `<button className="btn-secondary text-sm">임시저장</button>`에 onClick 핸들러가 없음. 교수자가 작업 중 이탈하면 내용 전체 소실.

---

### 1-2. 학생 시나리오

| 단계 | 구현 경로 | 정합성 | 이슈 |
|---|---|---|---|
| **퀴즈 목록 확인** | `/` StudentQuizList | 정상 | open/closed 구분, 응시 가능 배지 표시 |
| **응시하기** | `/quiz/:id/attempt` QuizAttempt | 정상 | 타이머, 답안 저장, 자동채점 동작 |
| **재응시** | QuizAttempt + allowAttempts 체크 | 부분 | 횟수 초과 비활성화 동작. 단 재응시 시 이전 답안 불러오기 기능 없음 |
| **결과 확인** | QuizAttempt > 제출 후 result 화면 | 부분 | 자동채점 점수만 표시. 수동채점 대기 안내 있음. 점수 공개 정책 반영 미확인 |
| **성적 조회** | 없음 | **미구현** | 학생이 채점 완료 후 점수를 별도 조회하는 페이지 없음 |

**주요 누락: 학생용 성적 상세 페이지 없음.**

채점 완료 후 학생이 자신의 점수, 정오답, 피드백을 확인하는 독립 화면이 없음. StudentQuizCard에서 최종 점수(자동채점 기준)만 인라인으로 표시됨.

---

### 1-3. 문제은행 시나리오

| 단계 | 구현 경로 | 정합성 | 이슈 |
|---|---|---|---|
| **은행 목록** | `/question-banks` | 정상 | 과목별 은행 카드 목록 |
| **문항 조회/편집** | `/question-banks/:bankId` | 정상 | 유형별 필터, 편집/삭제 동작 |
| **퀴즈에 가져오기** | QuizCreate > 문제은행 모달 | 정상 | 과목 필터, 복수 선택 추가 동작 |
| **은행 간 복사** | QuestionBankList > 가져오기 | 정상 | CopyFromBankModal 동작 |

## 2. 예외 케이스 및 엣지 케이스 목록

| 번호 | 케이스 | 우선순위 | 현재 처리 | 개선안 |
|---|---|---|---|---|
| E-01 | 제한 시간 만료 시 자동 제출 | High | 구현됨 (handleSubmit(true)) | 정상 처리 확인됨 |
| E-02 | 미답변 문항 있는 채로 제출 | Medium | 경고 텍스트만 표시, 제출 허용 | 모달 확인 또는 미답변 강조 표시 필요 |
| E-03 | 재응시 중 브라우저 새로고침 | High | 답안 유실, 응시 기록 없음 | sessionStorage 임시 저장 처리 필요 |
| E-04 | 지각 제출 (마감 후 제출 시도) | High | `allowLateSubmit` 필드만 존재, UI 미처리 | 마감 후 상태 변환 및 지각 제출 수락/거부 흐름 필요 |
| E-05 | 동일 학생이 다른 기기에서 동시 응시 | Medium | 처리 없음 (localStorage 분리) | 실데이터 연동 시 세션 충돌 처리 정책 필요 |
| E-06 | 재응시 시 이전 답안 표시 여부 | Medium | 미처리 (빈 상태로 시작) | Canvas LMS 기준: 이전 답안 표시하되 수정 가능 |
| E-07 | 존재하지 않는 퀴즈 ID로 직접 접근 | High | GradingDashboard는 에러 UI 존재. QuizStats는 mockQuizzes[0] 폴백 처리 | QuizStats에서 퀴즈 없음 처리 일관성 확보 필요 |
| E-08 | 점수 정책 적용 기준 불일치 | High | `getStudentAttempts`에서 policy는 마지막 응시 기록 기준으로 적용. 응시 중 정책 변경 시 소급 적용 가능성 있음 | 정책은 퀴즈 생성 시점 고정, 이후 변경 불가 룰 필요 |
| E-09 | 임시저장 미구현 상태의 이탈 | High | 경고 없음, 작업 내용 소실 | beforeunload 이벤트 + 임시저장 기능 구현 필요 |
| E-10 | 수동채점 완료 후 최종 점수 확정 흐름 | High | 채점 저장은 localStorage에 개별 문항 단위로 존재, 최종 합산 트리거 없음 | "채점 완료" 액션 및 점수 확정 이벤트 필요 |
| E-11 | 문제은행 문항을 퀴즈에 사용 후 은행에서 삭제 시 | Medium | 처리 없음 | 참조 무결성 정책 정의 필요 |
| E-12 | 총 배점이 0점인 퀴즈 발행 | Low | 검증 없음 | 발행 전 총점 > 0 검증 추가 |
| E-13 | 응시 가능 횟수 = 무제한(allowAttempts=-1) 시 학생 뷰 표시 | Low | 횟수 제한 없음으로 처리되어 항상 응시 가능 | "무제한" 텍스트 명시 표시 필요 |
| E-14 | accessCode 설정 퀴즈에서 코드 미입력 접근 | Medium | accessCode 필드 저장만 됨, 실제 접근 차단 로직 없음 | 응시 진입 시 코드 검증 화면 추가 필요 |

## 3. 기능 정책 완결성

### 3-1. 점수 정책

| 항목 | 구현 상태 | 문제점 |
|---|---|---|
| 최고 점수 유지 | 구현 | getStudentAttempts에서 max(totalAutoScore)로 적용. 수동채점 점수는 미반영 |
| 최신 점수 유지 | 구현 | 마지막 응시 반환 |
| 평균 점수 | 구현 | 자동채점 점수 평균만 계산. 수동채점 미포함으로 점수 과소 산정 가능 |
| **수동채점 포함 최종 점수 계산** | **미구현** | 수동채점 완료 후 자동+수동 합산 최종 점수 확정 흐름 없음 |

**정책 공백**: 수동채점 항목이 있는 퀴즈에서 점수 정책을 어떤 기준으로 적용할지 미정의.
예를 들어 학생이 2회 응시했는데, 1차 응시는 수동채점 완료 / 2차 응시는 수동채점 미완료인 경우 "최고 점수"를 어떻게 비교할지 기준이 없음.

---

### 3-2. 재응시 정책

| 항목 | 구현 상태 | 문제점 |
|---|---|---|
| 응시 횟수 제한 (1~3회, 무제한) | 구현 | allowAttempts 필드, StudentQuizCard에서 횟수 초과 비활성화 |
| 재응시 가능 시점 | 미정의 | 이전 제출 직후 즉시 재응시 가능. "이전 채점 완료 후" 등의 조건 없음 |
| 재응시 시 이전 답안 공개 여부 | 미정의 | 코드상 이전 답안 표시 기능 없음 (빈 상태로 시작) |
| 점수 공개 기간과 재응시 연계 경고 | 부분 구현 | 발행 시 경고 모달 존재. 단 경고만 하고 막지는 않음 |

---

### 3-3. 지각 제출 정책

`allowLateSubmit` 필드가 QuizCreate 폼에 존재하나, 실제 적용 로직이 전혀 없음.

- QuizAttempt에서 `quiz.status !== 'open'` 체크만 있고, dueDate 초과 여부 별도 판단 없음
- 마감 후에도 status가 'open'이면 응시 가능
- 지각 마크, 지각 제출 시 감점 정책, 교수자 확인 플로우 없음

Canvas LMS 기준: 지각 제출 시 제출물에 LATE 태그 표시, 교수자 채점 시 인지 가능. 감점은 별도 정책으로 설정.

---

### 3-4. 채점 정책

| 항목 | 구현 상태 | 문제점 |
|---|---|---|
| 자동채점 (객관식/참거짓/수치형 등) | 구현 | autoGradeAnswer 함수 동작 |
| 단답형 자동채점 | 부분 구현 | correctAnswer와 문자열 완전 일치만 체크. 부분 일치, 대소문자 변형 허용 안됨 |
| 서술형 수동채점 | 구현 | localStorage 저장, 점수 입력 UI 존재 |
| 재채점 | UI만 구현 | "재채점" 버튼 클릭 시 모달 표시되나 실제 로직 없음 |
| 채점 완료 후 점수 확정 | 미구현 | 수동채점 저장 → 최종 점수 확정 트리거 없음 |
| 부분 점수 (partial credit) | 미구현 | autoGrade: 'partial' 정의는 있으나 실제 부분 점수 계산 로직 없음 |

## 4. 상태 전이 정합성

### 4-1. 현재 구현된 상태

```
draft → open (발행 버튼)
open → [grading/closed] (실제 전이 로직 미구현, mockData 초기값으로만 존재)
```

### 4-2. 상태 정의 및 문제점

| 상태 | UI 레이블 | 색상 | 문제점 |
|---|---|---|---|
| `draft` | 발행 전 | indigo | 정상 |
| `open` | 진행중 | green | 정상 |
| `grading` | 진행중 | green | open과 동일 레이블/색상. 학생에게도 "응시중"으로 표시됨. 의미 모호 |
| `closed` | 마감 | gray | 정상 |
| `scheduled` | 예정 | amber | STATUS_CONFIG에는 존재하나 실제 발행 플로우에서 생성 불가 |

### 4-3. 누락된 상태 전이

| 전이 | 트리거 | 현재 처리 |
|---|---|---|
| open → grading | dueDate 도과 | 없음. dueDate가 지나도 status는 변경되지 않음 |
| grading → closed | 전체 채점 완료 | 없음. 수동으로 상태 변경 수단 없음 |
| open → closed (즉시 마감) | 교수자 수동 마감 | 없음 |
| draft → scheduled | 예약 발행 날짜 설정 | 없음 |

Canvas LMS 기준: dueDate 도과 시 자동으로 closed 처리, 교수자가 수동으로 개방/마감 가능.

## 5. 권한 설계

### 5-1. 현재 구현

- `RoleContext`를 통해 `role: 'instructor' | 'student'` 전환 지원
- QuizList에서 role에 따라 InstructorQuizList / StudentQuizList 분기
- QuizAttempt에서 `role !== 'student'`면 홈 리다이렉트 처리

### 5-2. 권한 설계 이슈

| 항목 | 이슈 | 우선순위 |
|---|---|---|
| **조교(TA) 역할 없음** | 채점 권한을 조교에게 위임하는 시나리오 미지원 | High |
| **URL 직접 접근 시 권한 처리 불완전** | `/quiz/:id/grade`는 role 체크 없음. 학생이 직접 URL 입력 시 채점 화면 접근 가능 | Critical |
| **문제은행 접근 권한 없음** | 학생이 `/question-banks` 직접 접근 가능. 문제은행은 교수자 전용이어야 함 | High |
| **퀴즈 편집 권한 체크 없음** | `/quiz/:id/edit` URL에 role 체크 없음 | High |
| **발행된 퀴즈 편집 제한 없음** | status='open'인 퀴즈도 편집 화면 진입 가능. 응시 중 문항/설정 변경 시 채점 이슈 발생 | High |
| **다른 교수자 퀴즈 접근 차단 없음** | 과목/교수자 기준 격리 없음 (프로토타입 한계) | Medium |

## 7. MVP 범위 내외 기능 구분 및 우선순위

### 7-1. 현재 MVP1 구현 현황

| 기능 | 구현 완료 | 이슈 |
