## 2. 보안 테스트 (CRITICAL)

### 2.1 XSS 취약점

**이슈 013**: 서술형 답변 XSS (QuizAttempt.jsx)
```
심각도: CRITICAL | CVSS 7.5
위치: QuizAttempt.jsx line 241-250
문제: textarea 값이 그대로 저장, 교수자가 보기 시 스크립트 실행 가능
악의적 입력: <img src=x onerror="alert('XSS')">
권고: DOMPurify.sanitize() 적용
```

**이슈 015**: 채점 대시보드 답변 표시 XSS (GradingDashboard.jsx)
```
심각도: CRITICAL
위치: GradingDashboard의 답변 표시 영역
문제: localStorage 데이터 미검증 렌더링
영향: 교수자가 악의적 학생 답변으로 인한 피해
권고: 모든 사용자 입력에 DOMPurify 적용
```

### 2.2 데이터 접근 제어

**이슈 024**: 학생 데이터 무제한 접근 (전역)
```
심각도: CRITICAL | CVSS 9.0
문제: localStorage의 xnq_student_attempts에 82명 모든 응시 데이터 평문 저장
재현: 콘솔에서 JSON.parse(localStorage.getItem('xnq_student_attempts')) 실행
영향: 학생 A가 학생 B의 답변, 채점 점수 모두 조회 가능
권고: 서버 API로 이동, 역할 기반 접근 제어(RBAC) 구현
```

### 2.3 데이터 보호

**이슈 025**: localStorage 평문 저장
```
심각도: CRITICAL
문제: 모든 응시 데이터, 채점 점수 평문 저장
영향: 기기 분실 시 개인정보 완전 노출
권고: 서버 DB 저장 또는 클라이언트 AES 암호화
```
