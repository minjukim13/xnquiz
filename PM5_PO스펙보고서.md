## 2. 확정 정책

### 재응시 정책
- 이전 응시 제출 완료 후 즉시 재응시 가능
- 무제한(-1) 상한 없음 (실서비스 시 쿨다운 추가 필요)
- 마감 후 시작은 차단, 응시 중 마감 도래 시 해당 건만 유효

### 점수 공개 정책
- 자동채점 점수는 응시 직후 즉시 표시
- 수동채점 포함 최종 점수는 showScore + 공개 기간으로 제어 (MVP2 구현 필요)

### 상태 전이
```
draft → scheduled → open → grading → closed
```
- 현재 draft→open만 구현. 나머지 자동 전이는 MVP2 범위

### 점수 입력 검증 정책 (2026-04-24 확정)

| 항목 | 정책 | 근거 |
|---|---|---|
| **배점 초과 허용** | **하드 블록** (저장 불가) | 보너스/가산 용도는 별도 가산점(Fudge Points) 필드로 분리됨. 문항 점수는 배점 범위 내로 엄격 관리 |
| **음수 점수 허용** | **금지** | 입력 실수 가능성 압도적. 감점은 문항 옵션 레벨에서만 처리 |
| **경고 표시 방식** | **인라인 필드 메시지 + 저장 시 실패 토스트** | 모달 confirm 은 일괄 저장 UX 저해 |
| **일괄 엑셀 업로드** | **개별 입력과 동일 정책 적용** | 경로별 정책 차이는 교수자 혼란 유발 |

### Acceptance Criteria — 점수 입력 검증

**AC-SCORE-01: 개별 점수 입력 시 배점 초과 차단**
- Given: 교수자가 문항 채점 패널에서 점수를 입력한다
- When: 입력값이 문항 배점(maxPoints)을 초과한다
- Then: 저장 버튼이 비활성화되고, 입력 필드 아래 "배점 {maxPoints}점을 초과할 수 없습니다" 인라인 메시지를 표시한다

**AC-SCORE-02: 개별 점수 입력 시 음수 차단**
- Given: 교수자가 문항 채점 패널에서 점수를 입력한다
- When: 입력값이 0 미만이다
- Then: 저장 버튼이 비활성화되고, 입력 필드 아래 "0점 이상 입력해 주세요" 인라인 메시지를 표시한다

**AC-SCORE-03: 일괄 저장 시 유효하지 않은 값 전체 블록**
- Given: 교수자가 학생 중심 탭에서 여러 학생의 점수를 일괄 입력한다
- When: 일괄 저장 버튼을 누른다
- And: 한 건이라도 배점 초과 또는 음수 값이 포함되어 있다
- Then: 저장이 중단되고, 실패 토스트 "N개 항목이 유효하지 않습니다"를 표시한다
- And: 실패 항목 전부를 인라인 메시지로 하이라이트한다
- And: 이미 유효한 항목도 함께 블록된다 (부분 저장 금지 — 감사 추적성 확보)

**AC-SCORE-04: 엑셀 업로드 시 동일 정책 적용**
- Given: 교수자가 엑셀 일괄 채점 모달에서 파일을 업로드한다
- When: 파일 내 점수 컬럼에 배점 초과 또는 음수 값이 하나라도 있다
- Then: 업로드 미리보기 단계에서 해당 행을 빨간색으로 표시한다
- And: "N개 행이 배점 범위(0~{maxPoints})를 벗어납니다" 요약 메시지를 상단에 표시한다
- And: 검증 실패가 있는 한 "적용" 버튼은 비활성화된다

**AC-SCORE-05: 가산점(Fudge Points) 은 검증 정책에서 제외**
- Given: 교수자가 학생 상세 패널의 가산점 팝오버를 연다
- Then: 가산점 입력 필드는 음수 허용, 상한 없음 (운영자 재량)
- And: 입력 필드에 "총점에 가감될 점수 (음수 가능)" 도움말 텍스트를 표시한다

**AC-SCORE-06: 과거 데이터 마이그레이션 — 경고 배너 + 수동 수정 유도**
- Given: 교수자가 채점 대시보드에 진입한다
- When: 저장된 점수 중 새 검증 정책을 위반하는 값(배점 초과 또는 음수)이 1건 이상 발견된다
- Then: 화면 상단에 경고 배너 "점수 정책 변경으로 {N}건의 점수가 현재 배점 범위를 벗어납니다. 해당 항목을 검토 후 수정해 주세요"를 표시한다
- And: 위반 항목의 점수 필드에 빨간색 테두리와 인라인 경고를 표시한다
- And: 자동 클리핑이나 강제 수정은 수행하지 않는다 (감사 추적성 보존)
- And: 교수자가 수동 수정하여 저장하면 경고는 해제된다

### API 응답 타입 계약 (2026-04-24 확정)

**원칙**: PM5 스펙이 API 응답 타입의 단일 진실 공급원 (Single Source of Truth). 백엔드/프론트는 본 계약을 기준으로 구현하고, 변경 시 PM5 리뷰 후 반영.

**퀴즈 요약 객체 (GET /api/quizzes, GET /api/quizzes/{id})**

| 필드 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `id` | string | ✓ | 퀴즈 ID |
| `title` | string | ✓ | 제목 |
| `status` | `'draft' \| 'open' \| 'grading' \| 'closed'` | ✓ | DB 원본 상태 (표시용 전환은 `resolveDisplayStatus` 참조) |
| `questions` | **number** | ✓ | **문항 수 요약** (배열 아님). 상세 문항은 별도 엔드포인트 |
| `totalPoints` | number | ✓ | 총 배점 요약 |
| `startDate` | string \| null | | ISO 형식 |
| `dueDate` | string \| null | | ISO 형식 |
| `gracePeriod` | number | | 분 단위 |
| `allowLateSubmit` | boolean | | |
| `lateSubmitDeadline` | string \| null | | |

**채점 레코드 객체 (점수 검증 대상)**

| 필드 | 타입 | 범위 | 설명 |
|---|---|---|---|
| `score` | number | `0 ≤ score ≤ maxPoints` | 문항/학생 점수. AC-SCORE-01~04 검증 대상 |
| `maxPoints` | number | `> 0` | 배점 |
| `fudgePoints` | number | 무제한 (음수 허용) | 학생 총점 가감. AC-SCORE-05 |

**위반 시 처리**
- 프론트는 타입 불일치 발견 시 콘솔 에러 + Sentry 로깅 (향후 연동)
- 백엔드는 PM5 계약과 다른 형태 반환 시 **버그**로 취급, 프론트는 폴백 로직 최소화 (무분별한 `Array.isArray` 분기 금지)
