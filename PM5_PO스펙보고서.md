## 1. 제품 개요 및 현황

XN Quizzes는 대학 교수자-학습자 간 퀴즈 생성, 배포, 응시, 채점, 통계 확인 전 과정을 지원하는 퀴즈 관리 시스템이다. 현재 React 19 + Vite 기반 프로토타입으로, Mock 데이터 기반 단일 강좌(CS301) 시나리오만 지원한다.

### 구현된 주요 기능 현황

| 기능 영역 | 구현 페이지/컴포넌트 | 구현 수준 |
|---|---|---|
| 퀴즈 목록 조회 | QuizList | 교수자/학습자 역할 분기, 주차/차시 필터 |
| 퀴즈 생성 | QuizCreate | 기본 정보 + 문항 구성 2단계, 발행 즉시 처리 |
| 퀴즈 편집 | QuizEdit | 기본 정보 수정 + 문항 순서/삭제 |
| 학생 응시 | QuizAttempt | 타이머, 자동 제출, 자동채점, 복수 응시 정책 |
| 채점 대시보드 | GradingDashboard | 문항 중심/학생 중심 모드, 수동채점, localStorage 저장 |
| 결과 통계 | QuizStats | 학생별 성적, 퀴즈 통계, 문항 분석(난이도/변별도) |
| 문제은행 | QuestionBankList, QuestionBank | 은행 CRUD, 문항 CRUD, 타 강좌 가져오기 |

### 지원 퀴즈 유형 (12종)

객관식, 참/거짓, 복수 선택, 단답형, 서술형, 수치형, 수식형, 연결형, 다중 빈칸 채우기, 드롭다운 선택, 파일 첨부, 텍스트

## 3. 정책 공백 및 모호 영역 — 룰 확정

### 3-1. 재응시 정책

**현재 구현 상태**: allowAttempts 값(1/2/3/-1)과 점수 정책(최고 점수 유지/최신 점수 유지/평균 점수) 조합으로 처리. localStorage에 응시 이력 저장.

**확정 룰**

| 항목 | 정책 |
|---|---|
| 재응시 가능 시점 | 이전 응시 제출 완료 후 즉시 가능 (현재 구현 유지) |
| 무제한(-1) 상한 | 프로토타입 제한 없음. 실서비스 전환 시 회당 쿨다운(예: 10분) 정책 추가 필요 |
| 재응시 중 마감 도래 | 마감 시각 기준 응시 시작 완료 건만 유효. 마감 후 시작은 차단 |
| 점수 정책 표시 시점 | 응시 결과 화면에 "적용 점수 정책: 최고 점수 유지" 형태로 명시 필요 (현재 미표시) |

### 3-2. 지각 제출 정책

**현재 구현 상태**: QuizCreate에 `allowLateSubmit` 필드 존재하나, QuizAttempt에서 마감 초과 여부 검증 로직 없음.

**확정 룰**

| 항목 | 정책 |
|---|---|
| 지각 제출 허용 여부 | 기본값 불허. 교수자가 "지각 제출 허용" 토글 활성화 시에만 가능 |
| 지각 제출 감점 처리 | 별도 감점률 설정 필드 필요 (예: 마감 후 1일 이내 70% 인정). 미설정 시 100% 인정 |
| 지각 제출 표시 | 채점 대시보드 학생 목록에 "지각 제출" 배지로 구분 |
| 현재 조치 필요 사항 | `allowLateSubmit` 필드는 존재하나 QuizAttempt 단계 검증 로직 부재 → MVP2 우선 처리 |

### 3-3. 자동채점 정확도 정책 (단답형)

**현재 구현 상태**: QUIZ_TYPES에서 단답형의 autoGrade는 `'partial'`로 정의되어 있으나, 실제 채점 로직에서는 `autoGrade: false`처럼 수동채점 대상으로 분류됨. mockData 일부 단답형 문항은 `autoGrade: true`로 혼재.

**확정 룰**

| 항목 | 정책 |
|---|---|
| 단답형 자동채점 기준 | 정답 문자열 완전 일치 + 대소문자 무시. 부분 점수 인정은 MVP3 범위 |
| 서술형 | 자동채점 불가. 항상 수동채점 대상 |
| 자동채점 오류 이의 | 학생 이의제기 기능은 MVP2 이후. 현재는 교수자 수동 재채점으로 처리 |
| 코드 정비 사항 | QUIZ_TYPES의 `'partial'` 값과 실제 채점 로직 간 불일치 해소 필요 |

### 3-4. 점수 공개 정책

**현재 구현 상태**: `showScore`, `scoreRevealStartDate`, `scoreRevealEndDate` 필드 존재. 그러나 QuizAttempt 결과 화면에서 응시 직후 자동채점 점수가 항상 표시됨. 공개 기간 설정이 응시 결과 화면에 반영되지 않음.

**확정 룰**

| 항목 | 정책 |
|---|---|
| 자동채점 점수 즉시 표시 | 응시 직후 표시 (현재 구현 유지) |
| 수동채점 포함 최종 점수 | `showScore` + 공개 기간 설정 기준으로 표시 시점 제어 → MVP2 구현 필요 |
| 오답 풀이 공개 | `showWrongAnswer`, `wrongAnswerRevealTiming` 필드 구현됨. 응시 결과 화면 반영 필요 |

### 3-5. 퀴즈 상태 전이 정책

**현재 구현 상태**: QuizList에서 "발행" 버튼으로 draft → open 전이. grading/closed 상태는 mockData에 하드코딩.

**확정 상태 전이도**

```
draft(초안) → scheduled(예정) → open(진행중) → grading(채점중) → closed(마감)
```

| 전이 | 트리거 | 조건 |
|---|---|---|
| draft → open | 교수자 발행 버튼 | 문항 1개 이상, 시작일/마감일 설정 완료 |
| draft → scheduled | 교수자 발행 버튼 (시작일이 미래) | 위 조건 동일, startDate > 현재 시각 |
| scheduled → open | 시작일 도래 | 시스템 자동 전이 (실서비스 기준) |
| open → grading | 마감일 도래 | 미채점 수동문항 존재 시 자동 진입 |
| open → closed | 마감일 도래 | 수동채점 문항 없는 경우 grading 스킵 |
| grading → closed | 모든 수동채점 완료 | pendingGrade = 0 시 자동 전이 가능 |

### 3-6. 접근 제어 정책

**현재 구현 상태**: RoleContext로 instructor/student 역할 분기. accessCode, ipRestriction 필드는 QuizCreate에 입력 가능하나, QuizAttempt에서 검증 로직 없음. grade/stats/edit 라우트에 권한 가드 없음.

**확정 룰**

| 항목 | 정책 |
|---|---|
| 접근 코드 | 설정 시 QuizAttempt 진입 전 입력 화면 노출. 불일치 3회 초과 시 잠금 |
| IP 제한 | 실서비스 전환 시 서버 사이드 검증 필요. 프로토타입에서는 UI 입력 지원만 |
| 교수자 전용 라우트 보호 | /quiz/:id/grade, /quiz/:id/stats, /quiz/:id/edit 접근 시 학생 역할이면 홈 리다이렉트 |

## 4. Acceptance Criteria (기능별 완료 기준)

### AC-01. 퀴즈 생성

**Given** 교수자가 "/quiz/new" 페이지에 접근했을 때
**When** 기본 정보(제목, 시작일, 마감일)와 문항 1개 이상을 입력하고 발행 버튼을 클릭하면
**Then** 퀴즈가 목록에 "진행중(open)" 상태로 표시되어야 한다

체크리스트:
- [ ] 제목, 시작일, 마감일 미입력 시 발행 버튼 비활성화
- [ ] 마감일이 시작일보다 빠를 경우 발행 불가 처리
- [ ] 문항 0개 상태에서 발행 버튼 비활성화
- [ ] 동일 학생이 복수 추가 기간 설정에 포함될 경우 경고 다이얼로그 노출
- [ ] 임시저장 클릭 시 draft 상태로 저장 (현재 UI 버튼만 존재, 저장 로직 미구현)
- [ ] 시작일이 미래인 경우 "예정(scheduled)" 상태로 발행

### AC-02. 학생 응시

**Given** 학습자가 "open" 상태 퀴즈의 응시 버튼을 클릭했을 때
**When** 제출 버튼을 클릭하거나 타이머가 만료되면
**Then** 자동채점 가능 문항의 점수가 즉시 계산되어 결과 화면에 표시되어야 한다

체크리스트:
- [ ] 타이머 설정 시 남은 시간 실시간 표시
- [ ] 타이머 만료 시 자동 제출 처리
- [ ] 미답 문항 존재 시 제출 전 확인 다이얼로그 노출
- [ ] 자동채점 점수 즉시 표시. 수동채점 대상 문항은 "채점 대기" 표시
- [ ] allowAttempts 초과 시 재응시 버튼 비노출
- [ ] draft/grading/closed 퀴즈 직접 URL 접근 시 "응시 불가" 안내 화면 표시
- [ ] allowLateSubmit = false 상태에서 마감 시각 이후 접근 시 응시 차단

### AC-03. 채점 대시보드 — 문항 중심 모드

**Given** 교수자가 채점 대시보드에 진입하여 수동채점 문항을 선택했을 때
**When** 학생 응답에 점수를 입력하고 저장하면
**Then** 해당 문항 채점 완료로 처리되고, 채점 진행률이 실시간 업데이트되어야 한다

체크리스트:
- [ ] 자동채점 완료 문항과 수동채점 필요 문항 구분 표시
- [ ] 점수 입력 시 배점 초과 불가 처리
- [ ] 미채점 우선/문항 번호순 정렬 전환 가능
- [ ] 채점 내용 저장 후 새로고침해도 유지 (localStorage 기반)
- [ ] 전체 채점 완료 시 "채점 완료" 상태 배너 노출 또는 상태 전환 트리거

### AC-04. 채점 대시보드 — 학생 중심 모드

**Given** 교수자가 채점 모드를 "학생 중심"으로 전환했을 때
**When** 학생을 선택하면
**Then** 해당 학생의 전 문항 응답 및 자동채점 결과가 표시되고 수동채점 문항에 점수 입력이 가능해야 한다

체크리스트:
- [ ] 학생 검색 기능 동작 (이름/학번)
- [ ] 미제출 학생은 회색 처리 또는 "미제출" 배지 표시
- [ ] 학생별 자동채점 합산 + 수동채점 입력값 합산 총점 실시간 표시

### AC-05. 결과 통계

**Given** 교수자가 채점 완료된 퀴즈의 결과 통계 페이지에 접근했을 때
**When** "퀴즈 통계" 탭을 클릭하면
**Then** 점수 분포 차트, 문항별 평균 점수, 난이도, 변별도, 점이연 상관계수가 표시되어야 한다

체크리스트:
- [ ] 채점 미완료 상태에서 통계 접근 시 "채점 진행 중" 안내 표시
- [ ] 엑셀 다운로드 (학생별 성적표, 문항 분석표) 각각 동작
- [ ] 학생별 성적 목록 검색/정렬 동작

### AC-06. 문제은행

**Given** 교수자가 문제은행에서 새 문제은행을 생성했을 때
**When** 문항을 추가하고 퀴즈 생성/편집 화면에서 "문제은행에서 가져오기"를 선택하면
**Then** 해당 은행의 문항이 퀴즈 문항 목록에 추가되어야 한다

체크리스트:
- [ ] 문제은행 삭제 시 해당 은행의 모든 문항 함께 삭제
- [ ] 문항 유형별 필터 동작
- [ ] 난이도, 그룹 태그 필터 동작
- [ ] 타 강좌 문제은행에서 가져오기(복사) 동작

## 6. KPI 지표

### [현재 측정 가능] 프로토타입/목업 단계에서 확인 가능한 지표

| 지표 | 측정 방법 | 목표 기준 |
|---|---|---|
| 채점 완료율 | (graded / submitted) × 100 | 100% (마감 후 7일 이내) |
| 미채점 문항 잔존 건수 | pendingGrade 합계 | 0건 (채점 종료 시점 기준) |
| 문항 유형별 사용 비율 | 생성된 문항 유형 분포 | 서술형/단답형이 전체의 30% 이상 |
| 응시율 (모의) | (submitted / totalStudents) × 100 | 목표 95% 이상 |
| 평균 점수 분포 | avgScore 필드 기준 | 총점의 60~80% 구간 |
| 문항별 난이도 분포 | 쉬움/보통/어려움 비율 (QuizStats 문항 분석 탭) | 보통 50%, 쉬움/어려움 각 25% 권장 |
| 문항별 변별도 | discrimination 값 | 0.3 이상 문항 비율 80% 이상 |

### [실데이터 연동 후 측정 가능] API/DB 연동 이후에만 수집 가능한 지표

| 지표 | 측정 방법 | 목표 기준 |
|---|---|---|
| 실제 응시율 | 응시 완료 학생 수 / 수강 등록 학생 수 | 95% 이상 |
| 재응시율 | allowAttempts > 1 퀴즈에서 2회 이상 응시 비율 | 모니터링 후 정책 조정 |
| 평균 풀이 시간 | timeTaken 평균값 (제한시간 대비 비율) | 제한시간의 70% 이내 |
| 채점 소요 시간 | 채점 시작 ~ 완료 기준 경과 시간 | 수동채점 문항당 평균 1분 이내 |
| 문제은행 재사용률 | 문제은행 출제 문항 수 / 전체 출제 문항 수 | 50% 이상 |
| 퀴즈 생성 완료율 | 생성 시작 후 발행 완료까지 이탈 없이 완료한 비율 | 90% 이상 |
| 엑셀 다운로드 사용률 | 채점 완료 퀴즈 중 엑셀 다운로드 발생 비율 | 모니터링 기준 수립 |
| 학생 이의제기 건수 | 채점 완료 후 이의제기 접수 건수 (MVP2 이후) | 전체 응시의 1% 미만 |

## 8. 리스크 및 즉시 처리 권고 항목

| 리스크 | 심각도 | 현황 | 권고 조치 |
|---|---|---|---|
| localStorage 기반 채점 저장 — 브라우저 초기화 시 데이터 소실 | Critical | 현재 유일한 저장 수단 | 서버 저장 전환 또는 엑셀 내보내기 백업 기능 보완 |
| 지각 제출 검증 로직 미구현 | High | `allowLateSubmit` 필드만 존재 | QuizAttempt에 마감 시각 검증 로직 추가 |
| 단답형 autoGrade 정책 불일치 | High | QUIZ_TYPES는 `'partial'`, 실제 동작은 수동채점 처리 | 정책 일원화 또는 부분 점수 로직 구현 |
| 복수 강좌 미지원 | High | CS301 단일 강좌 하드코딩 | mockData 구조에서 courseId 기반 다강좌 확장 설계 필요 |
| 라우트 직접 접근 시 권한 미검증 | Medium | QuizAttempt만 처리됨 | /grade, /stats, /edit 라우트에 교수자 권한 가드 추가 |
| 접근 코드/IP 제한 미작동 | Medium | UI 입력만 존재 | 응시 진입 단계에 검증 로직 구현 필요 |
