## 2. 정책 공백 식별 및 룰 확정

코드 직접 분석으로 확인한 정책 미정의 영역과 확정 룰을 기술합니다.

### 2-1. 퀴즈 발행 정책

**현황**: `QuizCreate.jsx:142` — 발행 시 상태가 'open'으로 즉시 변경되나, 문항 0개·배점 0점·시작일 > 마감일 상태에서도 발행이 가능합니다.

**확정 룰**

| **조건** | **정책** |
|---|---|
| 문항 수 | 발행 시 문항 최소 1개 이상 필수. 미충족 시 발행 차단 + "문항을 1개 이상 추가하세요" 안내 |
| 총점 | 총점 1점 이상 필수. 0점 상태 발행 차단 |
| 날짜 유효성 | 마감일이 시작일 이전인 경우 저장 차단. 당일 즉시 발행 허용 |
| 발행 후 수정 | 응시 시작 전: 전면 수정 허용. 응시 중: 문항 삭제/배점 변경 차단, 문구 수정만 허용 |
| 초안(draft) 상태 | 학습자에게 미노출. 교수자 본인만 조회 가능 |

### 2-2. 응시 횟수 및 재응시 점수 정책

**현황**: `QuizCreate.jsx:19~25` — 응시 횟수 옵션(1회/2회/3회/무제한)과 점수 정책(최고/최신/평균) UI는 존재하나, 실제 적용 로직 미구현.

**확정 룰**

| **항목** | **정책** |
|---|---|
| 기본 응시 횟수 | 1회 (변경 가능) |
| 기본 점수 정책 | 최고 점수 유지 |
| 재응시 시 이전 답안 | 초기화 후 새 응시 시작 (이전 답안 열람 불가) |
| 재응시 허용 시점 | 직전 응시 제출 직후 즉시 허용 (별도 대기 시간 없음) |
| 소급 적용 | 점수 정책 변경 시 기 응시 점수에 소급 적용하지 않음, 이후 응시분부터 적용 |
| 교수자 예외 허용 | 교수자가 특정 학생에게 추가 응시 기회 부여 가능 (MVP2) |

### 2-3. 지각 제출 정책

**현황**: `QuizCreate.jsx:72` — `allowLateSubmit: false` 기본값으로 UI 옵션 존재. 그러나 지각 허용 기간 설정 항목 미구현.

**확정 룰**

| **항목** | **정책** |
|---|---|
| 기본값 | 지각 제출 불허 (마감 후 제출 차단) |
| 지각 허용 시 | 교수자가 지각 허용 기간 직접 설정 (예: 마감 후 24시간) |
| 지각 점수 처리 | 선택 정책: 감점 없음 / 10% 감점 / 50% 감점. 기본: 감점 없음 |
| 자동 제출 | 제한 시간 종료 시 자동 제출 처리 (현재 코드의 DEFAULT_NOTICE에 "자동 제출되지 않습니다" 문구가 포함되어 있으나, 실서비스 전환 시 자동 제출 정책으로 변경 필수) |

### 2-4. 채점 공개 정책

**현황**: QuizStats는 상태 제한 없이 접근 가능. `closed` 상태 퀴즈에서만 "결과 보기" 버튼이 활성화되나, URL 직접 접근 시 채점 중 상태에서도 통계 노출됨.

**확정 룰**

| **항목** | **정책** |
|---|---|
| 학습자 점수 공개 시점 | 교수자가 채점 완료 후 수동으로 "점수 공개" 처리 또는 전체 채점 완료 시 자동 공개 중 선택 |
| 부분 채점 시 통계 | 교수자는 채점 진행 중에도 통계 조회 가능. 학습자는 전체 채점 완료 후에만 조회 가능 |
| 모범 답안 공개 | 점수 공개와 별도로 교수자가 수동 설정. 기본: 비공개 |
| 미제출 학생 점수 | 기본: 결시 처리 (성적부에 '-' 표시). 교수자가 0점으로 수동 변환 가능 |

### 2-5. 단답형(short_answer) 자동채점 정책

**현황**: `mockData.js:6` — `short_answer: { autoGrade: 'partial' }`. 그러나 `GradingDashboard.jsx:43` — `!q.autoGrade` 조건으로 short_answer가 수동채점 목록에 포함됩니다. 정책 불일치 확인.

**확정 룰**

| **항목** | **정책** |
|---|---|
| 단답형 자동채점 기본값 | 비활성화 (수동채점 대상으로 분류). autoGrade='partial' 삭제, false로 통일 |
| 자동채점 활성화 조건 | 교수자가 "정답 키워드"를 1개 이상 등록한 경우에 한해 키워드 포함 여부 자동 판별 |
| 키워드 매칭 방식 | 대소문자 구분 없음, 띄어쓰기 무시. 교수자가 정답 동의어 복수 등록 가능 |
| 부분 점수 | 단답형 부분 점수 허용 (예: 키워드 일부 포함 시 50% 부여) — MVP2 |

**현황**: `GradingDashboard.jsx:182` — 재채점 버튼 및 모달 UI 존재. 실제 재채점 로직 미구현.

## 3. Acceptance Criteria

각 기능이 "개발 완료"로 인정되는 조건을 정의합니다.

### AC-01: 퀴즈 생성

**Given** 교수자가 퀴즈 생성 화면에 접속했을 때
**When** 필수 항목(제목, 시작일, 마감일, 문항 1개 이상, 총점 1점 이상)을 입력하고 "발행하기"를 클릭하면
**Then**
- [ ] 퀴즈 목록에 신규 퀴즈가 'open' 상태로 노출된다
- [ ] 시작일 이전에는 'scheduled' 상태로 노출된다 (MVP2)
- [ ] 문항 0개 또는 총점 0점 상태에서는 "발행하기" 버튼이 비활성화된다
- [ ] 마감일이 시작일보다 이전인 경우 인라인 오류 메시지가 표시되고 저장이 차단된다
- [ ] "임시저장" 클릭 시 'draft' 상태로 저장되고 목록에 노출된다

### AC-02: 퀴즈 편집 (발행 후)

**Given** 교수자가 발행된 퀴즈의 편집 화면에 접속했을 때
**When** 퀴즈 정보 또는 문항을 수정하고 저장하면
**Then**
- [ ] 수정 내용이 퀴즈 목록 및 채점 화면에 즉시 반영된다
- [ ] 응시 중인 학생이 있는 경우 문항 삭제 및 배점 변경 시 경고 모달이 표시된다
- [ ] 문항 순서 변경(드래그)이 저장 시 반영된다

### AC-03: 채점 대시보드 — 수동 채점 저장

**Given** 교수자가 문항 중심 또는 학생 중심 채점 모드에서 점수를 입력했을 때
**When** "저장" 또는 "다음" 버튼을 클릭하면
**Then**
- [ ] 입력한 점수가 해당 학생의 해당 문항 점수로 저장된다
- [ ] 해당 학생의 총점이 자동채점 점수 + 수동채점 점수의 합으로 재계산된다
- [ ] 전체 학생 채점 완료 시 퀴즈 상태가 'grading' → 'closed'로 자동 전환된다
- [ ] 저장 없이 페이지 이탈 시 "저장되지 않은 변경 사항이 있습니다" 확인 모달이 표시된다
- [ ] 배점 초과 점수 입력 시 오류 메시지가 표시되고 저장이 차단된다

### AC-04: 채점 대시보드 — 자동채점 결과 표시

**Given** 학생이 자동채점 문항에 응시하여 제출했을 때
**When** 교수자가 채점 대시보드에 접속하면
**Then**
- [ ] 자동채점 문항(객관식, 참/거짓, 수치형 등)의 점수가 정답 기준에 따라 자동으로 계산된다
- [ ] 자동채점 문항은 채점 목록에서 "자동채점 완료" 상태로 구분 표시된다
- [ ] 수동채점 필요 문항만 채점 대기 목록에 노출된다

### AC-05: 퀴즈 통계 — 성적 분포 및 문항 분석

**Given** 교수자가 채점이 완료된 퀴즈의 통계 화면에 접속했을 때
**When** 학생별 성적 조회 탭 또는 퀴즈 통계 탭을 확인하면
**Then**
- [ ] 전체 학생의 점수 분포(히스토그램)가 표시된다
- [ ] 평균, 표준편차, 최고점, 최저점이 표시된다
- [ ] 문항별 평균 점수 및 정답률이 표시된다
- [ ] 성적 데이터를 CSV로 다운로드할 수 있다
- [ ] 학생 이름/학번으로 검색 및 필터링이 가능하다

### AC-06: 문제은행 — 문항 등록 및 퀴즈 연동

**Given** 교수자가 문제은행에서 문항을 관리할 때
**When** 문항을 추가, 편집, 삭제하거나 퀴즈에 불러오면
**Then**
- [ ] 신규 문항 추가 시 유형, 배점, 소속 문제은행 필수 입력 후 저장된다
- [ ] 객관식 유형 문항 저장 시 선택지 최소 2개 이상 필수 검증된다
- [ ] 문제은행에서 퀴즈로 불러온 문항은 문제은행 수정 시 연동 퀴즈에 영향 여부를 교수자가 선택할 수 있다 (독립 복사 vs 연동)
- [ ] 문항 사용 횟수(usageCount)가 표시된다
- [ ] 문제은행 삭제 시 해당 은행에 속한 문항 처리 방안(이동 또는 삭제)을 선택할 수 있다

## 4. 우선순위 매트릭스

| 학습자 응시 화면 구현 | 상 | 상 | **9** | Must-have (MVP2) |

## 5. MVP 범위 확인 및 Out-of-scope 명시

### MVP1 — 현재 프로토타입 (완료 기준 재정의)

MVP1은 교수자 단독 사용 기준의 퀴즈 관리 도구입니다. 현재 UI/UX는 완성에 가깝지만, 데이터 영속성(저장/API)이 누락된 상태입니다.

**MVP1 완료 조건**

- [ ] 퀴즈 생성, 편집, 발행 — 저장 시 상태 영속
- [ ] 채점 대시보드 — 점수 저장 및 총점 자동 계산
- [ ] 퀴즈 상태 전이 자동화 (draft → open → grading → closed)
- [ ] 발행 유효성 검증 (문항 수, 배점, 날짜)
- [ ] 성적 CSV 다운로드
- [ ] 문제은행 문항 CRUD 저장

**현재 미완료 항목** (코드 분석 기반)

| **항목** | **근거** |
|---|---|
| 채점 저장 | GradingDashboard에 저장 버튼 없음, localStorage 기반 임시 구현만 존재 |
| 퀴즈 생성 저장 | QuizCreate.jsx:142 — `mockQuizzes.push()` (메모리 한정, 새로고침 시 소실) |
| 퀴즈 편집 저장 | QuizEdit.jsx:58 — `mockQuizzes[idx] = {...}` (메모리 한정) |
| 상태 전이 트리거 | 상태 변경 UI 없음 (grading→closed 전환 불가) |
| 발행 유효성 검증 | isFormValid = title + 날짜만 체크 (문항 수, 총점 미체크) |
| 문제은행 저장 | QuestionBank의 추가/편집이 로컬 state에만 반영 |

### MVP2 — 학습자 응시 화면 + 알림 정책 (다음 스프린트 목표)

- 학습자 응시 화면 (문제 풀기, 시간 제한, 자동 제출)
- 채점 완료 후 학습자 점수 공개 흐름
- 재응시 흐름 (횟수 제한, 점수 정책 적용)
- 재채점 학생 알림

### MVP3 — Canvas LMS 연동 (중장기 목표)

- Canvas API 연동 (퀴즈 데이터 동기화)
- 학교 SSO 인증 (교수자/학습자 로그인)
- 학습자용 성적 조회 포털
- 학과/강좌 단위 통계 리포트

### Out-of-scope (현 단계 제외 항목)

- AI 기반 서술형 자동채점
- 동영상/파일 첨부 문항 응시 시 스트리밍
- 오픈북 시험용 외부 리소스 제한 (브라우저 락다운)
- 성적 이의신청 워크플로우
- 다국어(영어) 지원

## 자체 리뷰 (PO 관점)

**잘 설계된 부분**
- 채점 대시보드의 문항 중심 / 학생 중심 이중 모드는 교수자 업무 패턴을 정확히 반영한 차별화 포인트
- mockData의 autoScores + manualScores 분리 구조는 실데이터 전환 시 모델 확장이 용이함
- 퀴즈 유형 11종 정의(QUIZ_TYPES)는 Canvas LMS 스펙과 호환되어 연동 시 데이터 마이그레이션 비용 최소화

**우선 해결이 필요한 리스크**
- **데이터 영속성 부재**: 모든 저장이 메모리 한정. MVP1 완료 기준에서 가장 중요한 갭
- **상태 전이 트리거 부재**: 교수자가 퀴즈를 grading → closed로 수동 전환하는 수단이 없음. 채점 완료 시나리오 자체가 데모 외에는 불가능
- **DEFAULT_NOTICE의 "자동 제출 안 됨" 문구**: 실서비스 전환 시 반드시 자동 제출 정책으로 변경 필요. 현재 문구 그대로 배포 시 학생이 시간 초과 후 미제출 상태로 0점 처리되는 사고 가능성
