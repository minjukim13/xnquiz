## 1. 퀴즈 타입

Canvas Classic Quizzes는 4가지 퀴즈 타입을 지원한다. XN 퀴즈는 성적 반영 퀴즈(Graded Quiz)를 완전 구현한 상태이며, 연습 퀴즈와 설문 타입은 현 단계에서 구현 범위에 포함하지 않는다.

| Canvas 타입 | 설명 | XN 현황 |
|------------|------|---------|
| **성적 반영 퀴즈** | 성적 반영, 자동/수동 채점 | **구현 완료** |

---

## 2. 퀴즈 설정

### 2.1 구현 완료된 설정

| Canvas 설정 | XN 구현 내용 | 비고 |
|------------|-------------|------|
| 제한 시간 | 분 단위 시간 제한 + 무제한 옵션 | 시간 만료 시 자동 제출 |
| 선택지 섞기 | 학생마다 선택지 순서를 랜덤 배치 | |
| 문항 순서 섞기 | 학생마다 문항 순서를 랜덤 배치 | |
| 재응시 횟수 | 1~99회 또는 무제한 | |
| 복수 응시 시 점수 산출 | 최고 점수 / 최신 점수 / 평균 점수 중 선택 | |
| 성적 공개 | 공개 범위(오답만/정답 포함), 공개 시점(즉시/마감 후/기간 설정) | |
| 기간 제한 정답 공개 | 특정 기간에만 정답 공개 | 시작일/종료일 설정 |
| 날짜 설정 | 시작일, 마감일, 이용 종료일 | |
| **결과 조회 제어** | 성적 공개 정책 내 "1회만 공개" 토글 | **신규 구현** |
| **1회 열람** | 제출 후 1회만 결과 확인 가능, 이후 비공개 | **신규 구현** |

**결과 조회 제어 + 1회 열람 구현 방식:**

Canvas는 결과 조회 제어를 별도 설정(Always/Once/Until/Never)으로 제공한다. XN 퀴즈에서는 기존 성적 공개 정책에 자연스럽게 통합하여 설정 간 혼동을 방지했다.

| Canvas 옵션 | XN 대응 방식 |
|------------|-------------|
| 항상 비공개 (Always) | 성적 공개 토글을 끔 (기존 기능) |
| 특정 날짜까지 비공개 (Until) | 공개 시점을 "기간 설정"으로 선택하고 시작일만 지정 (기존 기능) |
| 1회만 공개 (Once) | "1회만 공개" 토글 활성화 (신규). 공개 시점과 독립적으로 조합 가능 |
| 제한 없음 (Never) | 성적 공개 토글 켬 + 1회만 공개 끔 (기본값) |

- 1회 열람 추적은 응시별로 독립 관리 (재응시 시 새 응시 결과는 다시 1회 열람 가능)
- 결과 화면에 안내 배너 표시: 첫 열람 시 "1회만 열람 가능" 경고, 재접근 시 "이미 열람한 결과" 안내

### 2.2 미구현된 설정

| Canvas 설정 | 동작 설명 | 구현 순위 |
|------------|----------|----------|
| **한 문항씩 표시** | 한 번에 하나의 문항만 표시, 이전/다음 버튼으로 이동 | **5순위 (필수)** |
| **답변 후 문항 잠금** | 위 설정 활성화 시 추가 옵션. 답변 후 이전 문항으로 돌아갈 수 없음 | **5순위 (필수, 동시)** |
| **차등 날짜 배정** | 특정 학생/그룹에 다른 마감일, 시작일, 종료일 설정 | **6순위 (필수)** |

## 4. 응시 관리 (Moderate) -- XN 고유 기능으로 대체

Canvas의 Moderate 기능은 교수자가 개별 학생의 응시 조건을 조정하는 기능이다. XN 퀴즈에서는 **조건부 재응시**와 **차등 날짜 배정** 조합으로 대체한다.

| Canvas 기능 | XN 대체 방안 | 상태 |
|------------|-------------|------|
| 학생별 추가 시도 부여 | 조건부 재응시 (점수 미달/미응시 자동 선별 + 일괄 부여) | **대체 완료** |
| 학생별 잠금 해제 | 차등 날짜 배정으로 해당 학생의 종료일을 개별 연장 | **6순위 구현 시 대체** |
| 학생별 추가 시간 | 조건부 재응시 시 시간 재설정으로 대체 | **대체 가능** |
| 미제출 퀴즈 수동 제출 | 시간 만료 시 자동 제출로 대체 | **대체 가능** |

## 10. XN 퀴즈 고유 기능 (Canvas 대비 차별점)

Canvas Classic Quizzes에는 없지만 XN 퀴즈에만 있는 기능들이다.

| 기능 | 설명 | 가치 |
|------|------|------|
| **문항 중심 채점 모드** | 한 문항에 대해 전체 학생 답안을 나열하며 일괄 채점 | 서술형 일괄 채점 효율 극대화 |
| **조건부 재응시** | 점수 미달/미응시 학생을 자동 선별하여 일괄 재응시 부여 | 대규모 수업에서 개별 설정 부담 감소 |
| **복수선택 채점 전역 설정** | 부분점수/감점 정책 3종 | Canvas는 전부 맞아야 정답 처리만 지원 |
| **대소문자 구분 전역 설정** | 단답형 대소문자 구분 전역 제어 | Canvas는 문항별 개별 설정만 가능 |
| **채점 진행률 실시간 표시** | 문항별/학생별 채점 완료 비율 대시보드 | 대규모 수업 채점 현황 파악 |
| **엑셀 일괄 채점 업로드** | 엑셀 파일로 점수 일괄 입력 | 오프라인 채점 후 업로드 |
| **주차/차시 필터** | 퀴즈를 주차/차시 기준으로 분류 및 필터링 | 한국 대학 수업 구조에 최적화 |
| **PDF 시험지/답안지 출력** | 시험지 및 학생 답안지 PDF 다운로드 | 오프라인 시험 겸용 |

## 11. 잔여 구현 사항 (5순위~11순위)

### [5순위] 한 문항씩 표시 + 답변 후 문항 잠금 -- 필수

**의존성**: 없음

| 항목 | 설명 |
|------|------|
| 한 문항씩 표시 | 퀴즈 설정에 토글 추가. 활성화 시 문항을 1개씩만 표시하고, 이전/다음 네비게이션 버튼과 문항 번호 인디케이터 제공 |
| 답변 후 문항 잠금 | 위 설정 활성화 시 추가 옵션. "다음" 클릭 후 이전 문항으로 돌아갈 수 없게 잠금. 시험 모드에서 부정행위 방지 효과 |

### [6순위] 차등 날짜 배정 (Assign To) -- 필수

**의존성**: 없음 (복잡도 높음)

| 항목 | 설명 |
|------|------|
| 차등 날짜 설정 | 특정 학생 또는 그룹에 다른 마감일, 시작일, 종료일 설정 |
| 참고 | 이전에 구현했다가 제거된 기능. 기존 구현 이력 참고 가능 |

### [7순위] 문항별 코멘트 3종 -- 권장

**의존성**: 결과 조회 제어(4순위) 이후 -- **해소됨**

| 항목 | 설명 |
|------|------|
| 정답 코멘트 | 학생이 정답을 맞혔을 때 표시되는 피드백 |
| 오답 코멘트 | 학생이 오답일 때 표시되는 피드백 |
| 공통 코멘트 | 정답 여부와 관계없이 항상 표시되는 피드백 |
| 결과 공개 연동 | 성적 공개 및 1회만 공개 설정에 따라 조건부 표시 |

### [8순위] 답변 자동 저장 -- 권장

**의존성**: 5순위(한 문항씩 표시) 이후 -- 단일 문항 모드에서의 저장도 함께 처리

| 항목 | 설명 |
|------|------|
| 주기적 저장 | 30초~2분 간격으로 현재 응답을 자동 저장 |
| 복구 | 브라우저 충돌이나 새로고침 시 저장된 응답 복구 |
| 서버 전환 대비 | 저장 로직을 분리하여 향후 서버 연동 시 교체 용이하게 설계 |

### [9순위] 마감 후 미제출자 일괄 자동 제출 -- 권장

**의존성**: 유예 시간(10순위)이 마감 기준을 변경할 수 있으므로, 마감 시점 판단을 별도 함수로 분리

| 항목 | 설명 |
|------|------|
| 자동 제출 | 마감일 경과 후 미제출 상태인 학생의 응시를 자동으로 제출 처리 |
| 마감 판단 분리 | 마감 시점 판단 로직을 별도 함수로 분리하여 유예 시간 추가에 대비 |

### [10순위] 유예 시간 (Grace Period) -- 선택

**의존성**: 9순위의 마감 판단 함수에 반영

| 항목 | 설명 |
|------|------|
| 유예 시간 설정 | 퀴즈 설정에 분 단위 유예 시간 필드 추가 |
| 적용 범위 | 마감일 + 유예 시간 이내 제출은 지각으로 처리하지 않음. 지각 제출 검증과 마감 후 자동 제출 모두에 반영 |

### [11순위] 활동 로그 (Quiz Log) -- 선택

**의존성**: 없음

| 항목 | 설명 |
|------|------|
| 추적 대상 | 응시 시작 시간, 각 문항 이동 시간, 답변 변경 기록, 포커스 이탈(탭 전환) |
| 교수자 조회 | 채점 대시보드에서 학생별 활동 로그 조회 가능 |
| 활용 | 부정행위 탐지, 응시 행동 분석 |
