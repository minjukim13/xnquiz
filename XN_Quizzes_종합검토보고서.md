## 1. 각 PM 보고서 요약

### PM1 — 디자인

Toss 스타일 기반 shadcn/radix UI 21개 컴포넌트로 전면 교체 완료. 색상 시스템은 oklch CSS 변수 기반 시맨틱 토큰으로 통일, WCAG AA 색상 대비도 수정 완료. **잔존**: 모바일 내비게이션 대안 부재(Critical 1건), 일부 하드코딩 hex(Low).

### PM2 — 기획

핵심 Happy Path(생성-발행-응시-채점-통계) 정상 동작. 권한 가드 6개 페이지 추가, 임시저장/데이터 영속화 구현 완료. **잔존**: 지각 제출 검증 로직 없음, 상태 자동 전이 미구현, 학생 성적 상세 페이지 없음, 알림 전면 부재.

### PM3 — 개발

Critical 변이 버그 해결(localStorage 영속화), xlsx 청크 분리 완료, 재채점 로직 4개 옵션 구현. **잔존**: GradingDashboard 1,300줄 단일 파일, mockData.js 2,400줄 미분리, localStorage JSON 무검증.

### PM4 — QA

테스트 통과율 70% → **87%** (55/63). Critical/High 전건 해결. **잔존 8건**: 이전 답안 확인 불가, 뒤로가기 상태 초기화, 가상 스크롤 미적용, mockData 미분리, localStorage 평문 저장 등 Medium/Low 수준.

### PM5 — PO

MVP1 기능 전수 완료. 정책 확정 (재응시/점수 공개/상태 전이). 재채점 로직 구현 완료. **잔존**: 지각 제출 검증, 채점 완료 확정 흐름, 단답형 autoGrade 불일치.

## 4. MVP 로드맵

### MVP1 — 완료

퀴즈 생성/편집/발행, 학생 응시(타이머/자동 제출), 자동/수동 채점, 재채점, 복수 응시+점수 정책, 결과 통계, 문제은행 CRUD, 엑셀 다운로드, 역할 기반 접근 제어, WCAG AA 접근성

### MVP2 — 다음 스프린트 우선순위

| 순위 | 기능 | Effort |
|---:|------|--------|
| 1 | 지각 제출 검증 로직 | 하 |
| 2 | 채점 완료 확정 흐름 | 중 |
| 3 | 상태 자동 전이 (open→grading→closed) | 중 |
| 4 | 학생용 성적 상세 페이지 | 중 |
| 5 | 모바일 내비게이션 (하단 탭 또는 햄버거) | 중 |

### MVP3 — 실데이터 전환 전 준비

mockData.js 분리, localStorage→API 전환, 가상 스크롤 적용, GradingDashboard 컴포넌트 분리

### Out-of-scope

AI 서술형 자동채점, LTI/Canvas 실연동, 실시간 감독, 다국어, TrustLock
