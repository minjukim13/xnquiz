## 1. PM1~PM5 요약

### PM1 — 디자이너 (검토일: 2026-03-27)

색상 시스템 혼용이 전체 프로젝트에 산재합니다. `#9E9E9E` 텍스트 색상이 WCAG AA 기준(4.5:1)을 미달하는 케이스가 40개소 이상 존재하며, Primary 색상이 `#4f46e5`(indigo-600)와 `#6366f1`(indigo-500) 혼재, 포커스 스타일 미정의 등 접근성 이슈가 Critical 수준입니다. 컴포넌트 일관성은 양호하나, 모바일 Nav 미구현과 탭·카드의 hover/focus 상태 불일치가 High 이슈로 남아 있습니다.

### PM2 — 서비스 기획자 (검토일: 2026-03-26)

교수자 퀴즈 생성→채점 Happy Path는 기본 동작하나, 채점 완료 후 상태 전환 트리거 부재(grading→closed), 임시저장 미구현이 Critical 수준입니다. 학습자 응시 화면은 이후 구현 완료(MVP1 달성). 지각 제출 정책, 재채점 정책, 소수점 점수 정책 등 다수의 정책이 미정의 상태입니다.

### PM3 — 시니어 개발자 (검토일: 2026-03-26)

번들 크기 총 ~208 KB(gzip)로 양호합니다. lazy loading + Suspense 적용, manualChunks 함수 형식 준수 등 핵심 성능 기반은 적절합니다. 주요 우려사항: `getLocalGrades()` 반복 호출(최대 328회 JSON.parse), 상태 관리가 컴포넌트 로컬+전역 혼재, 모듈 레벨 mockData 변수로 인한 새로고침 시 상태 소실. API 전환 시 교체 범위가 mockData.js 전체에 산재하여 추출 전략 수립이 필요합니다.

### PM4 — QA 엔지니어 (검토일: 2026-04-03)

전수 정적 분석 기준 총 25건 이슈 도출. Critical 2건은 이번 스프린트 내 해소(BUG-01 timeLimit=0, BUG-02 엑셀 채점 미반영). High 8건 중 5건 해소(BUG-03 NaN, BUG-05 횟수 초과, BUG-06 draft 접근, BUG-09 choices 미저장). 미해소 High 3건(BUG-04 multiple_answers UI, BUG-07 role 접근 제어, BUG-08 임시저장)이 잔존합니다.

### PM5 — 시니어 PO (검토일: 2026-04-03)

학습자 응시 화면 완성, 정오답 공개 설정 연동 구현으로 MVP1 핵심 기능은 대부분 UI 수준에서 완성되었습니다. 데이터 영속성 부재(새로고침 시 상태 소실)와 퀴즈 상태 전이 트리거 부재가 MVP1 완료를 막는 최우선 갭입니다. QuizEdit에서 정오답 공개 설정값이 미반영되는 문제가 방금 구현된 4-1 기능의 일관성을 훼손하는 신규 이슈로 확인됩니다.

## 3. Critical — 긴급 수정 필요

### C-01. 새로고침 시 상태 소실

**증상**: `mockQuizzes`, `mockStudents`가 모듈 레벨 변수로 메모리에만 존재. 페이지 새로고침 시 발행한 퀴즈·변경한 상태 전부 초기값으로 복원.

**영향**: MVP1 완료 기준의 핵심 갭. 데모 시 새로고침만 해도 작업 전체 소실.

**권고 처리 방향**: localStorage 기반 퀴즈 영속화 (mockQuizzes → localStorage 저장·로드). 실데이터 API 전환 전까지 임시 대안으로 충분.

---

### C-02. 상태 전이 트리거 없음

**증상**: 교수자가 채점을 완료해도 퀴즈 상태를 `grading` → `closed`로 전환할 방법이 없음. "채점 완료" 버튼 또는 자동 전환 로직 전무.

**영향**: 채점 완료 시나리오 자체가 데모 외에 불완전. 학습자 결과 공개 흐름도 연동 불가.

**권고 처리 방향**: GradingDashboard에 "채점 완료" 버튼 추가. 클릭 시 `quiz.status = 'closed'` 전환 + 확인 다이얼로그.

---

### C-03. 접근성 (WCAG AA 미달)

**증상**: 프로젝트 전반 `#9E9E9E` 텍스트가 흰 배경 대비비 2.85:1로 WCAG AA 기준(4.5:1) 미달. 포커스 링 미정의.

**영향**: 시각 장애 사용자 접근 불가. 실서비스 배포 시 법적 리스크.

**권고 처리 방향**: 보조 텍스트 색상을 `#757575`(대비비 4.54:1) 이상으로 일괄 변경. 포커스 링을 `outline: 2px solid #4f46e5` 등으로 전역 정의.

## PM 간 이견 조율

**PM2 vs PM3 — 임시저장 방식**: PM2는 서버 저장을 권고하나, PM3는 현재 아키텍처 상 localStorage 임시저장이 현실적임을 지적합니다. **결론**: 프로토타입 단계에서는 localStorage로 구현 후 API 전환 시 서버 저장으로 교체하는 2단계 접근을 권고합니다.

**PM1 vs PM4 — 색상 작업 우선순위**: PM1이 CSS 변수 토큰화를 권고하나, PM4는 기능 버그 우선 해결을 강조합니다. **결론**: `#9E9E9E` → `#757575` 일괄 변경은 WCAG AA 접근성 이슈이므로 기능 버그와 병행 처리하되, CSS 변수 토큰화는 MVP2 이후로 연기합니다.
