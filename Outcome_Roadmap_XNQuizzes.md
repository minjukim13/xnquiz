## Now — Q2 2026 (MVP1 완료)

**테마: 교수자 운영 신뢰도 확보**

> 현재 UI/UX는 완성에 가깝지만, 저장/상태 전이 미구현으로 실사용이 불가능한 상태.
> 교수자가 XN Quizzes를 신뢰하고 반복 사용하려면 "저장된다"는 확신이 선행되어야 한다.

| **아웃컴** | **성공 지표** | **핵심 과제** | **상태** |
|---|---|---|---|
| 교수자가 퀴즈를 출제·수정하고 안심하고 저장할 수 있다 | 새로고침 후 데이터 손실 0건 | 퀴즈 생성/편집 API 연동 (QuizCreate, QuizEdit 저장 로직) | 미착수 |
| 채점 결과가 영속적으로 저장되어 교수자가 신뢰하고 채점을 완료할 수 있다 | 채점 도중 이탈 후 재접속 시 이전 점수 복원 100% | 채점 저장 로직 구현 + 이탈 방지(beforeunload) | 미착수 |
| 잘못된 퀴즈 발행으로 인한 학생 혼란이 사전에 차단된다 | 유효성 오류 발행 건수 0건 | 발행 시 유효성 검증 (문항 수, 총점, 날짜 역전) | 미착수 |
| 채점이 완료되면 퀴즈 상태가 자동으로 전환되어 교수자의 수동 조작 부담이 없다 | 채점 완료 → closed 전환 수동 조작 0회 | 퀴즈 상태 전이 자동화 (draft → open → grading → closed) | 미착수 |
| 교수자가 원하는 시점에 학생에게 성적을 공개할 수 있다 | 성적 조기 노출 CS 민원 0건 | 점수 공개 시점 제어 기능 (수동 공개 / 전체 채점 완료 시 자동 공개) | 미착수 |
| 성적 데이터를 학사 시스템이나 Excel과 연계할 수 있다 | CSV 다운로드 성공률 100% | CSV 다운로드 완성 (현재 부분 구현 → 완료) | 진행 중 |
| 출제 효율이 높아져 교수자가 문항을 재사용할 수 있다 | 문제은행 문항 재사용률 50% 이상 | 문제은행 문항 CRUD 저장 (현재 로컬 state → DB 반영) | 미착수 |

## Next — Q3 2026 (MVP2)

**테마: 학습자 응시 경험 완성**

> MVP1은 교수자 도구. MVP2는 XN Quizzes를 실제 교육 서비스로 완성하는 단계.
> 학습자가 응시하고, 결과를 확인하고, 공정하게 재평가받을 수 있어야 한다.

| **아웃컴** | **성공 지표** | **핵심 과제** | **신뢰도** |
|---|---|---|---|
| 학습자가 실서비스 수준으로 퀴즈를 응시할 수 있다 | 응시율 95% 이상 | 학습자 응시 화면 구현 (문제 풀기, 시간 제한, 자동 제출) | 높음 |
| 학습자가 채점 완료 후 적시에 성적을 확인할 수 있다 | 채점 완료 48시간 이내 학습자 점수 공개 | 채점 완료 → 점수 공개 흐름 구현 + 학습자 알림 | 높음 |
| 공정한 재평가 기회를 통해 학습 동기가 유지된다 | 재응시율 20% 이하 유지 (과도한 재응시 방지) | 재응시 흐름 (횟수 제한, 최고/최신/평균 점수 정책 적용) | 중간 |
| 단답형 문항의 채점 부담이 줄어 교수자가 더 많은 유형을 활용할 수 있다 | 단답형 문항 출제 비율 20% 이상 | 단답형 키워드 자동채점 로직 구현 | 중간 |
| 점수 변경 시 학습자가 즉시 인지하여 성적 신뢰도가 유지된다 | 재채점 후 학습자 미확인율 5% 이하 | 재채점 학생 알림 + 재채점 이력 기록 | 낮음 |

## Later — Q4 2026 이후 (MVP3)

**테마: 기관 통합 및 확장**

> 대학/기업 기관 도입을 위해 기존 LMS 인프라와의 통합이 필요.
> Later 단계는 아웃컴 중심으로 방향만 정의하며, 구체 솔루션은 확정하지 않는다.

| **아웃컴** | **성공 지표** | **핵심 과제** | **의존성** |
|---|---|---|---|
| Canvas LMS를 사용하는 교수자가 추가 학습 없이 XN Quizzes를 즉시 사용할 수 있다 | Canvas → XN 퀴즈 데이터 마이그레이션 수동 작업 0건 | Canvas API 연동 (퀴즈 데이터 동기화) | Canvas API 파트너십, MVP2 완료 |
| 별도 계정 없이 학교 자격증명으로 즉시 로그인할 수 있다 | 가입 단계 이탈률 0% | 학교 SSO 인증 (교수자/학습자 통합 로그인) | 기관별 SSO 연동 계약 |
| 학습자가 모든 강좌의 성적을 한 곳에서 자기주도적으로 관리할 수 있다 | 성적 문의 CS 건수 30% 감소 | 학습자용 성적 조회 포털 | SSO 완료 |
| 학과/강좌 단위 성적 분석으로 교육 품질 개선에 활용할 수 있다 | 학과장/운영자 월간 리포트 활용률 50% 이상 | 학과/강좌 단위 통계 리포트 | Canvas 연동 완료 |

## 변환 노트

| **원래 기능 (출력 중심)** | **변환된 아웃컴 (결과 중심)** | **왜 이 프레이밍인가** |
|---|---|---|
| 채점 저장 로직 구현 | 채점 결과가 영속적으로 저장되어 교수자가 신뢰하고 채점을 완료할 수 있다 | 저장 기능 자체가 목적이 아니라, 교수자가 "채점이 날아가지 않는다"는 신뢰를 갖는 것이 목적 |
| 퀴즈 생성/편집 저장 로직 구현 | 교수자가 퀴즈를 출제·수정하고 안심하고 저장할 수 있다 | 두 기능(생성+편집)은 같은 아웃컴을 위한 과제 — 하나의 아웃컴으로 묶어 중복 제거 |
| 발행 시 유효성 검증 | 잘못된 퀴즈 발행으로 인한 학생 혼란이 사전에 차단된다 | 검증 로직은 수단, 실제 목적은 발행 실수로 인한 학생 피해 방지 |
| 퀴즈 상태 전이 자동화 | 채점이 완료되면 퀴즈 상태가 자동으로 전환되어 교수자의 수동 조작 부담이 없다 | 자동화의 가치는 교수자 조작 횟수 감소, 운영 실수 방지 |
| 학습자 응시 화면 구현 | 학습자가 실서비스 수준으로 퀴즈를 응시할 수 있다 | "구현"은 개발 과제, 실제 가치는 학습자가 실제로 사용할 수 있게 되는 것 |
| Canvas LMS API 연동 | Canvas LMS를 사용하는 교수자가 추가 학습 없이 XN Quizzes를 즉시 사용할 수 있다 | API 연동은 수단, 목적은 기존 Canvas 사용자의 전환 마찰 제거 |
| AI 기반 서술형 자동채점 | (Out-of-scope) | 현 단계에서 명확한 아웃컴-비용 정당성 없음 — 재검토 필요 |

## 로드맵 내러티브 변화 요약

**기존 로드맵 (출력 중심)**
기능 단위 작업 목록으로 구성되어 있어 "무엇을 만드는가"는 명확하지만, "왜 만드는가"와 "이것이 성공했을 때 무엇이 달라지는가"가 보이지 않았다. 우선순위도 Impact/Effort 수치 중심으로, 비즈니스 목적과의 연결이 약했다.

**변환 후 (아웃컴 중심)**
- 기능들이 전략 테마(신뢰도 → 경험 완성 → 통합 확장)로 묶여, 제품이 단계별로 어떤 사용자 가치를 만드는지 명확해진다.
- 성공 지표가 병기되어 "완료"의 기준이 코드 배포가 아닌 사용자 행동 변화로 정의된다.
- 여러 기능(생성 저장 + 편집 저장)이 동일한 아웃컴을 위한 수단으로 묶여, 중복과 분산이 제거된다.
- Out-of-scope 항목(AI 서술형 채점)에 대한 재검토 근거가 명시된다.
